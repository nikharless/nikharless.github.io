WCAG | 1.1.1 Non-text Content

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • CAPTCHAs must have a text alternative that describes its purpose (but not the visual elements).
  • CAPTCHAs must have alternate forms for different types of sensory perception.
Success: If you could not see the CAPTCHA, you would still know what it was and be able to complete it.

Testing Logic

Are there any CAPTCHA widgets on the page?

  • Yes; Does the CAPTCHA widget have a text alternative that describes its purpose?
    • Yes; Is there an alternate non-visual method to complete the CAPTCHA?
      • Yes; PASS
      • No; Ensure a non-visual alternative (e.g., audio) is available to complete the CAPTCHA.
    • No; Add alternative text describing the purpose of the CAPTCHA.
  • No; PASS

Web Test Procedure

Desktop

  1. Locate each CAPTCHA.
  2. Verify that it has a text alternative that describes its purpose.
  3. Verify that it includes an alternate non-visual form (e.g., audio).

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to the CAPTCHA.
  3. Verify that it has a text alternative that describes its purpose.
  4. Verify that it includes an alternate non-visual form (e.g., audio).

Failures

  • F1.1.1.g.a CAPTCHA does not have a text alternative. (Medium)
  • F1.1.1.g.b Existing text alternative does not describe the purpose of the CAPTCHA. (Medium)
  • F1.1.1.g.c CAPTCHA has no non-visual version. (High)
