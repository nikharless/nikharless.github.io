WCAG | 1.1.1 Non-text Content

Meaningful images must have a descriptive text alternative that conveys the same information as the image in context.

Success: If you could not see the image, you would still get all the information intended to be conveyed by the image.

Text alternatives for informative images include the following:

  • alt attribute value
  • title attribute value
  • aria-label property
  • aria-labelledby property
  • CSS clipped text

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Meaningful images must have a descriptive text alternative that conveys the same information as the image in context.
    • The same image may have different intentions in different contexts, so it's important to take this into consideration when deciding whether the same information is conveyed by the text alternative.
    • If the image does not provide any information not already conveyed elsewhere on the page, it may be considered redundant. Check against 1.1.1.d Alternative Text (Decorative Images).
  • If the image contains meaningful text, the text alternative should contain that exact text.
  • Text alternatives for informative images should not include words such as "image" or "graphic".
  • Trademarks, copyright symbols, etc. on an image should be included in the text alternative.
  • Text alternatives should be as succinct as possible.

Testing Logic

Are there informative, non-interactive images on the page?

  1. Yes; Does the image have alternative text?
    1. Yes; Does the text alternative serve the same purpose as the image?
      1. Yes; PASS
      2. No; Change the text to convey the same information as the image.
    2. No; Add alternative text to the active image.
  2. No; PASS

Web Test Procedure

Desktop

  1. Run axe DevTools or WAVE to flag missing text alternatives.
  2. Locate each informative image on the page using one or more of the following methods:
    • Mouse: Hover over images, noting cursor change if they are interactive.
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  3. Verify that each image has a text alternative which accurately describes its purpose.

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to each informative image.
  3. Verify that the name spoken by the screen reader accurately describes the purpose of the image.

Mobile App Test Procedure

  • Follow the procedure for Web Mobile.

PDF Test Procedure

Find missing text alternatives.

PDF Accessibility Checker

  1. Look for failures under Logical Structure > Alternative Descriptions > Alternative text for "Figure" structure elements.

Adobe Acrobat Pro

  1. Select Accessibility > Accessibility Check in the Task pane to open the Accessibility Checker Options dialog.
  2. Select "Alternate Text and Headings" from the Category drop-down, and make sure "Figures require alternate text" is checked.

Verify the text alternatives are sufficient.

  1. Locate each informative image in the PDF using one or more of the following methods:
    • Visually
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  2. Verify that each image has a text alternative which accurately describes its purpose.

Failures

  • F1.1.1.b.a Informative image is missing a text alternative. (High)
  • F1.1.1.b.b Existing text alternative does not serve the same purpose as the informative image. (Medium)
  • F1.1.1.b.c Existing text alternative does not include informative image text. (Medium)
  • F1.1.1.b.d Text alternative does not match dynamic image. (Medium)
