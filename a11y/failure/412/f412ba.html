Expected Result

Custom controls have accessible names.

Actual Result

A custom control does not have an accessible name.

Solution

HTML

Provide an accessible name for the custom control, using one of the following methods:

  • Use the aria-label attribute and set the value to the name, OR
  • Use the aria-labelledby attribute and point it to a text element which can function as the accessible name, OR
  • Use text between the start and end tags of the element as the accessible name.

Examples

<button>

Not Applicable

The following items are not applicable to this failure:

  • Native HTML controls:
    <input type="checkbox"> Subscribe to the mailing list
    File under 4.1.2.a Name, Role, Value
  • Custom controls that function as native HTML controls:
    <div role="checkbox" tabindex="0" aria-checked="false">Subscribe to the mailing list
    File under 4.1.2.a Name, Role, Value
  • Hidden:
    <div style="display:none;">
    Does not appear in accessibility tree
  • Role of "none":
    <div role="none">
    Does not appear in accessibility tree
  • No role:
    <div>Search</div>
    File under F4.1.2.a.g Button does not have a valid role.

Why It's Important

Assistive technology only works well when code is done properly.
