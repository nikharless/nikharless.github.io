Expected Result

Tree grids have all appropriate roles and attributes.

Actual Result

A tree grid does not have all the appropriate roles or attributes.

Solution

Tree View

  • Use the "treegrid" role on the main tree grid container.
  • Use the "row" role on each row container.
  • Use the "columnheader" role on each column header cell.
  • Use the "rowheader" role on each row header cell.
  • Use the "gridcell" role on each non-header cell.
  • Use the aria-expanded attribute on each expandable row or on a cell within the row; set it to "true" if expanded or "false" if collapsed.
  • If the tree grid allows multiple selections, use the aria-multiselectable="true" attribute on the treegrid element.
  • Use the aria-selected attribute on each selectable row; set to "true" if selected or to "false" if not. (Only one row may be marked as true if aria-multiselectable="true" is not set.)
  • Use the aria-label OR the aria-labelledby attribute on the tree element to provide an accessible name.
  • If the tree grid has a caption, use aria-describedby on the treegrid element and point it to the id of the caption element.
  • If the tree grid has sort functions, use the aria-sort attribute on the header cell element for the sorted column or row, and set it to an appropriate value.
