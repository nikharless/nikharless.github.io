Expected Result

Listboxes have all appropriate roles and attributes.

Actual Result

A listbox does not have all the appropriate roles or attributes.

Solution

  • Use the "listbox" role on the list (<ul>) element.
  • Use tabindex="0" on the listbox element to allow it to have keyboard focus.
  • Use aria-multiselectable="true" on the listbox element if the listbox allows multiple selections.
  • Use aria-activedescendant on the listbox element.
    • If the listbox is empty, set it to null (aria-activedescendant="").
    • If an option in the listbox has visual keyboard focus, set it to the id of the option.
    • When navigation keys such as arrow keys are pressed, change the value with JavaScript.
  • Use the "option" role on each list option (<li>) element.
  • Use aria-selected on the currently selected option and on each option in listboxes that allow multiple selections.
    • Set aria-selected="true" on the currently selected option.
    • If the listbox allows multiple selections, set aria-selected="false" on each option that is not selected.
  • Use aria-hidden="true" on any checkmark visuals to prevent them from being spoken with the accessible name.
