Expected Result

Custom text input has the contenteditable attribute.

Actual Result

Custom text input does not have the contenteditable attribute.

Solution

Use the contenteditable attribute on the custom text input, even if it is marked up as read-only.

Examples

Custom form fields

Email
<div role="textbox" contenteditable="true" style="border: 1px solid black;">Email</div>

Not Applicable

The following items are not applicable to this failure:

  • Hidden with CSS:
    <div role="textbox" style="display:none;">
    Does not appear in accessibility tree
  • Hidden with ARIA:
    <div role="textbox" aria-hidden="true">
    Does not appear in accessibility tree

Why It's Important

Assistive technology only works well when code is done properly.
