Expected Result

Custom controls have valid ARIA attributes and values.

Actual Result

A custom control has an invalid ARIA attribute or value.

Solution

Ensure states, properties, and values are valid.

Examples

Slider

{% include examples/slider-rating.html %} 
<div id="id-rating-label">Accessibility is the right thing to do.</div>
<div id="id-rating" role="slider" tabindex="0" aria-valuemin="0" aria-valuemax="10" aria-labelledby="id-rating-label">

Toolbar

{% include examples/toolbar.html %} 
<div role="toolbar" aria-controls="textarea1" aria-label="Text Formatting">

Not Applicable

The following items are not applicable to this failure:

  • Native HTML controls:
    <input type="checkbox"> Subscribe to the mailing list
    File under 4.1.2.a Name, Role, Value
  • Custom controls that function as native HTML controls:
    <div role="checkbox" tabindex="0" aria-checked="false">Subscribe to the mailing list
    File under 4.1.2.a Name, Role, Value
  • Hidden:
    <div style="display:none;">
    Does not appear in accessibility tree
  • Role of "none":
    <div role="none">
    Does not appear in accessibility tree

Why It's Important

Assistive technology only works well when code is done properly.
