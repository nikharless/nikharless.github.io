Expected Result

All content can be accessed when the screen is resized.

Actual Result

Elements are cropped or rendered unreachable when the content is resized.

Solution

  • Use a layout which reflows when the screen size changes, AND
  • Use relative measurements for fonts, controls, images, and containers

Examples

Responsive Design

The following working examples resize and reflow elements to fit them within the viewport.

  • Working example: Using media queries and grid CSS to reflow columns
  • Working example: Using CSS Flexbox for Reflow
  • Working example: Using Adjustable Labels and Inputs for Reflow

Preformatted Text

By default, pre-formatted text does not wrap, causing it to be cropped and unreadable when the content is wider than the container width. You must explicitly set it to wrap.

<a href="https://www.beavercountypa.gov/">
  <img src="BeaverCountyLogo.png" alt="Beaver County, Pennsylvania. Divided by rivers, united by people">
</a>
pre {
    background-color: #f1f1f1;
    overflow-x: auto;
    white-space: pre-wrap;
    white-space: -moz-pre-wrap;
    white-space: -pre-wrap;
    white-space: -o-pre-wrap;
    word-wrap: break-word;
}

Why It's Important

Users with visual disabilities need to be able to view and use the content when resized.
