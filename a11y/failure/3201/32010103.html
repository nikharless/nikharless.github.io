Expected Result

When a component gains focus, there is not a significant change of content before the focus.

Actual Result

When a component gains focus, there is a significant change of content before the focus.

Solution

  • Add new content after the focus, OR
  • Notify assistive technology of the change.

Failing Example

Change of Content Before Focus

When the Email field gains focus, checkboxes are revealed at the top of the form.

Select the notifications you would like to receive via email.

Passing Example

No Change of Content

In this example, the checkboxes are always visible and appear after the email field.

Select the notifications you would like to receive via email.

Not Applicable

  • Context changes on input:
    File under {{ site.data.3202.3202[3202].name }}.

Disabilities Affected

  • Blind/Visual Impairment: Unexpected changes of context can cause significant confusion if the user can't see what has happened.
  • Cognitive Impairment: Unexpected changes of context can cause the user to become disoriented.
  • Motor Impairment: Unexpected changes of context can make content significantly more difficult to navigate.
