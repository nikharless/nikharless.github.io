Expected Result

Complex tables use headers and id attributes to associate headers with data.

Actual Result

Table is too complex for the header and data relationships to be identified using <th> and scope, but the headers and id association is not used.

Solution

  • Add an id attribute to each header cell, set to a unique value.
  • Add a headers attribute to each data cell and set the value to a space-separated list of header cell id values that label the cell.

Example

Table with Changing Column Headers

Note: NVDA will always read both column headers (for example, Freddie and Phillip) even when the table is marked up correctly. To avoid this, it's best practice to restructure the data into a simple table.

Cats
Freddie Joaquin
Sex Female Male
Color Tortoiseshell White with gray cap and saddle
Weight 12 lbs 18 lbs
Phillip Myrtle
Sex Male Female
Color Black with white spot on belly Tortoiseshell
Weight 14 lbs 7 lbs 
<table>
    <caption>Cats</caption>
    <tr>
        <th id="blank"></th>
        <th id="fred" headers="blank">Freddie</th>
        <th id="joaq" headers="blank">Joaquin</th>
    </tr>
    <tr>
        <th id="sex1" headers="blank">Sex</th>
        <td headers="fred sex1">Female</td>
        <td headers="joaq sex1">Male</td>
    </tr>
    ...
     <tr>
        <th id="blank2"></th>
        <th id="phil" headers="blank2">Phillip</th>
        <th id="myrt" headers="blank2">Myrtle</th>
    </tr>
    <tr>
        <th id="sex2" headers="blank2">Sex</th>
        <td headers="phil sex2">Male</td>
        <td headers="myrt sex2">Female</td>
    </tr>
    ...
</table>

Table with Multiple Levels of Headers

Note: NVDA will always read the full span headers (in this case, Alaska and Arizona) even when the table is marked up correctly. To avoid this, it's best practice to break the complex table up into simple tables, such as one for each state.

High Temperatures - 2023
January June October
Alaska
Anchorage 50 85 64
Fairbanks 52 96 72
Juneau 60 86 61
Arizona
Flagstaff 66 96 85
Phoenix 88 122 107

<table>
    <caption>High Temperatures - 2023</caption>
    <tr>
        <th id="blank"></th>
        <th id="jan" headers="blank">January</th>
        <th id="jun" headers="blank">June</th>
        <th id="oct" headers="blank">October</th>
    </tr>
    <tr>
        <th colspan="4" id="ak" headers="blank">Alaska</th>
    </tr>
    <tr>
        <th id="anch" headers="ak">Anchorage</th>
        <td headers="jan ak anch">50</td>
        <td headers="jun ak anch">85</td>
        <td headers="oct ak anch">64</td>
    </tr>
    ...
</table>

Why It's Important

Table relationships are visible at a glance for sighted users. Assistive technology users need to get this relationship information from the semantic structure and to be able to easily navigate the data.
