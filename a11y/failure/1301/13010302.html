Expected Result

If the label for an individual control in a group of controls does not provide enough context to understand the purpose of the control, the control is programmatically associated with the group label.

Actual Result

The label for an individual radio button in a radio group does not provide enough context to understand the purpose of the radio button, and the radio button is not programmatically associated with the group label.

Solution

Native HTML (Preferred Method)

Use the <fieldset> and <legend> tags to group the radio buttons.

  • Enclose the entire group of radio buttons in a <fieldset> element.
  • Place the group label inside a <legend> element as a child of the <fieldset> element.
OR

ARIA

Use ARIA to group the radio buttons.

  • Locate or create a parent element for the radio group.
  • Add the role="radiogroup" attribute to the parent element.
  • Add an id attribute to the tag containing the group label text, and set it to a unique value.
  • Add an aria-labelledby attribute to the parent (radiogroup) element and point it to the id of the group label tag.

Example

Native HTML (Preferred Method)

Pancake Toppings 

<fieldset>
    <legend>Pancake Toppings</legend>
    <input type="radio" name="toppings" id="syrup">
    <label for="syrup">Maple syrup</label><br>
    <input type="radio" name="toppings" id="berries">
    <label for="berries">Blueberries</label><br>
    <input type="radio" name="toppings" id="cream">
    <label for="cream">Whipped cream</label>
</fieldset>

ARIA

Pancake Toppings



<p id="toppingsLabel">Pancake Toppings</p>
<div role="radiogroup" aria-labelledby="toppingsLabel">
    <input type="radio" name="toppings" id="syrup">
    <label for="syrup">Maple syrup</label><br>
    <input type="radio" name="toppings" id="berries">
    <label for="berries">Blueberries</label><br>
    <input type="radio" name="toppings" id="cream">
    <label for="cream">Whipped cream</label>
</div>

Why It's Important

Assistive technology users browsing through a form will not hear group labels unless they are programmatically associated. Screen readers in forms mode will ignore any text that is not programmatically associated with form controls.
