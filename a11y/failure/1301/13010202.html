Expected Result

Header cells are marked up semantically as header cells.

Actual Result

Header cells are not marked up semantically as header cells.

Solution

Mark up table header cells as semantic header cells.
  • Use native HTML tables (<th></th>).
  • If this is not possible, follow the recommended ARIA pattern for tables (role="columnheader" or role="rowheader").

Example

Simple Table with Column Headers

Name Time
John 18 min
Jane 20 min 
<table>
    <tr>
        <th>Name</th>
        <th>Time</th>
    </tr>
    <tr>
        <td>John</td>
        <td>18 min</td>
    </tr>
    <tr>
        <td>Jane</td>
        <td>20 min</td>
    </tr>
</table>

Simple Table with Row Headers

Name John Jane
Time 18 min 20 min 
<table>
    <tr>
        <th>Name</th>
        <td>John</td>
        <td>Jane</td>
    </tr>
    <tr>
        <th>Time</th>
        <td>18 min</td>
        <td>20 min</td>
    </tr>
</table>

Why It's Important

Table relationships are visible at a glance for sighted users. Assistive technology users need to get this relationship information from the semantic structure and to be able to easily navigate the data.
