Expected Result

Data tables are identified programmatically as tables.

Actual Result

A data table is not identified programmatically as a table.

Solution

Mark up data tables as programmatic tables.
  • Use native HTML tables, preferred (e.g., <table>, <th>, <td>), OR
  • Use ARIA to assign table roles (e.g., role="table", role="columnheader", role="rowheader", and role="cell").

Passing Examples

Native HTML (preferred)

Park Top Feature
Denali National Park Denali - North America's highest peak
Great Smoky Mountains National Park Clingmans Dome

Source

<table>
    <tr>
        <th>Park</th>
        <th>Top Feature</th>
    </tr>
    <tr>
        <td>Denali National Park</td>
        <td>Denali - North America's highest peak</td>
    </tr>
    <tr>
        <td>Great Smoky Mountains National Park</td>
        <td>Clingmans Dome</td>
    </tr>
</table>

ARIA

Park Top Feature
Denali National Park Denali - North America's highest peak
Great Smoky Mountains National Park Clingmans Dome

Source

<div role="table" aria-label="National Parks">
    <div role="row">
        <span role="columnheader">Park</span>
        <span role="columnheader">Top Feature</span>
    </div>
    <div role="row">
        <span role="cell">Denali National Park</span>
        <span role="cell">Denali - North America's highest peak</span>
    </div>
    <div role="row">
        <span role="cell">Great Smoky Mountains National Park</span>
        <span role="cell">Clingmans Dome</span>
    </div>
</div>

Why It's Important

  • Blind/visually impaired: Screen reader users rely on programmatic markup to understand how content is arranged. Table relationships are visible at a glance for sighted users. Assistive technology users need to get this relationship information from the semantic structure and to be able to easily navigate the data.
