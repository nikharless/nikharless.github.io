Expected Result

Headings are marked up hierarchically.

Actual Result

Headings are marked up semantically as headings, but the level hierarchy is incorrect.

Solution

Native HTML (Preferred Method)

Use the <h1>, <h2>, <h3>, <h4>, <h5> and <h6> tags to mark up headings in hierarchical order.

OR

ARIA

Use the aria-level attribute set to the correct heading level (1-6) to mark up headings in hierarchical order.

Example

Native HTML (Preferred Method)

<h1>Page-Level Heading</h1>
    <h2>Section Heading</h2>
        <h3>Level 3 Section Heading</h3>
        <h3>Another Level 3 Section Heading</h3>
    <h2>Another Level 2 Section Heading</h2>
        <h3>Level 3 Heading for This Section</h3>
            <h4>Level 4 Heading</h4>
            etc.

ARIA

<div role="heading" aria-level="1">Page-Level Heading</div>
    <div role="heading" aria-level="2">Section Heading</div>
        <div role="heading" aria-level="3">Level 3 Section Heading</div>
        <div role="heading" aria-level="3">Another Level 3 Section Heading</div>
    <div role="heading" aria-level="2">Another Level 2 Section Heading</div>
        <div role="heading" aria-level="3">Level 3 Heading for This Section</div>
            <div role="heading" aria-level="4">Level 4 Heading</div>
            etc.

Why It's Important

Visual users can see at a glance how information on the page is structured. Assistive technology users rely on semantic markup to understand how content is arranged.
