Expected Result

Form elements are programmatically associated with their corresponding table headers.

Actual Result

A form element is not programmatically associated with its corresponding table header.

Solution

  • Use native HTML table header elements (<tr>) along with scope="col" or scope="row" to define the header/cell relationship, OR
  • Use ARIA role="rowheader" or row="columnheader" on the table header elements.

Native HTML is the preferred solution and should be used whenever possible.

Failing Example

HTML

First Name Last Name 
<table>
    <tr>
        <td><b>First Name</b></td>
        <td><b>Last Name></b></td>
    </tr>
    <tr>
        <td><input type="text" id="fname1"></td>
        <td><input type="text" id="lname1"></td>
    </tr>
        <tr>
        <td><input type="text" id="fname2"></td>
        <td><input type="text" id="lname2"></td>
    </tr>
</table>

Passing Examples

HTML

First Name Last Name 
<table>
    <tr>
        <th scope="col">First Name</th>
        <th scope="col">Last Name</th>
    </tr>
    <tr>
        <td><input type="text" id="fname1"></td>
        <td><input type="text" id="lname1"></td>
    </tr>
        <tr>
        <td><input type="text" id="fname2"></td>
        <td><input type="text" id="lname2"></td>
    </tr>
</table>

ARIA

First Name
Last Name
<div role="table">
    <div role="row">
        <div role="columnheader">First Name</div>
        <div role="columnheader">Last Name</div>
    </div>
    <div role="row">
        <div role="cell"><input type="text" id="fname1"></div>
        <div role="cell"><input type="text" id="lname1"></div>
    </div>
    <div role="row">
        <div role="cell"><input type="text" id="fname2"></div>
        <div role="cell"><input type="text" id="lname2"></div>
    </div>
</div>

Disabilities Affected

  • Blind/visual impairment: Screen reader users hear the table header cell text when they navigate to the form element cell without having to search around to find it.
