WCAG | 1.1.1 Non-text Content | 1.1.1.h Audio or Video (Text Alternative)

Expected Result Each audio element must have a text alternative which summarizes its purpose.
Actual Result The audio element does not have a text alternative.
Solution Add a text alternative which summarizes the content of the audio:
  • Add an aria-label attribute to the element, OR
  • Add an aria-labelledby attribute to the element, set to the id value of the text to use as a label
Example

HTML <audio>

What a horse says:

<What a horse says:>
<audio controls aria-label="horse neighing">
    <source src="/a11y/assets/media/f111ha-horse.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>
Why It's Important Blind and visually impaired users need to have a text alternative to embedded audio so they know what it is without having to play it.
