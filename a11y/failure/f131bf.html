Expected Result

Complex tables use headers and id attributes to associate headers with data.

Actual Result

Table is too complex for the header and data relationships to be identified using <th> and scope, but the headers and id association is not used.

Solution

  • Add an id attribute to each header cell, set to a unique value.
  • Add a headers attribute to each data cell and set the value to a space-separated list of header cell id values that label the cell.

Example

Table with Multiple Rows of Headers

January June October
Alaska
Anchorage 50 85 64
Fairbanks 52 96 72
Juneau 60 86 61
Arizona
Flagstaff 66 96 85
Phoenix 88 122 107
Tucson 87 117 102 
<table>
    <tr>
        <th scope="col">Name</th>
        <th scope="col">Time</th>
        <th scope="col">Attempts</th>
    </tr>
    <tr>
        <th scope="row">John</th>
        <td>18 min</td>
        <td>3</td>
    </tr>
    <tr>
        <th scope="row">Jane</th>
        <td>20 min</td>
        <td>1</td>
    </tr>
</table>

Why It's Important

Table relationships are visible at a glance for sighted users. Assistive technology users need to get this relationship information from the semantic structure and to be able to easily navigate the data.
