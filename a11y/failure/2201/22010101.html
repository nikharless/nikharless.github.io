Expected Result

If the content has a time limit, the user must be given options to turn off, adjust, or extend that time limit before time expires.

Actual Result

Time expires without giving the user a warning or options to turn off or adjust the time limit.

Solution

Session Time Limits

  • Provide a checkbox on the first page of a multipart form that allows users to ask for a longer session time limit or no session time limit, OR
  • Provide a means to turn off the time limit before time expires

Time Limits Controlled by a Script on the Page

  • Provide a means to turn off the time limit before time expires, OR
  • Provide a means to adjust the time limit to at least ten times the length of the default setting before time expires, OR
  • Provide a warning before time expires AND
    • For a period of at least 20 seconds, allow the user to extend the time limit with a simple action (e.g., pressing the spacebar), AND
    • Provide a means for the user to extend the time limit at least 10 times.

Time Limits on Reading

  • Provide a means to pause and restart the content from where it was paused, OR
  • Provide a means to turn off the time limit before time expires, OR
  • Provide a means for users to display moving, scrolling, or auto-updating text in a static area.

Examples

  • A site logs the user out after a period of inactivity to protect their data. The site provides a dialog asking if the user needs more time. After 30 seconds, if there is no response, the user is logged out.

Why It's Important

  • Blind/visually impaired: Users need more time to locate things on screen and to read. Screen reader users need more time to understand screen layouts, find information, and operate controls.
  • Motion impaired: Users need more time to react, type, and complete activities.
  • Cognitively impaired: Users need more time to read and understand the information.
