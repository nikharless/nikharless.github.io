Expected Result

All functionality is available through the keyboard interface.

Actual Result

A control cannot be accessed using only a keyboard

Solution

  • Use standard HTML for links and controls. The following elements pass this checkpoint by default.
    • <a>
    • <button>
    • <fieldset>
    • <input>
    • <select>
    • <textarea>
    OR
  • Use tabindex="0" to add custom controls to the tab order, AND
  • Provide keyboard event handlers alongside all mouse event handlers, OR
  • Provide alternate, keyboard accessible controls that accomplish the same function

Examples

Native HTML

Native HTML is keyboard accessible by default.

Option 1
Option 2 
<a href="page.html">Click this link</a>
<script>
    htmlButton.onclick = function (e) { doStuff(); };
    htmlButton.onkeyup = function (e) { if(e.keyCode == 13) { doStuff(); } };
</script>

Custom Control

Using onclick and onkeyup allows activation with both the mouse and the keyboard.

Do Stuff
<div id="customButton" role="button" tabindex="0">Do Stuff</div>
<script>
    customButton.onclick = function (e) { doStuff(); };
    customButton.onkeyup = function (e) { if(e.keyCode == 13) { doStuff(); } };
</script>

Why It's Important

Some users with vision disabilities or hand tremors are unable to use a mouse well or at all.
