Expected Result

There is no loss of data when a user's authenticated session expires and they log back in.

Actual Result

Data is lost when the user's session expires, and they have to start their progress over.

Solution

Provide a mechanism to save data between authenticated sessions.

Examples

  • A shopping website saves cart contents and check-out progress when a user's session times out. When the user logs back in, they are taken to the same step in the process with all previously entered information retained.
  • A contact form includes a Save Draft button to allow the user to save their progress entering large amounts of text into a field. It also includes the session time limit in the instructions and provides a warning when the session is about to time out.

Why It's Important

Users with various disabilities often need more time to complete activities.

References
