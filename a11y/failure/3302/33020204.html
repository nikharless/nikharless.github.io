Expected Result

Form input labels and instructions have enough information for the user to know what input is expected.

Actual Result

Form input instructions are provided, but they are not read by assistive technology.

Solution

Ensure the instructions for completing the form inputs are accessible to assistive technology.

Examples

Required Fields

The required status of the fields is communicated both visually and programmatically.

All fields are required. 
<form>
    <legend>All fields are required.</legend>
    <label for="dob">Date of Birth (<em>MM/DD/YYYY</em>)<br>
        <input type="text" id="dob" required>
    </label>
    <label for="zip">Postal Code (<em>XXXXX</em> or <em>XXXXX-XXXX</em>)<br>
        <input type="text" id="zip" required>
    </label>
</form>

Associating Instructions with aria-describedby

This should be your company email.

<form>
    <label for="email">Email Address</label>
    <input type="email" id="email" aria-describedby="info">
    <p id="info">This should be your company email.</p>
    <input type="submit" value="Submit">
</form>

Why It's Important

Everyone, especially those with cognitive disabilities, will know how to respond.

References

  • ARIA1: Using the aria-describedby property to provide a descriptive label for user interface controls
  • ARIA9: Using aria-labelledby to concatenate a label from several text nodes
