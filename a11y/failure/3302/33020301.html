Expected Result

Form input labels and instructions have enough information for the user to know what input is expected.

Actual Result

Form input label does not indicate the input's required status.

Solution

Include the required status of form inputs in the label or instructions.

Examples

Individual Labels

Marking required fields with an asterisk (*) minimally passes. Screen readers should read the asterisk, and it is a common enough practice that it is usually understood to mean "required". An additional note such as "Fields marked with * are required" should be provided, as well as the required attribute on the <input>.

Using the word "required" is a better option, because the screen reader will read it. Avoid using abbreviations like "req".

Form-Level Instructions

All fields are required.

Optional Fields Labeled

Why It's Important

Everyone, especially those with cognitive disabilities, will know how to respond.
