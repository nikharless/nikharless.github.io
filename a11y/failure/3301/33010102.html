Expected Result

Error messages identify items with an error in visible text.

Actual Result

An error message identifies the item(s) in error but not with visible text.

Solution

  • Identify the item(s) with an error in an error message with visible text, OR
  • Modify the text of the component's label to indicate it has an error.

Examples

List of Errors

It's not required to include links to the fields in error. However, doing so increases accessibility for keyboard users, especially in long forms.

Please correct the following errors:

  • First Name is required.
  • Date of Birth must be in format: mm/dd/yyyy.

All fields are required.

Error Next to Field

All fields are required.

Error Alert

Submit the form to see the error.

All fields are required.

Error in Label

All fields are required.

Why It's Important

  • Blind/vision impaired: Users are able to perceive that an error has occured, where it has occurred, and why.
  • Colorblind: Users are able to perceive that an error has occured, where it has occurred, and why.
  • Cognitive impairments: Users are able to understand that an error has occured, where it has occurred, and why.
