WCAG | 1.1.1 Non-text Content | 1.1.1.i Text as Non-Text Content (Text Alternative)

Expected Result Text characters used to represent images have a text alternative describing their meaning.
Actual Result Text characters used to represent images do not have a text alternative describing their meaning.
Solution
  1. Enclose the text image in a <span> element and give it role="img".
  2. Add an aria-label attribute to the <span> element and set it to a text alternative which describes its meaning.
Examples

Emoticons

:)

>_>

<span role="img" aria-label="smile"> :) </span>
<span role="img" aria-label="sideways glance"> >_> </span>

ASCII Art

      \    /\
       )  ( ')
      (  /  )
       \(__)|
 
<span role="img" aria-label="cat"><pre>      \    /\
       )  ( ')
      (  /  )
       \(__)|</pre></span>
Why It's Important Without a text alternative, blind and visually impaired users would hear the actual text characters rather than what the author intended them to mean.
