Expected Result

If the label for an individual control in a group of controls does not provide enough context to understand the purpose of the control, the control is programmatically associated with the group label.

Actual Result

The label for an individual form control in a group of controls does not provide enough context to understand the purpose of the control, and the control is not programmatically associated with the group label.

Solution

Native HTML (Preferred Method)

Use the <fieldset> and <legend> tags to group the controls.

  • Enclose the entire group of controls in a <fieldset> element.
  • Place the group label inside a <legend> element as a child of the <fieldset> element.
OR

ARIA

Use ARIA to group the controls.

  • Locate or create a parent element for the group of controls.
  • Add the role="group" attribute to the parent element.
  • Add an id attribute to the tag containing the group label text, and set it to a unique value.
  • Add an aria-labelledby attribute to the parent (group) element and point it to the id of the group label tag.

Example

Native HTML (Preferred Method)

Shipping Address 


<fieldset>
    <legend>Shipping Address</legend>
    <label for="streetAddress">Street address:</label>
    <input type="text" id="streetAddress"><br>
    <label for="city">City:</label>
    <input type="text" id="city"><br>
    <label for="state">State:</label>
    <input type="text" id="state"><br>
    <label for="zip">Zip:</label>
    <input type="text" id="zip">
</fieldset>

ARIA

Shipping Address




<p id="shippingLabel">Shipping Address</p>
<div role="group" aria-labelledby="shippingLabel">
    <label for="streetAddress">Street address:</label><br>
    <input type="text" id="streetAddress">
    <label for="city">City:</label><br>
    <input type="text" id="city">
    <label for="state">State:</label>
    <input type="text" id="state">
    <label for="zip">Zip:</label>
    <input type="text" id="zip">    
</div>

Why It's Important

Assistive technology users browsing through a form will not hear group labels unless they are programmatically associated. Screen readers in forms mode will ignore any text that is not programmatically associated with form controls.
