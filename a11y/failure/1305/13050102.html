Expected Result

Form inputs that collect personal data that map to items on W3's Input Purposes for User Interface Components use the correct autocomplete value.

Actual Result

A form input that collects personal data that maps to an item on the list uses an incorrect autocomplete value.

Solution

  • Set the autocomplete attribute to the semantically correct value.

Failing Example

Personal Information

An incorrect autocomplete value will enter the wrong information into the field.


<label for="firstName">First Name 
        <input type="text" autocomplete="name">
    </label><br>
    <label for="lastName">Last Name 
        <input type="text" autocomplete="name">
    </label>

Passing Example



<label for="firstName">First Name 
        <input type="text" autocomplete="given-name">
    </label><br>
    <label for="lastName">Last Name 
        <input type="text" autocomplete="family-name">
    </label><br>
    <label for="email">Email Address 
        <input type="text" autocomplete="email">
    </label>

Not Applicable

  • Form inputs that don't map to items on the list
  • Incorrect values:
    File under {{ site.data.1305.1305[13050102].name }}.
  • Invalid values:
    File under {{ site.data.1305.1305[13050103].name }}.

Disabilities Affected

  • Motor Impairment: Users with dexterity impairments may have to exert a lot of time and effort entering information.
  • Cognitive Impairment: Users with language, memory, or executive function issues may struggle with filling in information.
