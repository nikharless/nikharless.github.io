Expected Result

Form inputs which collect personal data that map to items on W3's Input Purposes for User Interface Components use a valid autocomplete value.

Actual Result

Form inputs which collect personal data that maps to items on the list use an invalid autocomplete value.

Solution

  • Set the autocomplete attribute to a valid value. The text must exactly match an item on the list.

Example



<label for="firstName">First Name 
    <input type="text" autocomplete="given-name">
</label><br>
<label for="lastName">Last Name 
    <input type="text" autocomplete="family-name">
</label><br>
<label for="email">Email Address 
    <input type="text" autocomplete="email">
</label>

Why It's Important

An incorrect semantic value will enter the wrong information into the field. An invalid value will not enter anything.

References

  • H98: Using HTML 5.2 autocomplete attributes
