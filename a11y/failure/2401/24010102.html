Expected Result

A method is provided to skip blocks of content repeated across pages.

Actual Result

A skip link is provided, but the keyboard focus does not move to the correct location when it is activated.

Solution

Ensure that keyboard focus actually moves to the correct location, so that when the user presses Tab the focus moves where expected. It is not enough for the screen to just visually scroll.

Examples

Why It's Important

Screen reader and keyboard-only users are not able to scroll past blocks of content or go directly to the main content.

References

  • Understanding Success Criterion {{ sc.name }}
  • G1: Adding a link at the top of each page that goes directly to the main content area
  • G123: Adding a link at the beginning of a block of repeated content to go to the end of the block
  • G124: Adding links at the top of the page to each area of the content
