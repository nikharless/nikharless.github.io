Expected Result

A keyboard-accessible method is provided to skip repeated blocks of content.

Actual Result

A keyboard-accessible bypass method is provided, but the keyboard focus does not move to the correct location when it is activated.

Solution

Ensure that keyboard focus actually moves to the correct location, so that when the user presses Tab the focus moves where expected. It is not enough for the screen to just visually scroll.

Examples

Smooth Scroll

Disabilities Affected

  • Blind: Screen reader users may have to listen to all header content and dozens of links on every page before the main content is reached.
  • Vision impaired: Screen magnifier users may have to search through a large block of content before the main content is reached.
  • Motion impaired: Keyboard users and users of assistive technology that uses the keyboard interface may have to navigate through dozens of links before reaching the main content.
  • Cognitively impaired: Users may benefit when links are grouped into lists.
