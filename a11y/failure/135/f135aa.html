Expected Result

Form inputs which collect personal data that map to items on W3's Input Purposes for User Interface Components use the autocomplete attribute set to the matching value on the list.

Actual Result

Form inputs which collect personal data that maps to items on the list do not have an autocomplete attribute.

Solution

Add the autocomplete attribute, and set it to the correct matching value on the list.

Example



<label for="firstName">First Name 
    <input type="text" autocomplete="given-name">
</label><br>
<label for="lastName">Last Name 
    <input type="text" autocomplete="family-name">
</label><br>
<label for="email">Email Address 
    <input type="text" autocomplete="email">
</label>

Why It's Important

Users with dexterity impairments may have to exert a lot of time and effort entering information. Users with cognitive disabilities may have difficulty remembering information that needs to be entered.
