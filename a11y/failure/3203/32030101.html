Expected Result

Navigation mechanisms which appear across multiple web pages appear in the same relative order with regard to other repeated components on each page.

Actual Result

Navigation mechanisms which appear across multiple web pages do not appear in the same relative order with regard to other repeated components on each page.

Solution

Consistently order navigation that repeats across multiple pages.

  • Do not change the order of menu items between pages.
  • Do not change the order of navigational elements relative to other content in the DOM.
  • Removing or inserting items (such as submenus) is allowed.

Failing Example

Navigation Link Order Changes

Below are example navigation bars for a Home page and an About page for the same site. Each page places the current page first in the menu order. Therefore, the links are not in the same relative order on each page.

  • Services

Passing Example

Navigation Link Order Consistent

Below are example navigation bars for a Home page and an About page for the same site. Each page places the navigation links in the same order.

Page 1

Skip navigation
siteLogo
  • Home
  • About
  • Services
  • Contact
  • Log In
Home Page Example

This is the main content for the Home page example.

The Home page link is the first link in the menu.

Page 2

Skip navigation
siteLogo
  • Home
  • About
  • Services
  • Contact
  • Log In
About Us Example

This is the main content for the About page example.

All elements, including the skip link, remain in the same order as the Home page.

Not Applicable

  • Navigational mechanisms on a single-page website
  • Components are inserted or removed

Disabilities Affected

  • Blindness/Visual Impairment: Users can easily predict where to find things on each page.
  • Cognitive Impairment: Users can easily predict where to find things on each page.
