Expected Result

Positive tabindex values do not result in illogical focus order.

Actual Result

Positive tabindex values are used, resulting in an illogical focus order.

Solution

  • Do not use positive tabindex values, OR
  • Ensure that the focus order is logical.

Failing Example

Positive tabindex Values

When positive tabindex values are used, the tab order can easily become illogical. These buttons incorrectly receive focus before anything else on the page.

Button 1
Button 2
Button 3
Button 4
Button 5
<div role="button" tabindex="1">Button 1</div>
<div role="button" tabindex="3">Button 2</div>
<div role="button" tabindex="4">Button 3</div>
<div role="button" tabindex="5">Button 4</div>
<div role="button" tabindex="2">Button 5</div>

Passing Example

tabindex="0"

When tabindex is set to "0", the tab order is determined based on the order of focusable elements in the DOM.

Button 1
Button 2
Button 3
Button 4
Button 5
<div role="button" tabindex="0">Button 1</div>
<div role="button" tabindex="0">Button 2</div>
<div role="button" tabindex="0">Button 3</div>
<div role="button" tabindex="0">Button 4</div>
<div role="button" tabindex="0">Button 5</div>

Disabilities Affected

  • Blindness/Vision Impairment: Screen reader and screen magnifier users can become disoriented if tabbing moves the focus in an illogical order.
  • Cognitive Impairment: Users can become disoriented if tab focus is in an illogical order.
  • Motor Impairment: Users may have a more difficult time navigating a page if tab focus is in an illogical order.
