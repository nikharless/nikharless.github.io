Expected Result

If a link or button contains only an image AND the link or button itself doesn't have alternative text, the image must have alternative text which describes the destination or function.

Actual Result

The control is labeled with only an <svg>, but the image does not have alternative text and the link or button does not have an accessible name.

Solution

Add a text alternative which describes the purpose of the functional image using the following method:
  1. Add a <title></title> element as the first child of the the <svg> element, and enter the text alternative within, AND
  2. Add a unique id to the <title> element, AND
  3. Add the aria-describedby attribute to the <svg> element, and point it to the <title> element's id value, AND
  4. Add the role="img" attribute to the <svg> element to ensure that screen readers recognize it as an image and read the text alternative.

Example

Image as Link

<a href="a11y_policy.html"><svg role="img" aria-describedby="a11yPolicy">
    <title id="a11yPolicy">Accessibility Policy</title>
    <g id="SVGRepo_tracerCarrier" stroke-width="0">
    ...
</svg></a>

Why It's Important

  • Blind/visually impaired: Users need to have a text alternative for functional images so that the images can be interpreted by assistive technology such as screen readers and braille devices.
  • Cognitively impaired: Users who have difficulty understanding images may benefit from having a text description.
