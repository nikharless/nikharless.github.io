Expected Result

Meaningful images are not CSS background images.

Actual Result

Meaningful image is a CSS background image.

Solution

  1. Mark up the image as a regular <img> element.
  2. Add an alt attribute to the <img> tag.
  3. Use CSS to position other elements over the image.

Example

Background Image with Text

Assume this background image's text relates to the text content placed on top of it. In this case, the background image is meaningful and needs a text alternative.

If the element with the background image did not have other content inside it, you could use role="img" and aria-label to give it a text alternative. However, in that case, there's no reason not to just use the simpler <img alt="..."> markup.

Since the element does contain other content, it cannot have a text alternative. The screen reader will not read both. The image needs to be marked up as an image, and the block of text as a separate element positioned with CSS.

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Rhoncus aenean vel elit scelerisque mauris.
<div style="width: 649px; height: 427px;">
    <img src="time-for-change.jpg" alt="Time for Change">
    <div id="divText">
        Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Rhoncus aenean vel elit scelerisque mauris.
    </div>
</div>

#divText {
  width: 249;
  margin-right: 400px;
  margin-top: -350px;
  padding: 80px 0 20px 20px;
  color: white;
  background-color: rgba(0,0,0,0.5);
}

Disabilities Affected

  • Blindness/Visual Impairment: Users need to have a text alternative for meaningful images so that the images can be interpreted by assistive technology such as screen readers and braille devices.
  • Cognitive Impairment: Users who have difficulty understanding images may benefit from having a text description.
