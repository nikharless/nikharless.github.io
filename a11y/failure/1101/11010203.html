Expected Result

All meaningful images must have a text alternative that conveys the same information as the image.

Actual Result

The meaningful CSS background image does not have a text alternative.

Solution

Mark the image up as an actual image, and add a text alternative which describes the purpose of the meaningful image.

Example

Background Image with Text

Assume this background image's text relates to the text content placed on top of it. In this case, the background image is meaningful and needs a text alternative.

If the element with the background image did not have other content inside it, you could use role="img" and aria-label to give it a text alternative. However, in that case, there's no reason not to just use the simpler <img alt="..."> markup.

Since the element does contain other content, it cannot have a text alternative. The image needs to be marked up as an image, and the block of text as a separate element positioned with CSS.

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Rhoncus aenean vel elit scelerisque mauris.
<div id="bg"> /* This element has the background image. */
    <div id="divText">
        Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Rhoncus aenean vel elit scelerisque mauris.
    </div>
</div>

Why It's Important

  • Blind/visually impaired: Users need to have a text alternative for meaningful images so that the images can be interpreted by assistive technology such as screen readers and braille devices.
  • Cognitively impaired: Users who have difficulty understanding images may benefit from having a text description.
