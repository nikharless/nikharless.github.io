Expected Result

If a control contains only an image AND the control itself doesn't have alternative text, the image must have alternative text which describes the destination or function.

Actual Result

The custom control contains only a CSS background image, but the control does not have alternative text.

Solution

Add a text alternative which describes the purpose of the functional image using one of the following methods:
  • Add an aria-label attribute to the control element's tag, OR
  • Add an aria-labelledby attribute to the control element's tag and set the value to the id of the text used as a label, OR
  • Change the background image to an actual image element (e.g., <img>) and give the image alternative text according to the appropriate checkpoint.

Example

Custom Button

<style>
    #action {
        background-image: url("submit.png");
    }
</style>
<div tabindex="0" role="button" id="action" aria-label="Submit"></div>

Font Awesome CSS Pseudo-element

Icon fonts are commonly used in CSS pseudo-elements.

<style>
    .fb::before {
        display: inline-block;
        text-rendering: auto;
        -webkit-font-smoothing: antialiased;
        font: var(--fa-font-brands);
        content: "\f09a";
    }
</style>
<p><a href="#"><span class="fb fa-brands fa-facebook" aria-label="Nik's Facebook Page"></span></a></p>

Why It's Important

  • Blind/visually impaired: Users need to have a text alternative for functional images so that the images can be interpreted by assistive technology such as screen readers and braille devices.
  • Cognitively impaired: Users who have difficulty understanding images may benefit from having a text description.
