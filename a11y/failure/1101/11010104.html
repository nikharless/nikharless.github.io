Expected Result

If a control is labeled with only an image, the image must have alternative text which describes the function or destination of the control.

Actual Result

The custom control is labeled with only a CSS background image, but the image does not have a text alternative.

Solution

Add a text alternative which describes the purpose of the functional image using the following method:
  • Add an aria-label attribute to the container tag

Example

Custom Button

<style>
    #action {
        background-image: url("submit.png");
    }
</style>
<div tabindex="0" role="button" id="action" aria-label="Submit"></div>

Font Awesome CSS Pseudo-element

Icon fonts are commonly used in CSS pseudo-elements.

<style>
    .fb::before {
        display: inline-block;
        text-rendering: auto;
        -webkit-font-smoothing: antialiased;
        font: var(--fa-font-brands);
        content: "\f09a";
    }
</style>
<p><a href="#"><span class="fb fa-brands fa-facebook" aria-label="Nik's Facebook Page"></span></a></p>

Why It's Important

Blind and visually impaired users need to have a text alternative to functional images so that they can be interpreted by assistive technology such as screen readers and braille devices. Without a text alternative, these users would not know what the link or control does.
