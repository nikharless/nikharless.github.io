Expected Result

Each audio element has a text alternative that summarizes its purpose.

Actual Result

An audio element has a text alternative, but it does not adequately summarize the content of the audio.

Solution

Change the text alternative to a summary of the content of the audio.

Failing Example

HTML <audio>

A text alternative like "audio player" does not tell the user what the audio is. While there is descriptive text nearby, screen reader users do not hear this if they are navigating by focusable elements.

What a horse says:

What a horse says:
<audio controls aria-label="audio player">
    <source src="horse.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Passing Example

HTML <audio>

What a horse says:
<audio controls aria-label="horse neighing">
    <source src="horse.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Disabilities Affected

Blind and visually impaired users need to have a text alternative to embedded audio which summarizes the content so they know what it is without having to play it.
