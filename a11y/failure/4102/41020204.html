Expected Result

If a control functions as a link, it has the role of "link". The function of a link is to navigate to another location (i.e., a web page, document, or external app).

Actual Result

A link does not have an href attribute, and therefore does not have the role of "link".

Solution

Add an href attribute to the <a> tag, even if the link is scripted. This may also provide a fallback for users who do not have JavaScript enabled.

Failing Example

Scripted link with no href

This example uses the HTML <a> element, but it does not have an href attribute. Since <a> tags can also be used as link targets when linking to sections within the same page, it is not considered to have the role of "link" unless it has the href attribute.

NVDA: "HTML <a> Tag, link"

HTML <a> Tag 
<a tabindex="0" onclick="goHere(event);"><HTML <a> Tag</a>

Passing Examples

Redundant href value

This example also has a scripted link function attached, but it additionally has an href value for users without JavaScript.

NVDA: "HTML <a> Tag, link"

HTML <a> Tag 
<a href="https://www.w3schools.com/tags/tag_a.asp" onclick="goHere(event);"><HTML <a> Tag</a>

Non-functional href value

This is another scripted link, but instead of a functional value, it uses javascript:void() for the href value.

NVDA: "HTML <a> Tag, link"

HTML <a> Tag 
<a href="javascript:void()" onclick="goHere(event);"><HTML <a> Tag</a>
