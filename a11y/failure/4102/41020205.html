Expected Result

Custom accordion controls have all appropriate roles and attributes.

Actual Result

A custom accordion control does not have all the appropriate roles or attributes.

Solution

Assign the appropriate roles and attributes:
  • Use the "button" role on the button which operates the accordion.
  • Use the aria-expanded attribute on the button, and set it to "true" when expanded and "false" when collapsed.
  • Use the aria-controls attribute on the button, and set it to the id of the associated accordion panel.
  • Use the "region" role on the accordion panel.
  • Use aria-labelledby on the accordion panel, pointing to the associated accordion button.

Examples

{% include examples/accordion.html %} 
<button aria-expanded="true" aria-controls="sect1" id="accordion1id">
...
<div id="sect1" role="region" aria-labelledby="accordion1id">

Not Applicable

The following items are not applicable to this failure:

  • Native HTML controls:
    <input type="checkbox"> Subscribe to the mailing list
    File under 4.1.2.1 Name, Role, Value
  • Custom controls that function as native HTML controls:
    <div role="checkbox" tabindex="0" aria-checked="false">Subscribe to the mailing list
    File under 4.1.2.1 Name, Role, Value
  • Hidden:
    <div style="display:none;">
    Does not appear in accessibility tree
  • Role of "none":
    <div role="none">
    Does not appear in accessibility tree

Disabilities Affected

Assistive technology only works well when code is done properly.
