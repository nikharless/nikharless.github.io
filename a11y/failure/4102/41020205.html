Expected Result

If a control functions as a combobox, it has the role of "combobox". A combobox is an input which has an associated pop-up from which the user can select a value (i.e., listbox, grid, tree, or dialog).

Actual Result

A control functions as a combobox, but it does not have the role of "combobox".

Solution

Assign the correct role to the control, using one of the following methods:

  • Use the <select> element to create the combobox, OR
  • Use the role="combobox" attribute on the element.

Failing Example

Custom combobox

This custom combobox does not have an ARIA role assigned and is identified by AT as a textbox.

NVDA: "Select a language, edit, read only, blank"

{% include examples/combobox-fail.html %} 
<div class="dropdown">
    <input type="text" placeholder="Select a language" readonly>
    <ul class="options">
    ...

Passing Examples

HTML <select> tag

This example uses the native HTML <select> element which has the default role of "combobox".

NVDA: "Select a color: combobox, Red, collapsed"

<select>
    <option value="red">Red</option>
    ...
</select>

Custom combobox

This custom combobox has the ARIA role "combobox" explicitly assigned.

NVDA: "Language, combobox, Select a Language, collapsed, opens list"

{% include examples/select-only.html %} 
<div role="combobox" aria-controls="listbox1" aria-haspopup="listbox" aria-expanded="false" tabindex="0">

Not Applicable

The following items are not applicable to this failure:

  • Hidden:
    <div style="display:none;">...</div>
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Programmatic roles enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Programmatic roles enable compatibility with assistive technology such as speech recognition software.
