Expected Result

Tree views have all appropriate roles and attributes.

Actual Result

A tree view does not have all the appropriate roles or attributes.

Solution

Tree View

  • Use the "tree" role on the main tree container.
  • Use the "treeitem" role on each tree node element.
  • Use the "group" role on each sub list container.
  • Use the aria-expanded attribute on each expandable treeitem; set it to "true" if expanded or "false" if collapsed.
  • If the tree allows multiple selections, use the aria-multiselectable="true" attribute on the tree element.
  • Use the aria-selected OR aria-checked attribute on each selectable node; set to "true" if selected/checked or to "false" if not. (Only one node may be marked as true if aria-multiselectable="true" is not set.)
    • If the tree only allows a single selection, aria-selected is generally used.
    • If the tree allows multiple selections, aria-checked is generally used.
    • It is not recommended to use both in the same tree.
  • Use the aria-label OR the aria-labelledby attribute on the tree element to provide an accessible name.
