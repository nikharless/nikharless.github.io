Expected Result

States, properties, and values are valid.

Actual Result

A component has an invalid state, property, or value.

Solution

  • Ensure states, properties, and values are valid.

Failing Example

Button with aria-checked

This button uses aria-checked to indicate the pressed state. However, aria-checked is not valid on a button. The correct attribute would be aria-pressed.

  • NVDA: "Mute, button"
<button aria-checked="false">Mute</button>

Passing Example

Button with aria-pressed

This button uses aria-pressed to indicate the pressed state.

  • NVDA: "Mute, button"
<button aria-pressed="false">Mute</button>

Not Applicable

The following items are not applicable to this failure:

  • Hidden with CSS:
    <input type="textbox" style="display:none;">
    Does not appear in accessibility tree
  • Hidden with ARIA:
    <input type="textbox" aria-hidden="true">
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
