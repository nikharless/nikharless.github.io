Expected Result

Progress bars communicate the current value to assistive technology.

Actual Result

A progress bar does not communicate the current value to assistive technology.

Solution

  • Use the native HTML <progress> element, which automatically communicates the current value, OR
  • Use aria-valuenow on the progress element to represent the current value of the meter, and set it to the current value.

Failing Example

Custom Progress Bar

This example does not communicate the current value to assistive technology.

  • NVDA: "busy indicator"
Check Out Progress
<div role="progressbar" aria-labelledby="proglabel">

Passing Example

Native HTML <progress>

This example uses the native HTML <progress> element with the value attribute set to the current value.

Note that the min attribute is invalid for this element. The minimum is always 0.

  • NVDA (current value = 70): "Progress bar, 70"
70% 
<progress id="native-progressbar" max="100" value="70">70%</progress>

Custom Progress Bar

This example uses aria-valuenow set to the current value to communicate the value to assistive technology.

Note that aria-valuemin and aria-valuemax are not required and default to 0 and 100 respectively.

  • NVDA (current value at 70): "progress bar, 70"
Check Out Progress
<div role="progressbar" aria-labelledby="proglabel" aria-valuenow="70">

Not Applicable

The following items are not applicable to this failure:

  • Role of "none":
    <div role="none">
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
