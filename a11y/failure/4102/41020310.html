Expected Result

Disabled controls communicate the disabled state to assistive technology.

The following elements support the disabled HTML attribute:

  • <button>
  • <fieldset>
  • <input>
  • <optgroup>
  • <option>
  • <select>
  • <textarea>

The following roles support the aria-disabled property:

  • application
  • button
  • checkbox
  • columnheader (in context of grid or treegrid)
  • combobox
  • grid
  • gridcell
  • group
  • input
  • link
  • listbox
  • menu
  • menubar
  • menuitem
  • menuitemcheckbox
  • menuitemradio
  • option
  • radio
  • radiogroup
  • row (in context of grid or treegrid)
  • rowheader (in context of grid or treegrid)
  • scrollbar
  • searchbox
  • select
  • separator (if focusable)
  • slider
  • spinbutton
  • switch
  • tab
  • tablist
  • textbox
  • toolbar
  • tree
  • treegrid
  • treeitem

Actual Result

Disabled control does not communicate the disabled state to assistive technology.

Solution

  • Use the disabled attribute on a native HTML element in order to identify it as disabled, remove the keyboard focus, remove the functionality, and make it visually appear disabled,
    • Use the CSS :disabled pseudo-class to style the disabled state.
    OR
  • Use the aria-disabled attribute on a custom control in order to identify the control as disabled without changing the focusability, functionality or appearance.
    • Use JavaScript to prevent interaction.

Failing Example

ARIA

This example removes the functionality from the button and makes it visually appear disabled, but the disabled state is not communicated to assistive technology.

NVDA: ""

Save
<div role="button" tabindex="0" style="opacity: 0.5;">Save</div>

Passing Examples

HTML

This native HTML example uses the disabled attribute to identify it as disabled to all users.

NVDA: ""

<button disabled>Save</button>

ARIA

This ARIA example removes the functionality from the button, makes it visually appear disabled, and uses aria-disabled="true" to communicate the disabled state to assistive technology.

Save
<div role="button" tabindex="0" aria-disabled="true">Save</div>

Not Applicable

The following items are not applicable to this failure:

  • Hidden with CSS:
    <button style="display:none;">
    Does not appear in accessibility tree
  • Hidden with ARIA:
    <button aria-hidden="true">
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
