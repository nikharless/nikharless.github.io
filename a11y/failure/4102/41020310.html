Expected Result

Meters communicate the current value to assistive technology.

Actual Result

A meter does not communicate the current value to assistive technology.

Solution

  • Use the native HTML <meter> element with the value attribute set to the current value, OR
  • Use aria-valuenow on the meter element to represent the current value of the meter, and dynamically set it to the current value at regular intervals.

Failing Examples

Custom Meter

This example does not communicate the current value to assistive technology.

  • NVDA (regardless of value): "Progress bar, 0"
  • NVDA (on change): No feedback

Passing Example

Native HTML <meter>

This example uses the native HTML <meter> element with the value attribute set to the current value.

  • NVDA (current value = 90): "Progress bar, 90"
at 90/100 
<meter id="native-meter" min="0" max="100" value="90"> at 90/100</meter>

Custom Meter

This example uses aria-valuenow set to the current value to communicate the value to assistive technology.

  • NVDA: "Progress bar, [current value]"
  • NVDA (on change): Beeps which vary according to range between min and max
