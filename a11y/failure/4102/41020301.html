Expected Result

Automatically updated content is announced to assistive technology users.

Actual Result

Automatically updated content is not announced to assistive technology users.

Solution

Ensure that automatically updated content is announced to assistive technology users using one of the following:

  • Keyboard-accessible dialog, OR
  • Moving keyboard focus to the changed content, OR
  • Changing the content within an ARIA Live Region.

Examples

  • The updated content is announced with a dialog.
  • The focus is moved to the updated content.
  • The updated content is contained within an ARIA Live Region or status message.

Not Applicable

The following items are not applicable to this failure:

  • Form elements that change in response to value changes
    File under {{ site.data.4102.4102[41020302].name }}.
  • User-initiated updates to content
    File under {{ site.data.4102.4102[41020303].name }}.

Why It's Important

  • Blind/vision impaired: Screen reader users need to be alerted of changes in page content.

References

  • Understanding Success Criterion {{ sc.name }}
  • Creating components using a technology that supports the accessibility API features of the platforms on which the user agents will be run to expose the names and roles, allow user-settable properties to be directly set, and provide notification of changes
  • Creating modal dialogs with the HTML dialog element
  • Using aria-alertdialog to Identify Errors
  • Using ARIA role=alert or Live Regions to Identify Errors
