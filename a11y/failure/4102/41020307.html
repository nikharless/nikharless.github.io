Expected Result

If a control functions as a combobox, it communicates the expanded state to assistive technology.

Actual Result

A combobox does not communicate the expanded state to assistive technology.

Solution

  • Use the native HTML <select> element, which automatically communicates the expanded state to assistive technology, OR
  • Use the aria-expanded attribute on a custom combobox, and set it to "true" when expanded and "false" when collapsed.

Failing Example

Language

Passing Examples

Native HTML

This example uses the native HTML <select> element to communicate the expanded state to assistive technology.

  • NVDA: "Language, combobox, Select a Language, collapsed"
<select>
    <option value="">Select a Language</option>
    <option value="English">English</option>
    ...
</select>

Custom Select-Only Combobox

This custom combobox uses aria-expanded to communicate the expanded state to assistive technology.

  • NVDA: "Language, combobox, Select a Language, collapsed, opens list"
{% include examples/select-only.html %} 
<div role="combobox" 
    aria-controls="listbox1" 
    aria-expanded="false" 
    aria-haspopup="listbox" 
    aria-labelledby="combo1-label" 
    id="combo1" 
    tabindex="0" 
    aria-activedescendent>
    Select a Language
</div>

Not Applicable

The following items are not applicable to this failure:

  • Hidden:
    <div style="display:none;">...</div>
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
