Expected Result

If a control functions as a radio button, it communicates the checked state to assistive technology.

Actual Result

A radio button does not communicate the checked state to assistive technology.

Solution

  • Use the native HTML <input type="radio"> mark-up to create the radio button, OR
  • Use the aria-checked attribute on a custom radio button element.

Failing Example

Custom Radio Button

This custom radio button does not communicate the checked state to assistive technology.

  • NVDA (on focus, regardless of state): "Soup, radio button, not checked, 1 of 2

Side

Soup
Salad 
<div role="radio" tabindex="0"> Soup

Passing Example

Native HTML

This example uses native HTML, which automatically communicates the checked state to assistive technology.

  • NVDA (on focus when not checked): "Soup, radio button, not checked, 1 of 2"
  • NVDA (on focus when checked): "Salad, radio button, checked, 2 of 2"
  • NVDA (on state change): "checked"
Side 
<label><input type="radio" name="side"> Soup</label>
<label><input type="radio" name="side"> Salad</label>

Custom radio button

This custom radio button uses aria-checked to communicate the checked state to assistive technology.

  • NVDA (on focus when not checked): "Soup, radio button, not checked, 1 of 2"
  • NVDA (on focus when checked): "Soup, radio button, checked, 1 of 2"
  • NVDA (on state change): "checked"

Side

Soup
Salad 
<div role="radio" tabindex="0" aria-checked="false"> Soup

Not Applicable

The following items are not applicable to this failure:

  • Hidden:
    <div style="display:none;">...</div>
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
