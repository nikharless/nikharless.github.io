Expected Result

Form fields have roles appropriate for their function.

Actual Result

A form field is missing a role.

Solution

Assign the correct role to the control, using one of the following methods:

  • Use native HTML elements to create the control, OR
  • Use the role attribute on the element to assign the appropriate role.

Failing Examples

Checkbox

This is a custom checkbox that does not have an assigned ARIA role.

NVDA: "Subscribe to the mailing list

Subscribe to the mailing list. 
<div tabindex="0" aria-checked="false"> Subscribe to the mailing list.

Radio button

This is a custom radio button that does not have an assigned ARIA role.

NVDA: "Essential only"

Essential only 
<div tabindex="0" aria-checked="false">Check this radio button</div>

Passing Examples

Checkboxes

This example uses native HTML to provide the checkbox role.

NVDA: "Subscribe to the mailing list, checkbox, not checked"

<label><input type="checkbox"> Subscribe to the mailing list.</label>

This is a custom checkbox with the ARIA role of "checkbox" explicitly assigned.

NVDA: "Subscribe to the mailing list, checkbox, not checked"

Subscribe to the mailing list. 
<div role="checkbox" tabindex="0" aria-checked="false"> Subscribe to the mailing list.

Radio buttons

This example uses native HTML to provide the radio role.

NVDA: "Essential only, radio button, not checked, 1 of 1"

<label><input type="radio"> Essential only</label>

This is a custom radio button with the ARIA role of "radio" explicitly assigned.

NVDA: "Essential only, radio button, not checked, 1 of 1"

Essential only 
<div role="radio" tabindex="0" aria-checked="false"> Essential only</div>

Text fields

This example uses native HTML to provide the textbox role.

NVDA: "Name: edit blank"

<label>Name: <input type="text"></label>

This is a custom textbox with the ARIA role of "textbox" explicitly assigned.

NVDA: "Name: edit multiline blank"

Name: 
<div id="name">Name: </div><div role="textbox" contenteditable="true" aria-labelledby="name"> Name</div>

Disabilities Affected

  • Blindness/Vision Impairment: Programmatic roles enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Programmatic roles enable compatibility with assistive technology such as speech recognition software.
