Expected Result

If a control functions as a link, it has the role of "link". The function of a link is to navigate to another location (i.e., a web page, document, or external app).

Actual Result

A link does not have the role of "link".

Solution

Assign the correct role to the control, using one of the following methods:

  • Use the <a href=""> element to create the link, OR
  • Use the role="link" attribute on the element.

Examples

HTML anchor tag

HTML <a> Tag 
<a href="https://www.w3schools.com/tags/tag_a.asp"><HTML <a> Tag</a>

Role of "link"

<button role="link" onclick="openLink(event)">Web Accessibility Initiative (WAI)</button>

Not Applicable

The following items are not applicable to this failure:

  • Buttons:
    <button>Click Here</button>
    File under 4.1.2.1.7 Button does not have a valid role
  • Checkboxes, radio buttions, and text fields:
    <div role="checkbox" tabindex="0" aria-checked="false"> Subscribe to the mailing list
    File under 4.1.2.1.8 Form field has an incorrect role
  • Comboboxes:
    <div role="combobox" aria-controls="listbox1" aria-expanded="false" aria-haspopup="listbox" tabindex="0"></div>
    File under 4.1.2.1.11 Select (combobox) has an incorrect role
  • Hidden:
    <a href="#" style="display:none;">Click here</a>
    Does not appear in accessibility tree

Why It's Important

Assistive technology only works well when code is done properly.
