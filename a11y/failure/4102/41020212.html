Expected Result

Menus have all appropriate roles and attributes.

Actual Result

A menu does not have all the appropriate roles or attributes.

Solution

Menu bar

  • Use the "menubar" role on the menu bar (<ul>) element.
  • Use aria-label or another method to give the menu bar an accessible name.
  • Use the "none" role on each <li> element, because the parent element is a menu, not a list.
  • Use the "menuitem" role on each menu item (<a) element.
  • Use tabindex="-1" on each menu item element to allow it to have keyboard focus without being part of the tab sequence.
  • Use tabindex="0" on the first menu item to include it in the tab sequence.
  • Use aria-current="page" on the menu item element that corresponds to the current page.
  • Use aria-haspopup="true" on any menu item element that has a submenu.
  • Use aria-expanded on any menu item element that has a submenu.
    • If the submenu is closed, set it to aria-expanded="false".
    • If the submenu is open, set it to aria-expanded="true".

Submenu

  • Use the "menu" role on each submenu container (<ul>) element.
  • Use aria-label or another method to give the submenu an accessible name.
  • Use the "none" role on each <li> element, because the parent element is a menu, not a list.
  • Use the "menuitem" role on each menu item (<a) element.
  • Use tabindex="-1" on each menu item element to allow it to have keyboard focus without being part of the tab sequence.
  • Use aria-haspopup="true" on any menu item element that has a submenu.
  • Use aria-expanded on any menu item element that has a submenu.
    • If the submenu is closed, set it to aria-expanded="false".
    • If the submenu is open, set it to aria-expanded="true".
