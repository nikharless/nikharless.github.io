Expected Result

Date picker controls communicate to assistive technology that they open a dialog.

Actual Result

A date picker control does not communicate to assistive technology that it opens a dialog.

Solution

  • Use the aria-haspopup attribute on the button or combobox input.
  • Set it to "dialog" to indicate that it opens a dialog.

Failing Example

Select Date
MTW TFSS

Passing Example

Date picker with aria-haspopup

This example uses aria-haspopup="dialog" to communicate to assistive technology that it opens a dialog.

NVDA: "Date, combobox, collapsed, editable, opens dialog, (date format: mm/dd/yyyy), blank"

{% include examples/combobox-datepicker.html %} 

<input type="text" 
    aria-autocomplete="none" 
    role="combobox" 
    aria-expanded="false" 
    aria-haspopup="dialog"
    aria-controls="cb-dialog-1" 
    aria-describedby="cb-description-1">
...

Not Applicable

The following items are not applicable to this failure:

  • Hidden with CSS:
    <div role="textbox" style="display:none;">
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
