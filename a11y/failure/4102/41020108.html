Expected Result

Form fields have roles appropriate for their function.

Actual Result

A form field does not have the correct role.

Solution

Assign the correct role to the control, using one of the following methods:

  • Use native HTML elements to create the control, OR
  • Use the role attribute on the element to assign the appropriate role.

Examples

Checkbox

<label><input type="checkbox"> The information is correct.</label>
<div role="checkbox" tabindex="0" aria-checked="false"> Subscribe to the mailing list

Radio button

<label><input type="radio"> Essential only</label>
<div role="radio" tabindex="0" aria-checked="false"> Essential only</div>

Text field

<label>Name: <input type="text"></label>
<div role="textbox" contenteditable="true" style="border: 1px solid black;"> Email</div>

Not Applicable

The following items are not applicable to this failure:

  • Buttons:
    <button>Click Here</button>
    File under 4.1.2.1.7 Button does not have a valid role
  • Links:
    <a>Link text</a>Click Here</a>
    File under 4.1.2.1.9 Link has an incorrect role
  • Comboboxes:
    <div role="combobox" aria-controls="listbox1" aria-expanded="false" aria-haspopup="listbox" tabindex="0"></div>
    File under 4.1.2.1.11 Select (combobox) has an incorrect role
  • Hidden:
    <input type="checkbox" style="display:none;>
    Does not appear in accessibility tree

Why It's Important

Assistive technology only works well when code is done properly.
