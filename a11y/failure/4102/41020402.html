Expected Result

Any values associated with a component are valid.

Actual Result

An attribute value is invalid.

Solution

Ensure the value is valid for the attribute.

  • Ensure there are no typos in the value.
  • If the attribute allows only certain tokens, ensure the value matches one (or more) of them.
  • If the attribute points to an element's ID, ensure that ID exists in the current page.

Failing Examples

Invalid aria-haspopup Token

This example opens a dialog but aria-haspopup is set to "modal" which isn't a valid option.

  • NVDA: "Settings, button"
<button aria-haspopup="modal">Settings</button>

Invalid ID

This example uses aria-describedby to associate text with the form field. However, the ID it points to does not exist.

  • NVDA: "Email Address:, edit, blank"

We will not share your email with any third party.

<label for="email">Email Address:</label>
<input id="email" type="text" aria-describedby="description">
<p id="privacy">We will not share your email with any third party.</p>

Passing Examples

Valid aria-haspopup Token

This example opens a dialog and aria-haspopup is correctly set to "dialog".

  • NVDA: "Settings, button, opens dialog"
<button aria-haspopup="dialog">Settings</button>

Valid ID

This example uses aria-describedby with a valid ID to associate text with the form field.

  • NVDA: "Email Address:, edit, We will not share your email with any third party., blank"

We will not share your email with any third party.

<label for="email">Email Address:</label>
<input id="email" type="text" aria-describedby="privacy">
<p id="privacy">We will not share your email with any third party.</p>
