Expected Result

Buttons have accessible names.

Actual Result

A button does not have an accessible name.

Solution

HTML

Provide an accessible name for the button, using one of the following methods:

  • Use <button> and place the name between the start and end tags, OR
  • Use <input type="button"> and place the name in the value attribute, OR
  • Use <input type="submit"> or <input type="reset"> and use the default accessible name, OR
  • Use role="button" to create a custom button, and use text within the button element as the accessible name.

Failing Examples

<button> without text

This example uses the native HTML <button> element, but it does not have any text or image with a text alternative inside it.

<button><span class="arrow"></span></button>

This example uses the native HTML <button> element with a value attribute. However, the value attribute is not valid on the <button> element and does not provide an accessible name.

<button value="Next"></button>

This example uses the native HTML <button> element with role="none". However, because a <button> element is focusable, the role="none" is ignored, leaving the button without an accessible name.

<button role="none"></button>

Input button without text

This example uses the native HTML <input type="button"> element, but it does not have a value assigned.

<input type="button">

Custom button

This example uses the text inside a <div> with role="button", but it does not have any text or image with a text alternative inside it.

<div role="button" tabindex="0"><span class="arrow"></span></div>

Passing Examples

<button>

This example uses the text inside the native HTML <button> element as the accessible name.

<button>Next</button>

This example uses the text alternative for an image inside the native HTML <button> element as the accessible name.

<button><img src="submit.png" alt="Submit"></button>

This example uses an aria-label value on a CSS image as the accessible name.

<button><span class="arrow" aria-label="Next"></span></button>

This example uses an aria-label value on the <button> element as the accessible name.

<button aria-label="Next"><span class="arrow" ></span></button>

Input button

This example uses the value attribute of the native HTML <input type="button"> element as the accessible name.

<input type="button" value="Next">

Submit and Reset buttons

These examples use the native HTML <input type="submit"> and <input type="reset">. They have default accessible names of "Submit" and "Reset".

<input type="submit"> <input type="reset">

Custom button

This example uses the text inside a <div> with role="button" as the accessible name.

Next
<div role="button" tabindex="0">Next</div>

This example uses the text alternative for an image inside a <div> element with role="button" as the accessible name.

<div role="button" tabindex="0"><img src="submit.png" alt="Submit"></div>

This example uses an aria-label value as the accessible name.

<div role="button" tabindex="0" aria-label="Next"><span class="arrow"></span></div>

Not Applicable

The following items are not applicable to this failure:

  • Input type "image":
    <input type="image" src="search.png" alt="Search">
    File under {{ site.data.4102.4102[41020103].name }}.
  • Link role:
    <button role="link">Next Page</button>
    File under {{ site.data.2404.2404[24040101].name }}.
  • Hidden:
    <button style="display:none;">Search</button>
    Does not appear in accessibility tree
  • Role of "none":
    <button role="none">Search</button>
    Does not appear in accessibility tree
  • No role:
    <div>Search</div>
    File under {{ site.data.4102.4102[41020201].name }}.

Disabilities Affected

  • Blindness/Vision Impairment: Programmatic names enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Programmatic names enable compatibility with assistive technology such as speech recognition software.
