Expected Result

Carousels have all appropriate roles and attributes.

Actual Result

A carousel does not have all the appropriate roles or attributes.

Solution

Assign the appropriate roles and attributes:
  • Use the "region" role on the carousel (<section> or role="region").
  • Use aria-roledescription="carousel" on the carousel so that the screen reader identifies it as a carousel but preserves the region functionality.
  • Use aria-label or aria-labelledby on the carousel to give it an accessible name.
  • Use the aria-live property on the carousel. Set it to "off" when the carousel is automatically rotating. Set it to "polite" when it is not.
  • Use the "tablist" role on the element which contains the slides.
  • Use aria-label on the tablist to provide an accessible name.
  • Use the "tab" role on each slide button.
  • Use aria-label on each slide button to provide an accessible name (e.g., "slide 1").
  • Use the aria-selected attribute on each slide button. Set it to "true" for the current slide and to "false" for all other slides.
  • Set tabindex="-1" on each slide button when it is not currently selected.
  • Use the aria-controls attribute on each button and set it to the id of the corresponding tabpanel.
  • Use the "tabpanel" role on the slide content element.
  • Use aria-roledescription="slide" on the slide content element so that the screen reader identifies it as a slide but preserves the tabpanel functionality.
  • Use aria-label="X of Y" on each slide content element to provide an accessible name.

Examples

<section id="myCarousel" aria-roledescription="carousel" aria-label="Cats of House Harless">
...
<div role="tablist" aria-label="Slides">
    <button role="tab" aria-label="Slide 1" aria-selected="true" aria-controls="carousel-item-1">
    ...
<div id="myCarousel-items" aria-live="off">
    <div id="carousel-item-1" role="tabpanel" aria-roledescription="slide" aria-label="1 of 4">

Not Applicable

The following items are not applicable to this failure:

  • Native HTML controls:
    <input type="checkbox"> Subscribe to the mailing list
    File under 4.1.2.1 Name, Role, Value
  • Custom controls that function as native HTML controls:
    <div role="checkbox" tabindex="0" aria-checked="false">Subscribe to the mailing list
    File under 4.1.2.1 Name, Role, Value
  • Hidden:
    <div style="display:none;">
    Does not appear in accessibility tree
  • Role of "none":
    <div role="none">
    Does not appear in accessibility tree

Why It's Important

Assistive technology only works well when code is done properly.
