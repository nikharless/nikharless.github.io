Expected Result

Toolbars have all appropriate roles and attributes.

Actual Result

A toolbar does not have all the appropriate roles or attributes.

Solution

Tab List

  • Use the "toolbar" role on the element that contains a set of tool buttons.
  • Use the aria-label attribute to give the toolbar an accessible name.
  • Use the aria-controls attribute on the toolbar element and point it to the id of the associated text area.
  • Set tab-index="-1" on each inactive tool element to prevent it from getting keyboard focus.
  • Set tab-index="0" on only one item in the toolbar.
    • Set this on the first tool item when the page loads.
    • Change it with keyboard focus to the currently focused tool item.

Toggle Buttons (Bold, Italic, Underline)

  • Set aria-pressed="true" on each button when it is pressed.
  • Set aria-pressed="false" on each button when it is not pressed.
  • Set aria-hidden="true" on the image/icon to prevent it from being read by the screen reader.

Radio Group (Text Alignment)

  • Use the "radiogroup" role on the radio button container element.
  • Use the aria-label attribute to give the radio group an accessible name.
  • Set aria-checked="true on the currently selected radio button.
  • Set aria-checked="false" on each non-selected radio button.
  • Set aria-hidden="true" on the image/icon to prevent it from being read by the screen reader.

Button (Cut, Copy, Paste)

  • Set aria-disabled="true" on each button when it is unavailable.
  • Set aria-disabled="false" on each button when it is available.

Menu Button (Fonts)

  • Use aria-haspopup="true" on the button to indicate that it opens a menu.
  • Use the aria-controls attribute and point it to the id of the menu element.
  • Use the aria-expandedattribute on the menu button.
    • Set it to "true" when the menu is open.
    • Set it to "false" when the menu is closed.
  • Use aria-label to set the accessible name for the button.
    • For a font menu, set the name to "Font: [Font Name]" and update it to match the currently selected option.

Menu (Fonts)

  • Use the "menu" role on the menu element.
  • Use aria-label to set the accessible name (e.g., "Font Family").
  • Use the "menuitemradio" role on each menu item.
  • Use the aria-checked attribute on each menu item element.
    • Set to "true" for the currently selected option.
    • Set to "false" for each non-selected option.
  • Set tabindex="-1" on each menu item element to allow scripted focus.

Spin Button (Font Size)

  • Use the "spinbutton" role on the spin button element.
  • Use aria-label to set the accessible name (e.g. "Font size in points").
  • Use the aria-valuenow attribute on the spin button element to indicate the current value.
    • Update whenever the value changes.
  • Use the aria-valuetext attribute to give the spin button a more descriptive name (e.g., "[Number] Points").
  • Use the aria-valuemin attribute to set the minimum value for the spin button.
  • Use the aria-valuemax attribute to set the maximum value for the spin button.
