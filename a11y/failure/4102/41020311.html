Expected Result

Sliders communicate the current value to assistive technology.

Actual Result

A slider does not communicate the current value to assistive technology.

Solution

  • Use aria-valuemax on the slider element to represent the maximum value of the slider.
  • Optionally, use aria-valuetext on the slider element if the default number format does not make sense in the slider's context.

Failing Example

Custom Slider

This example does not communicate the current value to assistive technology.

  • NVDA/Chrome (on focus): "Volume, slider, 50"
  • NVDA/Chrome (on change): No feedback
{% include examples/slider-seek.html %}

Passing Example

Native HTML Slider

This example uses the native HTML <input type="range"> element, which automatically communicates the current value to assistive technology.

  • NVDA: ""
<input type="range" min="1" max="100" value="50">

ARIA Custom Slider

This example uses aria-valuenow to communicate the current value to assistive technology.

  • NVDA: ""
{% include examples/slider-rating.html %} 
<div role="slider" tabindex="0" aria-valuemin="0" aria-valuenow="0" aria-valuemax="10" aria-valuetext="Choose a rating from 1 to 10 where 10 is Strongly Disagree">

Not Applicable

The following items are not applicable to this failure:

  • Role of "none":
    <div role="none">
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
