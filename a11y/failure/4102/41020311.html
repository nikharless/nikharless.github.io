Expected Result

Required controls communicate the required state to assistive technology.

The following elements support the required HTML attribute:

  • <input>
  • <select>
  • <textarea>

The following roles support the aria-required property:

  • checkbox
  • columnheader
  • combobox
  • gridcell
  • listbox
  • radiogroup
  • rowheader
  • scrollbar
  • searchbox
  • spinbutton
  • switch
  • textbox
  • tree
  • treegrid

Actual Result

Required control does not communicate the required state to assistive technology.

Solution

  • Use the required attribute on a native HTML element,
    • Use the CSS :required pseudo-class to style elements with the required property.
    OR
  • Use the aria-required attribute on a custom control.

Failing Examples

HTML

This native HTML example indicates with an asterisk in the label that it is required. However, it does not clearly communicate to assistive technology that it is required.

NVDA: "Email*, edit, blank"

<input type="email">

ARIA

This custom textbox indicates with an asterisk in the label that it is required. However, it does not clearly communicate to assistive technology that it is required.

NVDA: "Email*, edit, blank"

Email* 
<div role="textbox" contenteditable aria-multiline="false">

Passing Examples

HTML

This native HTML example uses the required attribute on an email <input>.

NVDA: "Email*, edit, required, blank"

<input type="email" required>

ARIA

This custom textbox uses aria-required="true" to communicate the required state to assistive technology.

NVDA: "Email*, edit, required, blank"

Email* 
<div role="textbox" contenteditable aria-multiline="false" aria-required="true">

Not Applicable

The following items are not applicable to this failure:

  • Hidden with CSS:
    <input type="textbox" style="display:none;">
    Does not appear in accessibility tree
  • Hidden with ARIA:
    <input type="textbox" aria-hidden="true">
    Does not appear in accessibility tree

Disabilities Affected

  • Blindness/Vision Impairment: Properties enable compatibility with assistive technology such as screen readers and screen magnifiers.
  • Motor Impairment: Properties enable compatibility with assistive technology such as speech recognition software.
