Expected Result

Custom breadcrumb controls have all appropriate roles and attributes.

Actual Result

A custom breadcrumb control does not have all the appropriate roles or attributes.

Solution

Assign the appropriate roles and attributes:
  • Use the "navigation" role on the breadcrumb region (<nav> or role="navigation).
  • Use aria-label on the breadcrumb region to give it an accessible name.
  • Use the aria-current="page" property on the link to the current page.

Examples

{% include examples/breadcrumb.html %} 
<nav aria-label="Breadcrumb">
...
<a href="#" aria-current="page">4.1.2.b.f Breadcrumb is missing appropriate roles and/or attributes</a>

Not Applicable

The following items are not applicable to this failure:

  • Native HTML controls:
    <input type="checkbox"> Subscribe to the mailing list
    File under 4.1.2.1 Name, Role, Value
  • Custom controls that function as native HTML controls:
    <div role="checkbox" tabindex="0" aria-checked="false">Subscribe to the mailing list
    File under 4.1.2.1 Name, Role, Value
  • Hidden:
    <div style="display:none;">
    Does not appear in accessibility tree
  • Role of "none":
    <div role="none">
    Does not appear in accessibility tree

Why It's Important

Assistive technology only works well when code is done properly.
