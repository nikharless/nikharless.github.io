Expected Result

If a control functions as a toggle button, it has a "pressed" state.

Actual Result

A toggle button does not have a "pressed" state.

Solution

Use the aria-pressed attribute on the toggle button. For a simple two-state toggle button, set it to "true" when pressed and to "false" when not pressed. It must be explicitly set, as the default value is "undefined".

Do not change the label on a toggle button (e.g., "Mute"/"Unmute") when the state changes.

Failing Example

Regular Button

This button does not track pressed state changes.
