Success Criterion Structure

  1. Success Criterion: link and full text
  2. Training: list of resources

Checkpoint Structure

  1. Basic Requirements and Best Practices
    • Legend: Requirement/Best Practice icons ([icon] = Requirement)
    • Bullet list of requirements
      • Each bullet begins with icon
      • Requirements use "must"
      • Best Practices use "should"
    • User Story: what does the user want
    • Success: what defines success for the checkpoint
  2. Overview: what does it apply to, etc. (don't just repeat SC)
  3. Testing Logic: expandable tree
    • Begin section with question checking for applicability.
    • Each branch has Yes and No options, in that order.
    • Each branch begins with "Yes;" or "No;" in bold.
    • If the option indicates that the checkpoint does not apply, follow the answer with "This checkpoint is not applicable."
    • If the option indicates that the checkpoint applies, follow with another question to begin the test sequence; this option will be expandable.
    • If the option indicates that the element being tested passes the checkpoint, follow the answer with "PASS"; this option will be green and begin with a star icon.
    • If the option indicates that the element being tested fails the checkpoint, follow the answer with "See [link to failure page]." This option will be red and begin with an x icon.
  4. Web Test Procedure
    1. Desktop
  5. Failures: list of links to failure pages followed by severity rating in parentheses.
  6. Testing Tools: list of tools used to test the checkpoint
  7. References
    1. WCAG
      1. Link to Success Criterion
      2. Links to Related Techniques
        1. General
        2. HTML
        3. ARIA
        4. CSS
        5. PDF
    2. Additional Information

Failure Structure

  1. Expected Result: Phrased using a description of what the expected result looks like ("is" or "are" rather than "must").
  2. Actual Result: Description of failure
  3. Solution
  4. Failing Example(s) (add or drop "s" depending whether there are more than one)
    • Subtitle for each example
    • Description of why it fails
    • Working example (not an image, unless no other way)
    • Code block showing relevant code, with specific parts bolded, if applicable
  5. Passing Example(s) (add or drop "s" depending whether there are more than one)
    • Subtitle for each example
    • Description of why it passes
    • Working example (not an image, unless no other way)
    • Code block showing relevant code, with specific parts bolded, if applicable
  6. Not Applicable: bullets listing similar failures that might be confused with current one
    1. First line bold, ending with colon
    2. Second line "File under [link to correct failure or section]"
  7. Disabilities Affected: bullet list of disabilities affected by this checkpoint and a brief explanation of how they are affected
    • Blindness
    • Visual Impairment
    • Color blindness
    • Deafness
    • Hearing Impairment
    • Muteness
    • Motor Impairment
    • Mobility Impairment
    • Cognitive Impairment
  8. References: bullet list of links
    1. Link to Understanding Success Criterion WCAG page
    2. Links to relevant WCAG failure pages
    3. Links to relevant WCAG technique pages
