Overview

Auto-playing content may include sounds, music, and alerts.

Because automatically playing audio interferes with the ability to hear a screen reader and may make it difficult to find any mechanism to make it stop, it is highly recommended to avoid automatically playing audio at all. Instead, allow the user to choose to play the audio or not.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The user must be able to control automatic audio that plays for more than 3 seconds.
    • The user must be able to pause or stop the audio, OR
    • The user must be able to adjust the volume of the audio independently of the system volume control
  • The mechanism for pausing, stopping, or controlling the volume must itself pass all applicable Success Criteria.
User Story: As a screen reader user, I want to be able to control automatically playing audio so that I can hear my screen reader.
Success: If you used a screen reader to read the page, when audio automatically played for more than 3 seconds, you would be able to turn it off or adjust the volume independently of the system volume control.

Testing Logic

Is there audio on the page that starts playing automatically?

  • Yes; Does the audio play for more than 3 seconds?
    • Yes; Is there a mechanism available to pause or stop the audio?
      • Yes; PASS
      • No; Is there a mechanism available to adjust the volume of the audio independently of the system audio control?
        • Yes; Does the mechanism pass all applicable Success Criteria?
          • Yes; PASS
          • No; See {{ site.data.1402.1402[14020102].name }}.
        • No; See {{ site.data.1402.1402[14020101].name }}.
    • No; This checkpoint is not applicable.
  • No; This checkpoint is not applicable.

Web Test Procedure

  1. Ensure the browser is not configured to block auto-playing content.
  2. Locate any audio that automatically plays for more than 3 seconds.
  3. Verify that the audio can be paused, stopped, muted, or the volume can be adjusted independently of the system volume control.
  4. Verify that the controls work with a screen reader turned on.
