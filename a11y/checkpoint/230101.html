Overview

This criterion only applies to web content, not flashing caused by user agents. Examples include:

  • Video
  • Animations
  • Blinking elements

Note: Though this checkpoint technically passes if the flashing area is small enough, the size is calculated using standard screen size and viewing distance. A user who is closer to their screen or who zooms in to view content could cause the content to then fail. Therefore, it's safer to ensure flashing is no more than 3 times per second.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Content must not flash more than 3 times per second, OR the flash must be below the flash thresholds.
  • Content should not flash at all.
User Story: As a user with a seizure disorder, I want to view content that does not flash so that it does not trigger seizures.
Success: If you had a seizure disorder, when viewing flashing content, you would not have seizures.

Testing Logic

Is there any flashing content on the page?

  • Yes; Is the flash below the general flash and red flash thresholds? (This can be measured with PEAT.)
    • Yes; PASS
    • No; Is the flashing area smaller than 341 x 256 pixels?
      • Yes; PASS
      • No; See {{ site.data.2301.2301[23010101].name }}.
  • No; PASS

Web Test Procedure

Desktop

  1. Locate any flashing content.
  2. Determine the size of the flashing area.
  3. If the area is bigger than 341 x 256 pixels, count the number of flashes per second.
  4. If the content flashes more than 3 times per second, test using PEAT.
    1. Open the web page in Firefox on Windows.
    2. In PEAT, select Start Capture from the Capture menu.
    3. Play the video you just captured to make sure it recorded correctly.
    4. Select Analyze Video from the Analysis menu.
    5. Review the PEAT Test Results and Analysis.

Mobile

  1. Locate any flashing content.
  2. Count the number of flashes per second.
  3. If the content flashes more than 3 times per second, test using PEAT on a desktop computer.
