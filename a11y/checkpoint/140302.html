Overview

Be sure to check for text that is not initially visible on the page, such as text in tooltips, dialogs, menus, edit fields, and expandable content.

The following are not covered by this checkpoint:

  • Logo text
  • Disabled form controls
  • Purely decorative text
  • Text within an image that conatins significant other visual content
  • Visited link text

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Large text, including images of large text, must have a contrast ratio of at least 3:1 with the background.
  • If either background or text color is specified, the other must also be specified.
  • Large sized synchronized media captions must must have at least 3:1 contrast with the background, or the player must allow the user to adjust the appearance of captions.
User Story: As a user with a visual impairment, I want text to have good contrast so that I can more easily read it.
Success: If you had a visual impairment, when you viewed text, you would be able to see it well enough to read it.

Web Test Procedure

Desktop

ANDI

  1. Use ANDI: color contrast to automatically check for text contrast issues.
  2. Examine any Contrast Alerts under Accessibility Alerts which identify text that fails.
  3. Identify any alerts for Manual Contrast Tests Needed.
  4. Perform the Manual Checks steps below.

WAVE

  1. Use WAVE to automatically check for text contrast issues.
  2. Select the Contrast tab in the results pane.
  3. Select the low contrast icons in the pane or on the page to view details.

axe DevTools

  1. Run axe DevTools to automatically check for text contrast issues.
  2. Check for "Elements must meet minimum color contrast ratio thresholds" in the results.

Manual Checks

  1. Use a tool such as one listed in the Testing Tools below to manually check any text not covered by the automated tool. For example:
    • Text in hover states
    • Meaningful text embedded in images (check the area with the least contrast)
    • Text over images or gradients
    • Text in modal and modeless dialogs
    • Text in menus
    • Placeholder text

    Note: Check hex codes rather than using an eyedropper tool whenever possible to ensure you're checking the exact color.

  2. Verify that all large text has a contrast ratio of at least 3:1 with its background.

Mobile

  1. Open the page on a desktop computer in Chrome.
  2. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the Developer Tools.
  3. Activate the Toggle device toolbar button or press Ctrl+Shift+M.
  4. Select a mobile device from the Dimensions drop-down menu.
  5. Follow the procedure for Desktop Web.

Mobile App Test Procedure

iOS and Android

  1. Ensure that any contrast enhancements, such as Increase Contrast on iOS, are turned OFF.
  2. Visually check for large text which appears to have poor contrast with its background.
  3. Take a screenshot on the device and open it on a desktop computer.
  4. Use a manual tool such as one listed in the Testing Tools below to check the contrast ratio of the colors in the screenshot.
  5. Verify that the contrast ratio between the large text and its background is at least 3:1.
  6. If the app supports Dark Mode, test both Light and Dark Mode.
  7. If the app has any alternate themes that affect text and background colors, test each of these.

PDF Test Procedure

  1. Use PAC to automatically check for contrast issues.
  2. Look under WCAG > 1 Perceivable > 1.4 Distinguishable > 1.4.3 Contrast (Minimum) for failures.
  3. Use a manual tool such as one listed in the Testing Tools below to manually check any large text.
