Overview

Some examples of content that may obscure focus include:

  • Sticky footers
  • Sticky headers
  • Modeless dialogs

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Focus must not be entirely hidden by other content.
  • Focus should not be obscured by other content at all. (See 2.4.12 Focus Not Obscured (Enhanced).)
User Story: As a sighted keyboard user, I want the focus indictor to not be hidden so that I can see where I'm at on the page.
Success: If you were a sighted keyboard user, when navigate through a page, you could see what element was in focus.

Testing Logic

As you navigate through the page using the keyboard, is any part of the keyboard focus hidden behind other content?

  • Yes; Are you still able to tell what the element is that has focus?
    • Yes; PASS
    • No; Is the content that is obscuring the focused element in its initial position (e.g., it hasn't been opened or moved by the user)?
      • Yes; See {{ site.data.2411.2411[24110101].name }}.
      • No; Can the content be removed without changing keyboard focus?
        • Yes; PASS
        • No; See {{ site.data.2411.2411[24110102].name }}.
  • No; PASS

Web Test Procedure

  1. Using only the keyboard, navigate to and interact with all interactive elements and widgets.
  2. Verify that the element in focus is never entirely hidden behind other content.
