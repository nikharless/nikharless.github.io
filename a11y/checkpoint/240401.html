Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The purpose of each link must be determinable from the link text alone, or from the link text and its context.
  • Link text should be clear from the link text alone.
  • Link text and target page titles should match or be very similar.
  • Context for link purpose should be located before the link text.
  • Links to the same destination should have constistent text.
  • Links to different destinations should have different text.
  • Links to conforming alternate versions should indicate that they are for more accessible pages.
User Story: As a user with a motion impairment, I want links to clearly indicate their purpose so that I don't have to navigate to them in order to see what they are.
Success: If you were a user with a motion impairment, when links appear on the page, you can know what they are for without having to take extra steps to explore them.

Testing Logic

Can the purpose of each link be determined from the link text alone?

  • Yes; PASS
  • No; Can the purpose of the link be determined from the link text together with the text in the same sentence, paragraph, list item, or table cell?
    • Yes; PASS
    • No; Can the purpose of the link be determined from the link text together with additional text provided by aria-label or aria-labelledby attributes?
      • Yes; PASS
      • No; Is the purpose of the link ambiguous to users in general?
        • Yes; PASS
        • No; Is an image the only content in the link?
          • Yes; Does the image have alternate text which describes the link purpose?
            • Yes; PASS
            • No; See 2.4.4.1.2 Image link does not have alternative text describing purpose.
          • No; See 2.4.4.1.1 The purpose of the link cannot be determined without leaving the link.

Web Test Procedure

Desktop

  1. For each link on the page, check if the visible link text (or text alternative for an image link or image map link) alone is sufficient to describe the purpose of the link.
  2. If not, check if the visible link text (or text alternative for an image link or image map link) plus the text in the same sentence, paragraph, list item, table cell, or table header for the cell is sufficient to describe the purpose of the link.
  3. If not, check the accessible name for the link to see if the visible link text has been supplemented by an ARIA or title attribute or screen-reader-only text.

Desktop Screen Reader

  1. Using a screen reader like NVDA or JAWS, pull up a list of links.
    • NVDA: NVDA (Insert)+F7
    • JAWS: F7
  2. Check if the link text for each link adequately describes its purpose, without other contextual information.
  3. If not, check if tabbing to it announces contextual information. (This should happen if ARIA attributes are used.)
  4. If not, use screen reader keys to check if the immediate context of the link (sentence, paragraph, list item, associated row/column header cells) provides supplemental text properly describing the purpose of the link.
    • NVDA and JAWS
      • Read current line: NVDA+up arrow
      • Read current sentence: Alt+Numpad 5
    • JAWS
      • Read current paragraph: Ctrl+Numpad 5
    • VoiceOver
      • Read current line: VO+L
      • Read current sentence: VO+S
      • Read current paragraph: VO+P
      • Read current table row: VO+R
      • Read current table column header: VO+C

Mobile Screen Reader

  1. For each link on the page, check if the visible link text (or text alternative for an image link or image map link) alone is sufficient to describe the purpose of the link.
  2. If not, check if swiping to it announces contextual information. (This should happen if ARIA attributes are used.)
  3. If not, check if the visible link text (or text alternative for an image link or image map link) plus the text in the same sentence, paragraph, list item, table cell, or table header for the cell is sufficient to describe the purpose of the link.

PDF Testing

Adobe Acrobat Pro

  1. For each link in the document, check if the link text itself provides enough information, within its immediate context (the linkâ€™s parent container), to understand the purpose of the link.
  2. If not, locate the <Link> elements in the Tags tree.
  3. Right-click the <Link> elements to show their properties.
  4. Check if a more meaningful Alternate Text is provided for links that are not meaningful enough.

OR

Use the Read Out Loud feature (Menu > View > Read out loud) to have the link text read out.

Failures

  • 2.4.4.1.1 The purpose of the link cannot be determined without leaving the link. (Medium)
  • 2.4.4.1.2 Image link does not have alternative text describing purpose. (Medium)
  • 2.4.4.1.3 The purpose of the PDF link cannot be determined without leaving the link. (Medium)
