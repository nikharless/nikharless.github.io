Overview

This checkpoint applies to any changes of content occurring on the web page.

Forms

Form-related content changes may be triggered by things such as:

  • Changing values or states of of other form components
  • Toggling a function
  • Entering data in a text field
  • Mouseover
  • Keyboard focus

Form-related changes include but are not limited to:

  • Changes to instructions
  • Error notifications
  • Addition or removal of content, including other form components
  • Button name changes

Links

This checkpoint only applies to links which change the content of the current page, such as with a single-page application (SPA). It does not apply to links which reload the current page or load another page.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Automatic content changes must be communicated to assistive technologies.
  • Content changes to forms must be communicated to assistive technologies.
  • Content changes initiated by links must be communicated to assistive technologies.
  • If a dialog is used to alert the user of a content change, it must provide sufficient description of the content change.
  • If a focus change is used to alert the user of a content change, it must provide sufficient description of the content change.
  • If a status message or other ARIA Live Region is used to alert the user of a content change, it must provide sufficient description of the content change.
User Story: As a screen reader user, I want to be notified of any changes of content on the page so that I know they have occurred.
Success: If you were a screen reader user, when page content updates, you would receive a notification.

Web Test Procedure

  1. Using a screen reader, interact with every user interface component in the page.
  2. Verify that the screen reader notifies the user of any changes to states, properties, or values.
  3. Activate any links which change the content of the current page.
  4. Verify that the screen reader notifies the user of the content change.
  5. Note any automatic updates to the page content.
  6. Verify that the screen reader notifies the user of the content change.
