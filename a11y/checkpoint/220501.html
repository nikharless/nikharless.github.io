Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • There must be no loss of data when a user's authenticated session expires and they log back in.
User Story: As a user with a disability, I want my data from an expired session to remain available so that I can continue my task without starting over when I log back in.
Success: If you had a disability, when a session times out and you log back in, you can continue your task instead of having to start over.

Testing Logic

Is there a session time limit?

  • Yes; When the session times out, can the user log back in and continue where they left off?
    • Yes; PASS
    • No; See {{ site.data.2205.2205[22050101].name }}.
  • No; PASS

Web Test Procedure

Desktop

  1. Sign in to the site.
  2. Take some steps in a process. (Enter data, track progress, etc.)
  3. Allow the session to time out.
  4. Log back in and verify that you can continue where you left off and any progress has not been lost.

Mobile

Follow the process for Desktop.

Mobile App Test Procedure

Follow the process for Desktop.
