Overview

This criterion is similar to 2.2.1 Timing Adjustable, except time outs are only allowed for video and real-time events.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • There must not be time limits on the content.
User Story: As a user with a disability, I want to not encounter time limits so that I have adequate time to complete tasks.
Success: If you had a physical disability, when you use a web page, you are not subject to time limits on your interaction.

Testing Logic

Is there a time limit controlled by a script on the page?

  • Yes; Is the time limit required as part of a real-time event?
    • Yes; PASS
    • No; Is the content video?
      • Yes; PASS
      • No; See 2.2.3.1.1 Content requires timed interaction.
  • No; PASS

Web Test Procedure

Desktop

  1. Identify any content that has a time out function.
  2. Verify that the timed content is either video or a real-time event.

Mobile

Follow the process for Desktop.

Mobile App Test Procedure

Follow the process for Desktop.

Failures

  • 2.2.3.1.1 Content requires timed interaction. (Medium)
