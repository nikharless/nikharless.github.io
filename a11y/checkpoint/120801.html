Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Prerecorded synchronized media and video-only media must have a text transcript which includes the visual content.
  • If the video itself is an alternative to text, it must be clearly indicated as such.
  • The text transcript should be presented next to the video and be easy to find.
User Story: As a deafblind user, I want to read a transcript with my braille device so that I can get all of the same audio and visual information that is conveyed in the video.
Success: If you could not see or hear the video, when you read the transcript with assistive technology, you would get all of the information conveyed in the audio and video portions.

Testing Logic

Does the page contain video-only content or synchronized media (e.g., video with dialogue)?

  • Yes; Is the video purely decorative?
    • Yes; PASS (Note: Make sure it's hidden from assistive technology.)
    • No; Does the media have a text transcript?
      • Yes; Does the transcript include all the important dialog, sounds, actions, visuals, and text in the video?
        • Yes; PASS
        • No; See {{ site.data.1208.1208[12080102].name }}
      • No; Is the video itself a media alternative for text?
        • Yes; Is it clearly labeled as such?
          • Yes; PASS
          • No; See {{ site.data.1208.1208[12080103].name }}
        • No; See {{ site.data.1208.1208[12080101].name }}
  • No; PASS

Web Test Procedure

  1. Locate each video element on the page.
  2. Check whether the video itself is an alternative to text on the page.
  3. If it is, verify that it is clearly labeled as such.
  4. If it isn't, verify that it has a text transcript that includes both the audio and the visual information.

Mobile App Test Procedure

  • Follow the procedure for Web.
