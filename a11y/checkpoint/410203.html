Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The states of user interface components which can be set by the user must be programmatically determinable by assistive technologies.
  • Notifications of changes to states of user interface components must be communicated to assistive technologies.
User Story: As a screen reader user, I want the states of user interface components to be programmatically determinable so that I can get this information via my screen reader.
Success: If you were a screen reader user, when accessing user interface components, you would be able to determine the states of the components.

Web Test Procedure

Desktop

User Interface Components

  1. Using a screen reader, navigate to every user interface component in the page.
  2. Verify that the screen reader speaks any state that can be set by the user.
  3. Verify that the screen reader notifies the user of any changes to states.

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Follow the procedure for Desktop Web.

Mobile App Test Procedure

iOS

  1. Turn off image recognition features.
  2. Follow the procedure for Desktop Web.
  • Selected: “[accessible name], button, selected”
    • Note: “button” may be optional, depending on whether the selected item is within a listview.
  • Expanded: “[accessible name], button, expanded [or] collapsed”
    • Note: “button may be optional, depending on whether the selected item is within a listview.
  • Disabled: “[accessible name], dimmed, button”

Android

  1. Follow the procedure for Desktop Web.
  • Buttons (including text, image, and floating action buttons):
    • “[accessible name], button”, OR
    • “[accessible name] (pause), Double tap to activate”, OR
    • “[accessible name], button (pause), Double tap to activate” or similar
  • Links:
    • An earcon plays identifying the text span as a link (“link” is NOT spoken), AND
    • “[accessible name], Double tap to activate” or similar
  • Alert or popup: “Alert” or “Popup”, then:
    • First text block, OR
    • Pane title as set by accessibilityPaneTitle
  • Badge: “[accessible name], has [x] notifications, [rest of announcement]”
  • Bottom navigation:
    • Selected state for currently active top-level destination, AND
    • “[accessible name], Double tap to activate” or similar
  • Checkbox or radio button:
    • “Checked” or “Not checked”, then “[accessible name], Double tap to activate” or similar
      • May or may not say “checkbox” or “radio button” before “Double tap to activate” or similar
  • Compound button or toggle button (i.e., a button with a “pressed” state):
    • “Checked” or “Not checked”, then [accessible name], Double tap to activate” or similar (no “checkbox” or “button” role is read)
  • Text field:
    • “[accessible name], Edit box”, OR
    • “[accessible name], Edit box for [accessible name]”, OR
    • “[entered value], Edit box for [accessible name]”, OR
    • “Editing Edit box for [accessible name]”, OR
    • “Editing [entered value], Edit box for [accessible name]”
      AND
    • “(pause) Double tap to enter text. Double tap and hold to long press”, OR
    • “(pause) Double tap and hold to long press” or similar
  • Expandable list view: “[accessible name], Collapsed (or) Expanded, Double tap to activate” or similar
  • Menu (various types): “[role], [accessible name of first menu item]
  • Progress bar:
    • Indeterminate: “Loading… [accessible name]”
    • Determinate: “Loading… [accessible name], [current %]”
  • Rating bar: “[Percentage], [accessible name], rating”
  • Seek bar:
    • “[Percentage], [accessible name], Seek control (pause), Use volume keys to adjust”, OR
    • “[Percentage], Seek control for [accessible name] (pause), Use volume keys to adjust” or similar
  • Snackbar or toast: A polite accessibility announcement of the text in the snackbar or toast
  • Spinner:
    • Collapsed state: “Drop-down list, [value] for [accessible name], Double tap to change, OR
    • “Drop-down list, [value], Double tap to change” or similar (may or may not say “drop-down list”
      AND/OR
    • Expanded state: “[accessible name], [x] of [x]” for each list item
  • Switch:
    • “[accessible name], [state], Switch (pause), Double tap to activate, OR
    • “[state], Switch for [accessible name] (pause), Double tap to toggle” or similar
