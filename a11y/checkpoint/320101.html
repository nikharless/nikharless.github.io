Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • When a control gains focus, it must not result in a change of context.
User Story: As a screen reader user, I want the context to not change unexpectedly so that I know where I am.
Success: If you were a screen reader user, when you navigate to a control, the context would not unexpectedly change.

Testing Logic

Is there a change of context when a control gains focus?

  • Yes; Is the focus automatically removed or redirected?
    • Yes; See {{ site.data.3201.3201[32010101].name }}.
    • No; Is a new window, tab, or dialog launched?
      • Yes; See {{ site.data.3201.3201[32010102].name }}.
      • No; Is there a significant change of content before the current focus?
        • Yes; See {{ site.data.3201.3201[32010103].name }}.
        • No; Is a form automatically submitted?
          • Yes; See {{ site.data.3201.3201[32010104].name }}.
          • No; See {{ site.data.3201.3201[32010105].name }}.
  • No; PASS

Web Test Procedure

Desktop

  1. Using only the keyboard, navigate through all controls on the page.
  2. Verify that there is no unexpected change of context when any control gains focus.

Mobile

  1. Turn on the screen reader.
  2. Swipe to navigate through all controls on the page.
  3. Verify that there is no unexpected change of context when any control gains focus.

Mobile App Test Procedure

Follow the procedure for Desktop Mobile.
