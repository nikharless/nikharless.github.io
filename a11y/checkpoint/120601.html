Example

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Prerecorded synchronized media must have a sign language interpretation for the audio content.
User Story: As a deaf user, I want to view a sign language interpretation so that I can get all of the same information that is conveyed in the audio portion of the video.
Success: If you could not hear the video, when you watch the sign language interpretation you would get all of the information conveyed in the audio portion.

Testing Logic

Does the page contain synchronized media (e.g., video with dialogue)?

  • Yes; Does the audio content in the synchronized media have a sign language interpretation?
    • Yes; Is the sign language interpretation accurate?
      • Yes; PASS
      • No; See {{ site.data.1206.1206[12060102].name }}
    • No; See {{ site.data.1206.1206[12060101].name }}
  • No; PASS

Web Test Procedure

  1. Locate each video element on the page.
  2. Verify that it has a sign language interpretation that provides the audio information.
  3. Use a sign language interpreter to verify the accuracy.

Mobile App Test Procedure

  • Follow the procedure for Web.
