Overview

If any functionality is dropped because of responsive design changes, this is not a failure if the functionality can be accessed by clicking or tapping to a new location (e.g., expandable menu, link, etc.).

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Text content must be resizable to 200% without the use of assistive technology.
  • Resizing must not be disabled using <meta name="viewport" content="user-scalable=no">.
  • Resizing must not be disabled using <meta name="viewport" content="maximum-scale=[<2 or yes]">.
  • Text must not be clipped, truncated, or obscured when resized to 200%.
  • All content must be available within one click or tap when resized to 200%.
  • All functionality must be retained when resized to 200%.
  • Text content should be resizable to 500%.
  • Mobile apps should use the mobile platform's dynamic text accessibility feature.
User Story: As a user with a visual impairment, I want text to be zoomable and remain readable and functional up to 200% so that I can more easily read it.
Success: If you had a visual impairment, when you enlarged text up to 200%, you would be able to read it and use its functionality.

Web Test Procedure

Desktop

  1. Use the browser zoom function (Ctrl++) to resize the text to 200%.
  2. If any content does not zoom, verify that there is another non-assistive-technology mechanism to resize the page content (e.g. operating system, platform, or author-provided mechanism).

Mobile

  1. Open the page on a desktop computer in Chrome.
  2. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the Developer Tools.
  3. Use axe DevTools to scan the page, and check for the error: "Zooming and scaling must not be disabled."

OR

  1. Open the page on the mobile device.
  2. Attempt to zoom the page using the pinch to zoom gesture.
  3. Note: Newer mobile devices usually ignore code which disables zoom, so testing on an older device or checking the code is recommended.

Mobile App Test Procedure

iOS and Android

This checkpoint is automatically passed due to the platform's built-in zoom feature.

PDF Test Procedure

Set the zoom level to 200%, and verify that all text content remains readable and usable.
