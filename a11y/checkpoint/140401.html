Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Text content must be resizable to 200%.
  • Text content must be readable and functional when resized up to 200%.
  • Text content should be resizable to 500%.
  • Mobile apps should use the mobile platform's dynamic text accessibility feature.
User Story: As a user with a visual impairment, I want text to be zoomable and remain readable and functional up to 200% so that I can more easily read it.
Success: If you had a visual impairment, when you enlarged text up to 200%, you would be able to read it and use its functionality.

Testing Logic

Can the text be resized without assistive technology up to 200%?

Note: This includes text the user enters into text fields.

  • Yes; PASS
  • No; Is the text a caption or image of text?
    • Yes; PASS
    • No; Is the font size set in viewport units (vw)?
      • Yes; See {{ site.data.1404.1404[14040101].name }}
      • No; Does the viewport meta element prevent user scaling (user-scalable=no or maximum-scale set to yes or a number less than 2)?
        • Yes; See {{ site.data.1404.1404[14040102].name }}
        • No;

When text is resized up to 200%, is the content truncated?

  • Yes; Is the full content available on focus or after user activation?
    • Yes; Is there an indication that this information can be accessed (other than the fact that it is truncated, such as “…”)?
      • Yes; PASS
      • No; See {{ site.data.1404.1404[14040104].name }}
    • No; See {{ site.data.1404.1404[14040103].name }}
  • No; PASS

When text is resized up to 200%, is the functionality still available?

  • Yes; PASS
  • No; Has the functionality been moved to an area that is one click away (such as in a pop-up menu)?
    • Yes; PASS
    • No; See {{ site.data.1404.1404[14040106].name }}

Web Test Procedure

Desktop

  1. Use the Web Developer browser extension for Firefox and Chrome to compare zoom levels. Both browsers will be used for this test.
    1. Browser 1 (Firefox):
      1. In the first browser, set the zoom level to 100%.
      2. Open the Web Developer extension, and select Resize > Resize Window....
      3. Set Width to "1366" and Height to "768".
      4. Open the Web Developer extension again and select Display Window Size.
      5. Note the difference between the Window and Viewport size, and adjust the Viewport size to 1366x768. For example:
        • (Window) 1366px - (Viewport) 1355px = 11px
        • 1366px + 11px = 1377px
        • (Window) 768px - (Viewport) 675px = 93px
        • 768px + 93px = 861px
        • Open the extension, select Resize > Resize Window..., and set it to Width: 1377 and Height: 861px.
    2. Browser 2 (Chrome):
      1. In the second browser, follow steps 1-5 of Browser 1 instructions.
      2. Zoom the window to 200%.
  2. Compare the 100% zoom view to the 200% zoom view, and verify that:
    • All text content has been resized
    • All content and functionality has been retained
  3. If either of the above fail, in browser 2 (Chrome):
    1. Set the zoom level to 100%.
    2. Go to Settings > Appearance > Customize fonts.
    3. Slide the Font size slider to "32".
    4. Compare the 100% zoom view to this view and verify the above.

Note: If any functionality is dropped because of responsive design changes, this is not a failure as long as the functionality can be accessed by clicking or tapping to a new location (e.g., expandable menu, link, etc.).

Mobile

  1. Open the page on a desktop computer in Chrome.
  2. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the Developer Tools.
  3. Use axe DevTools to scan the page, and check for the error: "Zooming and scaling must not be disabled."

OR

  1. Open the page on the mobile device.
  2. Attempt to zoom the page using the pinch to zoom gesture.
  3. Note: Newer mobile devices usually ignore code which disables zoom, so testing on an older device or checking the code is recommended.
