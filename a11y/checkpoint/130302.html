Overview

Example

If a multiple screen process requires the user to hear a sound cue before being allowed to move on to the next screen using the right arrow key, this indication must also be provided visually in text. (From Deque)

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Instructions provided for understanding and operating content do not rely solely on sound cues such as beeps, chimes, or speech.
User Story: As a deaf screen reader user, I want instructions to not require hearing so that I can follow the them.
Success: If you could not hear, when you read instructions, you would still understand how to follow them.

Testing Logic

Do any instructions on the page refer to sound cues such as beeps, chimes, or speech?

  • Yes; Do the instructions also include references which do not rely on sound cues?
    • Yes; PASS
    • No; See 1.3.3.2.1 Instructions require the ability to hear.
  • No; PASS

Web Test Procedure

Desktop

  1. Locate any instructions related to understanding or operating content.
  2. Determine if any of the instructions reference auditory characteristics such as beeps, chimes, or speech.
  3. If so, verify that they also reference another non-sound characteristic, such as a text label, that can be accessed people who cannot hear.

Failures

  • 1.3.3.2.1 Instructions require the ability to hear. (Medium)
