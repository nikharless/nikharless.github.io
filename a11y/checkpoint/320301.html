Overview

Navigation mechanisms are any components that provide a means for the user to find and navigate to specific content on the website. Navigation mechanisms may include:

  • Search fields
  • Navigation menus
  • Tables of Content
  • Skip to main content links
  • Back to top links

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Navigation mechanisms which appear across multiple web pages must appear in the same relative order with regard to other repeated components on each page.
User Story: As a user with a disability, I want the navigation items to be in the same place each time they appear so that I don't have to rediscover them.
Success: If you were a user with a disability, when navigation items appear on multiple web pages in a set, they appear in the same relative position each time.

Testing Logic

Are there navigation mechanisms that are repeated across multiple web pages in the set of web pages?

  • Yes; Do they appear in the same relative order each time they appear?
    • Yes; PASS
    • No; See {{ site.data.3203.3203[32030101].name }}.
  • No; This checkpoint is not applicable.

Web Test Procedure

  1. Review the web pages within a website, a logical section of a site, or a web application. Do not change zoom level, orientation, or viewport size between pages.
  2. Look for navigation components such as navigation menus, search fields, or tables of contents that are repeated on multiple pages.
  3. Verify that the main navigation links are in the same relative order on each page.
  4. If there are multiple navigation components, verify that the components are in the same relative place and order on each page.
  5. The ANDI: focusable elements "tab order" feature may help for checking the focus order of components, and the ANDI: structure "reading order" feature may help for checking the content order of both focusable and non-focusable components.

PDF Test Procedure

  1. Review the pages within the document.
  2. Look for navigation components that are repeated on multiple pages.
  3. If headers, footers, and/or page numbers exist, verify that they are in the same relative location on each page.
