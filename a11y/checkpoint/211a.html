Basic Requirements and Best Practices

= Requirement
= Best Practice

  • All functionality must be available through the keyboard interface.
  • Keyboard functionality must not require specific timings for individual keystrokes.
  • Modals must be dismissable with the keyboard without having to submit anything.
    • Modals should be dismissable using the Esc key.
    • Modals should be dismissable using a button.
  • Instructions should be provided to explain any non-standard keyboard interaction required to use the functionality.
User Story: As a keyboard user, I want to be able to access all functionality with the keyboard so that I can use the page.
Success: If you used only a keyboard, when you access a page, you could use all functionality.

Exception: Function where input depends on the path of the user's movement

Examples that can and must be made accessible include:

  • Drag and drop
  • Resizing
  • Drawing a straight line
  • Drawing preset geometric shapes (circle, square, triangle, star, spiral)

Testing Logic

Is all page functionality available using the keyboard?

  • Yes; PASS
  • No; Does the underlying function require input that depends on the path of the user's movement?
    • Yes; PASS
    • No; See

Is there an image map on the page?

  • Yes; Is it a client-side image map?
    • Yes; PASS
    • No; Does the page also include redundant text links for each active region within the server-side image map?
      • Yes; Are the links easily located by users (for example, immediately following the image map)?
        • Yes; PASS
        • No; See
      • No; See
  • No; PASS

Can all interactive elements (including all scrollable regions) receive keyboard focus?

  • Yes; PASS
  • No; Is there an accessible version of the same functionality on the same page?
    • Yes; PASS
    • No; See

Web Test Procedure

Desktop Keyboard

  1. Using only the keyboard (no mouse or screen reader), navigate through the page.
    • Press the Tab key to navigate the focus through menu items, form inputs, links, help items, and other interactive UI elements.
    • Press Shift+Tab to navigate the focus in the reverse direction through all interactive UI elements.
    • In grouped inputs, menus, or list item elements, press the up, down, right, or left arrow keys to move between items inside the element.
    • Press the Enter key or the Space bar to select or activate the interactive UI element that has focus.

Desktop Screen Reader

  1. Turn on the screen reader.
  2. Follow the process for Desktop Keyboard.

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to each functional image.
  3. Verify that the name spoken by the screen reader accurately describes the purpose of the functional image.

Mobile App Test Procedure

  • Follow the procedure for Web Mobile.
  • iOS: Verify that the button, button image, or link image trait is spoken, AND/OR hint text such as "Double tap to activate" is spoken.
  • Android: Verify that the button role is spoken, AND/OR hint text such as "Double tap to activate" is spoken.

PDF Test Procedure

Find missing text alternatives.

PDF Accessibility Checker

  1. Run PAC on the PDF.
  2. Select the WCAG tab and check if 1.3 Adaptable is marked as a failure.
  3. If it is, activate the Results in Detail button.
  4. Check if Alternative text for "Figure" structure elements is marked as a failure.

Adobe Acrobat Pro

  1. To find missing text alternatives, select Accessibility > Accessibility Check in the Task pane to open the Accessibility Checker Options dialog.
  2. Select Alternate Text and Headings from the Category drop-down, and make sure Figures require alternate text is checked.

Verify the text alternatives are sufficient.

  1. Locate each functional image in the PDF using one or more of the following methods:
    • Mouse: Hover over images, noting cursor change if they are functional.
    • Keyboard: Use the Tab key to focus functional images.
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  2. Verify that each functional image has a text alternative which accurately describes its purpose.

Failures

  • F2.1.1.a.a The functionality cannot be accessed using only a keyboard. (High)
  • F2.1.1.a.b The element uses mouse event handlers but does not have equivalent keyboard event handlers. (High)
  • F2.1.1.a.c The link does not have an href value. (High)
  • F2.1.1.a.d The modal cannot be dismissed using the keyboard without submitting data. (High)
  • F2.1.1.a.e The server-side image map does not have redundant text links for keyboard users. (High)
  • F2.1.1.a.f The drag and drop feature does not have keyboard functionality. (High)
