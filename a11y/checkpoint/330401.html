Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • If the user can carry out legal or financial transactions, submit test responses, or change or delete stored data, the submissions must be reversible, verifiable, or confirmable.
User Story: As a user with a disability, I want to be able to check, confirm, or reverse transactions so that I can avoid serious consequences of mistakes.
Success: If you were a user with a disability, when submitting transactions with serious consequences, you would have the opportunity to check, confirm, or reverse the transaction.

Testing Logic

Does the page functionality cause transactions with serious consequences to occur?

  • Yes; Does the page cause legal commitments or financial transactions to occur?
    • Yes; Is the data entered by the user checked for input errors and the user given the opportunity to correct them?
      • Yes; PASS
      • No; Is the user given the opportunity to review, confirm, and correct information before finalizing the transaction?
        • Yes; PASS
        • No; Is the submission reversible?
          • Yes; PASS
          • No; See {{ site.data.3304.3304[33040101].name }}.
    • No; Does the page cause user-controllable data to be changed or deleted?
      • Yes; Is the user given the opportunity to confirm the action before finalizing it?
        • Yes; PASS
        • No; Is the submission reversible?
          • Yes; PASS
          • No; See {{ site.data.3304.3304[33040102].name }}.
      • No; Does the web page submit test answers?
        • Yes; Is the user given the opportunity to review and confirm the answers before final submission?
          • Yes; PASS
          • No; See {{ site.data.3304.3304[33040103].name }}.
        • No;
  • No; Not applicable

Web Test Procedure

Desktop

  1. Locate any pages where a user action can cause an important legal, financial, test/exam or unrecoverable/unchangeable user data transaction to occur.
  2. Verify that the transaction can be either:
    • Reversed by the user, OR
      • Instructions are provided explaining how to reverse or cancel the transaction.
      • A keyboard-only user can reverse or cancel the transaction.
    • Verified by the user, OR
      • Data entered by the user is provided for review before final submission, and the user is able to make changes to this data.
    • Confirmed by the user
      • A mechanism (such as a checkbox) is provided for the user to confirm the transaction.
      • Validate that if the checkbox is left unchecked, the transaction is not processed.

Mobile

Using VoiceOver or TalkBack, follow the Desktop Web procedure.

Mobile App Test Procedure

Follow the procedure for Mobile Web.
