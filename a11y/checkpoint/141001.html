Overview

The content must be viewable and usable without scrolling in two dimensions when the viewport is set to 320 CSS pixels wide.

Exceptions may include:

  • Maps
  • Diagrams
  • Video
  • Games
  • Presentations
  • Data tables (not individual cells)
  • Interfaces where itâ€™s necessary to keep toolbars in view while manipulating content

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The content must be viewable without scrolling in two dimensions when the viewport is set to 320 CSS pixels wide.
  • The content must not be cropped or rendered unreachable when the viewport is set to 320 CSS pixels wide.
  • Videos should resize to fit within the viewport.
  • Images should resize to fit within the viewport.
User Story: As a user with low vision, I want to be able to zoom the content without having to scroll horizontally so that I don't become disoriented.
Success: If you had low vision, when you enlarged page content, you would be able to read it without having to scroll in more than one direction.

Testing Logic

When the content is presented at an equivalent width of 320 CSS pixels or an equivalent height of 256 CSS pixels, does the content require scrolling in more than one direction?

  • Yes; Is the presentation of the text essential (e.g., a logo)?
    • Yes; PASS
    • No; Is two-dimensional scrolling essential for the content (e.g., maps, games, presentations, etc.)?
      • Yes; PASS
      • No; See {{ site.data.1410.1410[14100102].name }}
  • No; Is any content cropped or unreachable due to not being able to scroll in a second dimension?
    • Yes; See {{ site.data.1410.1410[14100101].name }}
    • No; PASS

Web Test Procedure

Desktop

  1. Open the page in Chrome at the actual size (e.g., no zoom, magnification, or user-supplied CSS features are turned on).
  2. Resize the browser viewport to either:
    • 640px width by 512px height at 200% zoom, OR
    • 1280px width by 1024px height at 400% zoom
  3. Verify that you can see all the content on the page without scrolling horizontally.

Mobile

This checkpoint does not apply to mobile, as horizontal scrolling with pinch-zoom is not something the developer can control.

PDF Test Procedure

  1. View the document at 200%.
  2. Select View > Zoom > Reflow.
  3. Verify that the content remains readable when reflowed.
