Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Focus order must preserve the meaning and operability of the content.
    • Non-linear DOM order must be logical and intuitive.
    • Postive tabindex values must not cause illogical focus order.
  • Focus must not be lost or incorrectly placed during interaction or content update.
  • Modeless dialogs must appear immediately after the trigger element.
  • Focus must move to modals when they are opened.
  • Focus must be restricted within modals while they are open.
  • When a modal is closed, focus must be returned to the trigger element or to the most logical location.
  • Keyboard focus must follow a scripted visual scroll.
  • Keyboard focus must not move to hidden elements.
  • Focus order and visual order should match.
User Story: As a keyboard-only user, I want the focus order to be logical so that I can easily navigate the content.
Success: If you were a keyboard user, when you navigate a web page, the focus order would be logical and intuitive.

Web Test Procedure

Desktop

  1. Starting at the top of the page, press Tab to navigate through the page.
  2. Verify that focus moves through the page in a logical and predictable manner.
  3. Navigate to interactive elements such as buttons, links, or tabs that show or hide content within the page.
  4. Activate or interact with the elements using Enter, Space bar, or arrow keys, as appropriate.
  5. When content is added or removed, navigate through the content using the Tab or arrow keys as appropriate.
  6. Verify that the focus order preserves meaning and operability.
  7. For modals, verify that the focus moves to the modal when it is opened, remains within the modal while it is open, and moves to the opener control or the next logical place when the modal is closed.
  8. Repeat the above steps if the interactive content introduces additional interactive functionality.
  9. Activate in-page links or interactions, such as back to top links, skip to main content links, links to and from footnotes, etc.
  10. Verify that when visual focus is moved (i.e., the viewport is shifted), keyboard focus is also moved to the intended target.
  11. Note: It may be helpful to launch ANDI: focusable elements and select tab order.

Mobile Screen Reader

  1. Turn VoiceOver or TalkBack on.
  2. Starting at the top of the page, swipe to navigate through the page.
  3. Verify that focus moves through the page in a logical and predictable manner.
  4. Interact with elements such as buttons, links, or tabs that show or hide content within the page.
  5. When content is added or removed, verify that the focus order preserves meaning and operability.
  6. Repeat the above steps if the interactive content introduces additional interactive functionality.
  7. Activate in-page links or interactions, such as back to top links, skip to main content links, links to and from footnotes, etc.

PDF Testing

  1. Tab through all the active elements in the document (links and form controls).
  2. Verify that the tabbing order through active elements is logical and predictable.
