Overview

  • Dynamic changes to form labels must be included.
  • Form elements are not required to be programmatically associated with form section headings unless there is significant risk of confusion.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Form elements must be programmatically associated with their visible labels and/or instructions, including expected format and when fields are required.
  • Form elements in tables must be programmatically associated with their corresponding table headers.
  • Form elements must be programmatically associated with any other visual associations.
User Story: As a blind user, I want to hear the roles of elements so that I know their purpose.
Success: If you could not see the structure of a page, when you read the page with a screen reader, you would understand the purpose of each element.

Testing Logic

Does the page contain form elements?

  • Yes; Do the form elements have visible labels and/or instructions?
    • Yes; Are the labels and/or instructions programmatically associated with the corresponding form elements?
      • Yes; PASS
      • No; See {{ site.data.1301.1301[13010601].name }}.
    • No; Do the form elements have associated table row or column headers?
      • Yes; Are the table headers marked up as header cells?
        • Yes; PASS
        • No; See {{ site.data.1301.1301[13010602].name }}.
      • No; Do the form elements have any other visible associations that describe their purpose?
        • Yes; Are the visible associations programmatically associated with the corresponding form elements?
          • Yes; PASS
          • No; See {{ site.data.1301.1301[13010603].name }}.
  • No; This checkpoint is not applicable.

Web Test Procedure

  1. Launch ANDI: focusable elements.
  2. Review each form element and determine if the ANDI Output adequately describes the form element.
  3. If the form elements are within tables, launch ANDI: tables.
  4. Review each form element cell and determine if the ANDI Output adequately describes the form element.
  5. If the form elements have any other visible associations, verify that they are programmatically associated.

PDF Test Procedure

Adobe Acrobat Pro

  1. Select Accessibility > Accessibility Check from the Task pane toolbar.
  2. Ensure all Checking Options are selected, and activate the Start Checking button.
  3. In the Accessibility Checker results, check for any failures related to untagged content.
  4. Select Tags from the Navigation pane toolbar.
  5. In the Options menu, verify that Document is Tagged PDF is selected.
  6. Go through the Tags tree, and make sure every element is tagged with the appropriate semantic tag.

Screen Reader

  1. Examine the page for information and relationships that are presented visually.
  2. Verify that these are identified correctly by the screen reader.
  3. Verify that the page numbering is consistent.
