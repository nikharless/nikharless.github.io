Overview

All functionalities based on custom gestures must also be available when a screen reader is turned on.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • All functionalities based on custom gestures must also be available when a screen reader is turned on.
User Story: As a keyboard and screen reader user, I want to be able to access all functionality with the keyboard so that I can use the page.
Success: If you used a screen reader along with the keyboard, when you attempt to use the functionality on the page, it does not conflict with screen reader gestures.

Testing Logic

Does the app/web page have any functionalities that are based upon custom gestures?

  • Yes; Do the custom gestures function properly with a screen reader activated?
    • Yes; PASS
    • No; See {{ site.data.2101.2101[21010301].name }}
  • No; PASS

Mobile App Test Procedure

iOS

  1. Using VoiceOver, swipe to a control which accepts custom gestures.
  2. Verify that VoiceOver announces “Actions available” for custom gestures.
    1. If VoiceOver does not say “Actions available,” open the Rotor.
    2. Actions should become an available option in the Rotor, and the Rotor should automatically be set to Actions when it opens.
  3. Verify that you can activate the action with VoiceOver.

Android

  1. Using TalkBack, swipe to a control which accepts custom gestures.
  2. Verify that TalkBack announces “Actions available” for custom gestures.
  3. Verify that you can swipe up and right to open the context menu for the available actions.
  4. Verify that you can activate the action.
