Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The user must be able to pause, stop, hide, or control the frequency of automatically updating content.
User Story: As a user with an attention deficit disorder, I want to be able to pause, stop, hide, or control the frequency of automatically updating content so that I can focus on other page content.
Success: If you had an attention deficit disorder, when you encounter automatically updating content, you are able to pause, stop, hide it or change the frequency of updates to allow you to focus on other page content.

Testing Logic

Is there any auto-updating content on the page which starts automatically?

  • Yes; Is the auto-updating content presented in parallel with other content?
    • Yes; Is the auto-updating content part of an activity where it is essential?
      • Yes; PASS
      • No; Is there a mechanism for the user to pause, stop, hide, or control the frequency of the updates?
        • Yes; Does the mechanism allow the user to interact with other content while the updates are paused?
          • Yes; PASS
          • No; See
        • No; See
    • No; PASS
  • No; PASS

Web Test Procedure

Desktop

  1. Identify all automatically updating content that starts automatically and is presented along with other content.
  2. Verify that there is a mechanism available to stop, pause, or hide the content or to control the frequency of the update.

Desktop Screen Reader

  1. Turn on the screen reader.
  2. Follow the process for Desktop.

Mobile

Screen Reader (iOS and Android)

Follow the process for Desktop Screen Reader.

Mobile App Test Procedure

Follow the procedure for Web Mobile.

Failures

  • F2.2.2.b.a Automatic carousel has no means to pause, stop, hide or control the frequency. (Medium)
  • F2.2.2.b.b Auto-updating content has no means to pause, stop, hide or control the frequency. (Medium)
