Overview

When content is inserted into or unhidden in a web page, the reading order must remain logical both from the overall page sequence and starting from the user's current position. It's not logical, for instance, to insert new content before the user's current position in the page unless the user is alerted that it has been inserted.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The reading order of content which is dynamically inserted or unhidden must be logical and intuitive in the context of the page.
  • Dynamic content should be inserted after the triggering element.
User Story: As a blind screen reader user, I want the content to be read in a meaningful sequence so that I can understand the content.
Success: If you could not see the web page, when you use a screen reader, you would hear the content in an order that makes logical sense.

Testing Logic

Is content dynamically inserted or unhidden on the page?

  • Yes; Is the added content read in a logical order by assistive technology?
    • Yes; PASS
    • No; See {{ site.data.1302.1302[13020301].name }}
  • No; This checkpoint is not applicable.

Web Test Procedure

Desktop

  1. Use ANDI: hidden content.
  2. Activate reveal all.
  3. Verify that the reading order of the content still follows a logical sequence with the hidden content revealed.

Screen Reader

  1. Starting at the top of the page, navigate to and activate any controls that add or delete content from the page.
  2. Verify that the reading order of the new content is logical in the natural flow of the interaction.
