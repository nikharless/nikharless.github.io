Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Live audio must have a text equivalent.
User Story: As a deaf user, I want to read a text equivalent for an audio broadcast so that I can get all of the same information that is conveyed through the audio.
Success: If you could not hear the audio, when you read the text equivalent, you would get all of the information conveyed in the audio.

Testing Logic

Does the page contain any live audio broadcasts?

Note: This criterion does not apply to two-way audio calls between multiple individuals.

  • Yes; Does the audio broadcast have a procedure in place to provide real-time text equivalent?
    • Yes; Is the text equivalent accurate?
      • Yes; Do the text alternatives include all dialogue?
        • Yes; Do the text alternatives include important sound effects?
          • Yes; Do the text alternatives identify the speakers?
            • Yes; Are the text alternatives easy to read, or can their appearance be adjusted by the user?
              • Yes; PASS
              • No; See F1.2.9.a.g Existing live text alternatives are difficult to read.
            • No; See F1.2.9.a.f Existing live text alternatives do not identify speakers.
          • No; See F1.2.9.a.e Existing live text alternatives are missing important sound effect descriptions.
        • No; See F1.2.9.a.d Existing live text alternatives are missing dialogue.
      • No; See F1.2.9.a.b Existing live text alternatives are not accurate.
    • No; See F1.2.9.a.a Live audio broadcast does not have a text equivalent.
  • No; PASS

Web Test Procedure

  1. Locate each live audio element on the page.
  2. Verify that there is a procedure in place to provide real-time text alternatives.
  3. Verify that the text alternatives match the audio.
  4. Verify that the text alternatives identify the speaker.
  5. Verify that the text alternatives can be easily read or that their appearance can be adjusted by the user.

Mobile App Test Procedure

  • Follow the procedure for Web.

Failures

  • F1.2.9.a.a Live audio broadcast does not have a text equivalent. (High)
  • F1.2.9.a.b Existing live text alternatives are not accurate. (Medium)
  • F1.2.9.a.c Existing live text alternatives are missing dialogue. (Medium)
  • F1.2.9.a.d Existing live text alternatives are missing important sound effect descriptions. (Medium)
  • F1.2.9.a.e Existing live text alternatives do not identify speakers. (Low)
  • F1.2.9.a.f Existing live text alternatives are difficult to read. (Medium)
