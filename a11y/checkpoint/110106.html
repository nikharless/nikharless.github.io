Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Each audio and video element must have a text alternative which summarizes its purpose.
  • If there is text in the audio/video preview, that exact text should be included in the text alternative.
User Story: As a blind screen reader user, I want to hear a summary description of the audio or video content so that I know whether I want to explore it further.
Success: If you could not see the audio or video element, when you read it with a screen reader, you would hear a summary description of the media.

Testing Logic

Is there audio content on the page?

  • Yes; Does the audio have a short text alternative?
    • Yes; Does the text alternative sufficiently summarize the content of the audio?
      • Yes; PASS
      • No; See {{ site.data.1101.1101[11010605].name }}
    • No; See {{ site.data.1101.1101[11010601].name }}
  • No; PASS

Does the text alternative for the audio-only content refer to the accompanying transcript?

Is there video content on the page?

      • Yes; PASS

Web Test Procedure

Desktop

  1. Locate each audio or video element using one or more of the following methods:
    • Visually
    • Windows / NVDA: Use the O key to cycle through objects.
    • MacOS / VoiceOver: Use VO+Command+O to cycle through objects.
  2. Verify that it has a text alternative that summarizes its content.

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to each audio or video element.
  3. Verify that it has a text alternative that summarizes its content.

Mobile App Test Procedure

  • Follow the procedure for Web Mobile.
