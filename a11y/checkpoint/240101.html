Overview

Examples of acceptable mechanisms include:

  • Skip links
  • Semantic heading structure
  • ARIA landmarks

Note that landmarks and headings are only usable by screen reader users. Keyboard-only users who do not use screen readers only benefit from skip links, so it's recommended that they be used even in addition to the other methods.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • A method must be provided to skip navigation and other elements that are repeated across screens.
  • Landmarks should be used.
  • All sections of the page should be defined within landmarks.
  • A skip link should not be included when there is no repetitive block of content on the page.
  • A skip navigation link should be available to screen reader users and keyboard-only users regardless of other methods available.
  • The skip link to the main content should be the first link on the page.
User Story: As a keyboard or screen reader user, I want to be able to skip blocks of repeated content so that I do not have to navigate through it each time I encounter it.
Success: If you were a keyboard-only user, when blocks of content are repeated across pages, you would be able to skip past them to the content you want.

Testing Logic

Does the page contain blocks of content (such as navigation links, heading graphics, and advertising frames) that are repeated across multiple pages?

  • Yes; Does the page include a link at the very top that goes to the main content area, or a link at the beginning of a repeated content block that goes to the end of the block?
    • Yes; When activated, does the link move the focus to the correct area?
      • Yes; PASS
      • No; See 2.4.1.1.2 Skip link does not move focus to the correct area.
    • No; Is the page sectioned using ARIA landmarks?
      • Yes; PASS
      • No; Does the page include heading elements at the beginning of each section of content?
        • Yes; PASS
        • No; Is the repeated content contained within a collapsible region, such as a menu?
          • Yes; PASS
          • No; See 2.4.1.1.1 No skip link, headings, or landmarks are available to skip content blocks.
  • No; PASS

Web Test Procedure

Desktop

  1. When the page loads, press Tab to navigate into the page.
  2. Check if one of the first focusable items on the page is a skip link.
    1. If so, press Enter to activate the link.
    2. Verify that the focus moves to the main content area of the page.
    3. Check if there are additional skip links provided (e.g., “Skip to search,” “Skip to navigation,” etc.).
    4. If so, verify that these links move the focus to the appropriate area of the page.
  3. Check if there is a semantic heading (generally level 1) at the top of the main content area.
  4. Check if there is a <main> or role="main" landmark area.

Desktop Screen Reader (NVDA)

  1. Open web page in Chrome with NVDA turned on.
  2. Check for skip links:
    1. Press NVDA (Insert)+F7 to display the Elements List.
    2. Select Links.
    3. Check if one of the first few focusable items on the page is a skip link.
    4. If so, verify that the link moves the focus to the main content area.
  3. Check for semantic heading structure:
    1. Press NVDA (Insert)+F7 to display the Elements List.
    2. Select Headings.
    3. Check if there is a heading near the top of the main content that allows you to jump to the main content by selecting that heading.
  4. Check for landmarks:
    1. Press NVDA (Insert)+F7 to display the Elements List.
    2. Select Landmarks.
    3. Check if the page has a main landmark that allows you to jump to the main content.
  5. Verify that one of the above options is available.

Mobile Screen Reader

Mobile content often does not require a method to bypass blocks, because the heading content is condensed to a small number of elements (which may include an expandable hamburger menu).

If a bypass method is required:

  1. Using VoiceOver or TalkBack, check if one of the first few focusable items on the page is a skip link.
  2. Check if there is a heading near the top of the main content that allows you to jump to the main content by selecting that heading.
  3. Check if the page has a main landmark that allows you to jump to the main content.
    • iOS:
      1. Use the Rotor by placing two fingers on the screen and turning, like you are turning a dial.
      2. Switch the Rotor to Landmarks.
      3. Swipe up or down through the page and look for a “main” landmark.
    • Android:
      1. Open the local context menu in Chrome by tapping inside the browser screen and swiping up and then down, or down and then up.
      2. Select Landmarks from the navigation options.
      3. Swipe left or right through the page and look for a “main” landmark.

PDF Testing

Adobe Acrobat Pro

  1. Select Accessibility > Reading Order in the Task pane to open the Reading Order dialog.
  2. Check the Show page content groups checkbox and select the Structure types radio button. This will display the tag value of every tagged object in the Document pane.
  3. Verify that each section of the document begins with a heading of the appropriate hierarchical level, OR
  4. Go through the Tags tree in the Tags pane and verify that each section of content begins with a visible heading tagged with <Hx> tags.

Screen Reader (NVDA or JAWS)

  1. Press the H key to navigate through the headings in the document.
  2. Verify that each section of the document begins with a heading of the appropriate hierarchical level.

Failures

  • 2.4.1.1.1 No skip link, headings, or landmarks are available to skip content blocks. (Medium)
  • 2.4.1.1.2 Skip link does not move focus to the correct area. (Medium)
  • 2.4.1.1.3 The skip link is not visible when focused. (Medium)
