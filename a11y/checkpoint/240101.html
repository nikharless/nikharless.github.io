Overview

Blocks of content do not have to be exactly the same to be considered repetitive. They may differ in detail but contain the same type of information and serve the same purpose. Small items such as repeated individual words, phrases, or single links are not considered to be blocks of content.

Examples of acceptable mechanisms include:

  • Skip links
  • Hide options
  • Collapsible menus
  • Keyboard shortcuts

Note that landmarks and headings are only automatically usable by screen reader users. Keyboard-only users and users of assistive technologies which use the keyboard interface (such as a switch) require in-page links or keyboard shortcuts to navigate to them. (Landmarks and properly coded headings are already required under {{ site.data.1301.1301[1301].name }}.)

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • A keyboard-accessible method must be provided to skip blocks of content that are repeated across pages.
  • The bypass function must not skip content that is not repeated across pages.
  • A skip link to the main content should be the first link on the page.
User Story: As a keyboard-only user, I want to be able to skip blocks of repeated content so that I do not have to navigate through them each time I encounter them.
Success: If you were a keyboard-only user, when blocks of content are repeated across pages, you would be able to skip past them to the content you want.

Testing Logic

Does the page contain blocks of content (such as navigation links, page headers, tabs, and banners) that are repeated across multiple pages?

  • Yes; Does the page provide a keyboard-accessible method to bypass the repeated content?
    • Yes; Is the method visible when in focus (if applicable)?
      • Yes; When activated, does the method move the focus to the correct area?
        • Yes; PASS
        • No; See {{ site.data.2401.2401[24010102].name }}.
      • No; See {{ site.data.2401.2401[24010103].name }}.
    • No; See {{ site.data.2401.2401[24010101].name }}.
  • No; This checkpoint is not applicable.

Web Test Procedure

Desktop

  1. When the page loads, press Tab to navigate into the page. Some skip links are not visible until they receive focus.
  2. Check if there is a keyboard-accessible method to skip repetitive blocks of content.
    1. Use ANDI: focusable elements to check for skip links, hide options, collapse menu options, etc.), OR
    2. Use ANDI: links/buttons and select the "view links list" button.
    3. Check page instructions for any keyboard shortcut functionality.
  3. Use keyboard commands to activate the bypass function.
  4. Verify that the focus moves directly past the repeated content.

Desktop Screen Reader (NVDA)

  1. Open web page in Chrome with NVDA turned on.
  2. Check for skip links:
    1. Press NVDA (Insert)+F7 to display the Elements List.
    2. Select Links.
    3. Check if there is a keyboard-accessible method to skip repetitive blocks of content, such as skip links, keyboard shortcuts, scripted elements, etc.
    4. Use keyboard commands to activate the bypass function.
    5. Verify that the focus moves directly past the repeated content.

Mobile Screen Reader

Mobile content often does not require a method to bypass blocks, because the heading content is condensed to a small number of elements (which may include an expandable hamburger menu).

If a bypass method is required:

  1. Using VoiceOver or TalkBack, check if one of the first few focusable items on the page is a skip link.
  2. Check if there is a method to skip repetitive blocks of content, such as skip links, scripted elements, etc.
  3. Check if the page has a main landmark that allows you to jump to the main content.
    • iOS:
      1. Use the Rotor by placing two fingers on the screen and turning, like you are turning a dial.
      2. Switch the Rotor to Landmarks.
      3. Swipe up or down through the page and look for a “main” landmark.
    • Android:
      1. Open the local context menu in Chrome by tapping inside the browser screen and swiping up and then down, or down and then up.
      2. Select Landmarks from the navigation options.
      3. Swipe left or right through the page and look for a “main” landmark.

PDF Testing

Adobe Acrobat Pro

  1. Select Accessibility > Reading Order in the Task pane to open the Reading Order dialog.
  2. Check the Show page content groups checkbox and select the Structure types radio button. This will display the tag value of every tagged object in the Document pane.
  3. Verify that each section of the document begins with a heading of the appropriate hierarchical level, OR
  4. Go through the Tags tree in the Tags pane and verify that each section of content begins with a visible heading tagged with <Hx> tags.

Screen Reader (NVDA or JAWS)

  1. Press the H key to navigate through the headings in the document.
  2. Verify that each section of the document begins with a heading of the appropriate hierarchical level.
