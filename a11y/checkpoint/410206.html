Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The states of custom user interface components which can be set by the user must be programmatically determinable by assistive technologies.
  • Notifications of changes to states of user interface components must be communicated to assistive technologies.
  • HTML markup must meet W3C specifications and facilitate accessibility support with assistive technologies.
  • ARIA should be used to indicate name, role, and value for tabs, modals, or other dynamic widgets.
User Story: As a screen reader user, I want the state of user interface components to be programmatically determinable so that I can get this information via my screen reader.
Success: If you were a screen reader user, when accessing user interface components, you would be able to determine the state of the components.

Web Test Procedure

  1. Using a screen reader, interact with every custom user interface component in the page.
  2. Verify that the screen reader speaks any properties required for understanding the functionality of the component.
  3. Verify that the screen reader speaks any states that can be set by the user.
  4. Verify that the screen reader notifies the user of any changes to states.
