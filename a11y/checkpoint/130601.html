Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The purpose of user interface components, icons, and regions can be programmatically determined.
User Story: As a non-verbal user who uses symbols to communicate, I want assistive technology to convert elements to familiar symbols so that I can understand their purpose.
Success: If you use assistive technology that converts elements to recognizable symbols, when you view a web page, the elements can be automatically translated.

Overview

This checkpoint adds an additional layer beyond identifying names, roles, and values, by requiring that the purpose of components can be determined by assistive technology. For example:

  • Where {{ site.data.4102.4102[4102].name }} requires that assistive technology can read and understand how to use a control, this checkpoint requires that the specific purpose is also communicated.
  • Where {{ site.data.1305.1305[1305].name }} applies to form inputs, this checkpoint includes non-interactive elements such as icons and landmarks.

Often this checkpoint is already satisfied if semantic HTML elements or ARIA roles are used.

Testing Logic

Does the page contain informational icons?

  • Yes; Does the icon have a text alternative?
    • Yes; Does the text alternative accurately describe the purpose of the icon?
      • Yes; Is the text alternative for each icon consistent across the site?
        • Yes; PASS
        • No; See {{ site.data.1306.1306[13060105].name }}.
      • No; See {{ site.data.1306.1306[13060101].name }}.
    • No; See {{ site.data.1101.1101[1101].name }}.
  • No; This test is not applicable.

Does the page contain form elements?

  • Yes; Do the form elements have a programmatically identifiable purpose?
    • Yes; Is the programmatically identifiable purpose consistent across the site?
      • Yes; PASS
      • No; See {{ site.data.1306.1306[13060105].name }}.
    • No; See {{ site.data.1306.1306[13060102].name }}.
  • No; This test is not applicable.

Does the page contain links?

  • Yes; Can the purpose of each link be programmatically determined?
    • Yes; Is the programmatically determined link purpose consistent across the site?
      • Yes; PASS
      • No; See {{ site.data.1306.1306[13060105].name }}.
    • No; See {{ site.data.1306.1306[13060103].name }}.
  • No; This test is not applicable.

Are all page regions marked up as landmarks?

  • Yes; PASS
  • No; See {{ site.data.1306.1306[13060104].name }}

Web Test Procedure

Desktop

  1. Verify that icons have a programmatically determinable purpose.
    1. Use ANDI: graphics/images to check the ANDI Output of informational icons.
    2. Verify that the ANDI Output provides an accurate description of the purpose of the icon.
    3. Verify that icons with the same purpose are identified consistently.
  2. Verify that user interface components have a programmatically determinable purpose.
    1. Use ANDI: focusable elements to check the ANDI Output of keyboard-focusable elements.
    2. Verify that the ANDI Output provides an accurate description of the purpose of the element.
    3. Verify that components with the same purpose are identified consistently.
  3. Verify that links have a programmatically determinable purpose.
    1. Use ANDI: links/buttons to check the ANDI Output of links.
    2. Verify that the ANDI Output provides an accurate description of the purpose of the link.
    3. Verify that links with the same destination are identified consistently.
  4. Verify that landmark regions are marked up programmatically.
    1. Visually locate the regions of the web page.
    2. Use ANDI: structures and select "landmarks".
    3. Verify that ANDI identifies a landmark region for each visible page region.
  5. Verify that the programmatically determinable purpose for informational icons, user interface components, and links are consistent across the site.
