Overview

When additional content is triggered on pointer hover or on keyboard focus, that additional content can be visually perceived. The additional content must also be dismissible.

Note: If there is additional content that can only be triggered on hover, it should be marked as a failure of {{ site.data.2101.2101[2101].name }}.

Note: Modal dialogs by definition must take keyboard focus and therefore should not appear on hover or focus. See {{ site.data.3201.3201[3201].name }}.

Examples

  • Custom tooltips
  • Sub-menus
  • Non-modal popups

Exceptions

  • Additional content controlled by the user agent, such as browser tooltips
  • The trigger components themselves

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • When additional content is triggered on pointer hover or on keyboard focus, that additional content:
    • Can be dismissed without moving the mouse or keyboard focus
    • Does not disappear when the mouse pointer is moved over it
    • Remains visible until dismissed, hover or focus is removed, or its information is no longer valid
User Story: As a user with a vision impairment, I want to be able to move around the screen without losing the popup so that I don't have to reduce my preferred magnification.
Success: If you had a vision impairment, when you triggered popup content, you would be able to view it without losing it and dismiss it without having to relocate the trigger.

Testing Logic

Does the page have additional content that is triggered by mouse hover or keyboard focus?

  • Yes; Can the additional content be dismissed without moving the mouse hover or keyboard focus?
    • Yes; Can the mouse pointer be moved over the additional content without it disappearing?
      • Yes; Does the additional content remain visible until the mouse hover or keyboard focus is removed, the user dismisses it, or its information is no longer valid?
        • Yes; PASS
        • No; See {{ site.data.1413.1413[14130103].name }}.
      • No; See {{ site.data.1413.1413[14130102].name }}.
    • No; Does the additional content communicate an input error?
      • Yes; PASS
      • No; Does the additional content obscure or replace other content?
        • Yes; See {{ site.data.1413.1413[14130101].name }}
        • No; PASS
  • No; PASS

Web Test Procedure

Desktop

  1. With the page in its default state, locate any additional content that appears on mouse hover or keyboard focus.
  2. Verify that each instance of additional content:
    • Can be dismissed without moving the mouse hover or keyboard focus
    • Can be hovered over without disappearing
    • Remains visible until the mouse hover or keyboard focus is removed
  3. Repeat with any content inside expanded menus, expanded content, modal dialogs, and other dynamic features.
