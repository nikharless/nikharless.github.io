WCAG | 1.3.1 Info and Relationships

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Elements must be used according to their meaning, not their appearance.
  • Error messages should be programmatically associated with the corresponding form field.
  • If a field has additional instructions, they should be programmatically associated with the field.
  • <strong> should be used instead of <b> to indicate importance. <em> should be used instead of <i> to indicate emphasis.
Success: You can understand the semantic roles of elements without being able to see them.

Examples of scenarios that fall under this checkpoint include:

  • Page landmarks
  • Blockquotes
  • Inserted and/or deleted text
  • Relationships conveyed by visual characteristics such as color or shape

Testing Logic

Is the page visually divided into distinct regions such as header, footer, navigation, and main content?

  • Yes; Are the essential regions (header, footer, navigation, and main) marked up as landmarks?
    • Yes; Are the landmarks used according to their semantic meaning?
      • Yes; Are header, main, aside, and footer regions top-level landmarks and not nested within other landmarks?
        • Yes; Is there more than one of a particular type of landmark?
          • Yes; Is the landmark a header, footer, or main content area?
            • Yes; Remove or combine any multiples of these. There must be only one of each.
            • No; Does each instance have a unique label?
              • Yes; PASS
              • No; Give each instance a unique label.
          • No; PASS
        • No; Move header, main, aside, and footer landmarks to the top level beneath the <body> tag.
      • No; Ensure correct and valid tags or ARIA roles are used.
    • No; Is the region a header?
      • Yes; Use <header> or role="banner" to mark the region as a header.
      • No; Is the region a footer?
        • Yes; Use <footer> or role="contentinfo" to mark the region as a footer.
        • No; Is the region a navigation area?
          • Yes; Use <nav> or role="navigation" to mark the region as a navigation area.
          • No; Is the region the main content area?
            • Yes; Use <main> or role="main" to mark the region as the main content area.
            • No; PASS
  • No; PASS

Are there parts of the content that have a semantic function not covered by the other 1.3.1 checkpoints?

  • Yes; Is there semantic markup that corresponds to the function of the content part?
    • Yes; Is the content marked up with the corresponding semantic markup?
      • Yes; PASS
      • No; Mark up the content with the appropriate semantic markup.
    • No; PASS
  • No; PASS

Are visual relationships also conveyed either programmatically or in text? Some examples include: comments visually presented as replies using indentation, blockquotes, blocks of elements linking to the same target, and deleted/inserted text.

  • Yes; PASS
  • No; Is the element's role overridden with role="presentation" or role="none">
    • Yes; Remove the presentation role; add a semantic role if the element does not have one by default.
    • No; Add semantic HTML tags, ARIA roles, or screen-reader-only text to ensure the relationship is understandable to assistive technology users.

Web Test Procedure

  1. Examine the page for information and relationships that are presented visually.
  2. Verify that these are also presented programmatically OR in text.

Mobile App Test Procedure

  • Follow the procedure for Web.

PDF Test Procedure

Adobe Acrobat Pro

  1. Select Accessibility > Accessibility Check from the Task pane toolbar.
  2. Ensure all Checking Options are selected, and activate the Start Checking button.
  3. In the Accessibility Checker results, check for any failures related to untagged content.
  4. Select Tags from the Navigation pane toolbar.
  5. In the Options menu, verify that Document is Tagged PDF is selected.
  6. Go through the Tags tree, and make sure every element is tagged with the appropriate semantic tag.

Screen Reader

  1. Examine the page for information and relationships that are presented visually.
  2. Verify that these are identified correctly by the screen reader.
  3. Verify that the page numbering is consistent.

    4. Failures

    • F1.3.1.a.a Landmark region is not marked up semantically. (Medium)
    • F1.3.1.a.b Quotation is not marked up semantically. (Low)
    • F1.3.1.a.c Important visual relationship is not conveyed programmatically. (Medium)
    • F1.3.1.a.d Semantic markup is used for visual appearance. (Medium)
    • F1.3.1.a.e Semantic markup is incorrect. (Medium)
    • F1.3.1.a.f Semantic information is suppressed with role="presentation" or role="none". (High)
    • F1.3.1.a.g Content with semantic function is not marked up with corresponding semantic markup. (High)
