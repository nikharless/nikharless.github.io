Overview

When page content is presented in <iframe>s, screen reader users can navigate to these in the same way they can navigate to section headings.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • <iframe>s must have a title attribute set to a unique, descriptive name for the content.
User Story: As a keyboard or screen reader user, I want to be able to skip blocks of repeated content so that I do not have to navigate through it each time I encounter it.
Success: If you were a keyboard-only user, when blocks of content are repeated across pages, you would be able to skip past them to the content you want.

Testing Logic

Does the page use <iframe> elements to group blocks of repeated material?

  • Yes; Does each <iframe> have a title attribute?
    • Yes; Does the title describe the <iframe>'s content?
      • Yes; Is the title unique?
        • Yes; PASS
        • No; See {{ site.data.2401.2401[24010203].name }}.
      • No; See {{ site.data.2401.2401[24010202].name }}.
    • No; See {{ site.data.2401.2401[24010201].name }}.
  • No; PASS

Web Test Procedure

Desktop

  1. Run axe DevTools to flag any <iframe> with a missing title attribute.
  2. Verify that every <iframe> on the page has title text that describes its purpose.
  3. Verify that each <iframe> title is unique.

Desktop Screen Reader (NVDA)

  1. Open web page in Chrome with NVDA turned on.
  2. Press the M key and listen for occurrences of <iframe>s.
  3. Press the M key for each <iframe> found.
  4. Verify that the text description provided for each <iframe> properly describes its purpose.

Mobile

  1. Open the page in Chrome on a desktop computer.
  2. Press Control+Shift+I, OR right-click in the page and select Inspect, OR from the 3-vertical-dot menu on the top right, select More tools > Developer tools.
  3. Within Developer Tools, activate Toggle device toolbar to on.
  4. From the Dimensions drop-down, select the appropriate iPhone or Android device.
  5. Follow the procedure for Desktop.
