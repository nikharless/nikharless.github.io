Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Content must not be restricted to portrait or landscape orientation (unless a particular orientation is essential).
    • Content must display according to the orientation (portrait or landscape) of the device.
    • Content must always respect system-level orientation lock settings.
    • Changes in content or functionality due to the size of display are not covered by this criterion which is focused on restrictions of orientation.
User Story: As a user with dexterity impairments who has a mounted device, I want content to be displayed in the orientation of my device so that I can easily view it.
Success: If you could not rotate your device, when you view content, you would see it displayed in the appropriate orientation.

Testing Logic

Does the content open in the current orientation of the device (portrait or landscape)?

  • Yes; When the device is rotated, does the content rotate to accommodate the new orientation?
    • Yes; PASS
    • No; Is it essential for the content to be restricted to a single display orientation?
      • Yes; PASS
      • No; See F1.3.4.a.b The web page or app does not rotate when the orientation changes.
  • No; Is it essential for the content to be restricted to a single display orientation?
    • Yes; PASS
    • No; See F1.3.4.a.a The web page or app does not load in the correct orientation.

Web Test Procedure

Desktop

  1. Open the web page in Chrome.
  2. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the Developer Tools.
  3. Activate the Toggle device toolbar button or press Ctrl+Shift+M.
  4. Select Responsive from the Dimensions drop-down, and set the width to 1366. The page should display in landscape mode.
  5. Activate the Rotate button in the device toolbar.
  6. Verify that the page adjusts to display in portrait orientation.
  7. Activate the Rotate button again, and verify that the page adjusts to display in landscape orientation.

Mobile

  1. Unlock the device rotation to allow the screen to automatically rotate with the device's orientation:
    • Android: Turn on the Auto rotate setting.
    • iOS: Turn off the Portrait Orientation Lock setting.
  2. With the device in portrait orientation, open the web page.
  3. Verify that the web page loads in portrait orientation.
  4. Rotate the device to landscape orientation, and verify that the web page rotates to accommodate it.
  5. Close the web page, and while holding the device in landscape orientation, re-open it.
  6. Verify that the web page loads in landscape orientation.
  7. Rotate the device to portrait orientation, and verify that the web page rotates to accommodate it.

Mobile App Test Procedure

Android or iOS

  1. Unlock the device rotation to allow the screen to automatically rotate with the device's orientation:
    • Android: Turn on the Auto rotate setting.
    • iOS: Turn off the Portrait Orientation Lock setting.
  2. With the device in portrait orientation, open the app.
  3. Verify that the app loads in portrait orientation.
  4. Rotate the device to landscape orientation, and verify that the app rotates to accommodate it.
  5. Close the app, and while holding the device in landscape orientation, re-open it.
  6. Verify that the app loads in landscape orientation.
  7. Rotate the device to portrait orientation, and verify that the app rotates to accommodate it.

Failures

  • 1.3.4.1.1 The web page or app does not load in the correct orientation. (Medium)
  • 1.3.4.1.2 The web page or app does not rotate when the orientation changes. (Medium)
