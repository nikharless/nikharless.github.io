Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The navigation order of focusable elements must be logical and intuitive.
  • Modeless dialogs must appear immediately after the trigger element.
  • Focus must move to modal dialogs when they are opened.
  • Focus must be restricted within modal dialogs while they are open.
  • When a modal is closed, focus must be returned to the trigger element or to the most logical location.
  • Focus order and visual order should match.
User Story: As a keyboard-only user, I want the focus order to be logical so that I can easily navigate the content.
Success: If you were a keyboard user, when you navigate a web page, the focus order would be logical and intuitive.

Testing Logic

Is the navigation order of focusable elements logical and intuitive?

  • Yes; PASS
  • No; Is the focus lost after user interaction or content update?
    • Yes; See F2.4.3.a.b Focus is lost or incorrectly placed during interaction or content update.
    • No; Does the issue involve a modeless dialog?
      • Yes; See F2.4.3.a.c Focus does not go to modeless dialog immediately after the trigger.
      • No; Does the issue involve a modal dialog?
        • Yes; Does the focus move to the modal when it is opened?
          • Yes; Does the focus remain within the modal while it is open?
            • Yes; Does the focus return to the trigger element when it is closed?
              • Yes; See F2.4.3.a.a Focus order does not preserve the meaning or operability of the content.
              • No; See F2.4.3.a.f When modal is closed, focus is not returned to trigger or next logical location.
            • No; See F2.4.3.a.e Focus is not maintained within modal while it is open.
          • No; See F2.4.3.a.d Focus does not move to modal when it is opened.
        • No; Does the focus not follow a scripted visual scroll?
          • Yes; See F2.4.3.a.g Focus does not follow visual scroll.
          • No; Does the focus move to a hidden element?
            • Yes; See F2.4.3.a.h Focus moves to hidden element.
            • No; Does the element use a positive tabindex value?
              • Yes; See F2.4.3.a.i Positive tabindex value causes illogical focus order.
              • No; See F2.4.3.a.a Focus order does not preserve the meaning or operability of the content.

Web Test Procedure

Desktop

  1. Starting at the top of the page, press Tab to navigate through the page.
  2. Verify that focus moves through the page in a logical and predictable manner.
  3. Navigate to interactive elements such as buttons, links, or tabs that show or hide content within the page.
  4. Activate or interact with the elements using Enter, Space bar, or arrow keys, as appropriate.
  5. When content is added or removed, navigate through the content using the Tab or arrow keys as appropriate.
  6. Verify that the focus order preserves meaning and operability.
  7. Repeat the above steps if the interactive content introduces additional interactive functionality.
  8. Activate in-page links or interactions, such as back to top links, skip to main content links, links to and from footnotes, etc.
  9. Verify that when visual focus is moved (i.e., the viewport is shifted), keyboard focus is also moved to the intended target.

Mobile Screen Reader

  1. Turn VoiceOver or TalkBack on.
  2. Starting at the top of the page, swipe to navigate through the page.
  3. Verify that focus moves through the page in a logical and predictable manner.
  4. Interact with elements such as buttons, links, or tabs that show or hide content within the page.
  5. When content is added or removed, verify that the focus order preserves meaning and operability.
  6. Repeat the above steps if the interactive content introduces additional interactive functionality.
  7. Activate in-page links or interactions, such as back to top links, skip to main content links, links to and from footnotes, etc.

Mobile App Test Procedure

Follow Mobile Web procedure.

PDF Testing

  1. Tab through all the active elements in the document (links and form controls).
  2. Verify that the tabbing order through active elements is logical and predictable.

Failures

  • F2.4.3.a.a Focus order does not preserve the meaning or operability of the content. (Medium)
  • F2.4.3.a.b Focus is lost or incorrectly placed during interaction or content update. (Medium)
  • F2.4.3.a.c Focus does not go to modeless dialog immediately after the trigger. (Medium)
  • F2.4.3.a.d Focus does not move to modal when it is opened. (Medium)
  • F2.4.3.a.e Focus is not maintained within modal while it is open. (Medium)
  • F2.4.3.a.f When modal is closed, focus is not returned to trigger or next logical location. (Medium)
  • F2.4.3.a.g Focus does not follow visual scroll. (Medium)
  • F2.4.3.a.h Focus moves to hidden element. (Medium)
  • F2.4.3.a.i Positive tabindex value causes illogical focus order. (Medium)
