Overview

Examples of dragging movement alternatives:

  • Text fields which allow for typed entry of values
  • Sliders (linear or radial) that allow clicking or tapping on the area to select a value
  • Accompanying arrow buttons
  • Pop-up menus with move options

This criterion does not apply to:

  • Content where dragging is essential
  • Functionality is determined by the user agent, not the author
  • The operation of the user agent or assistive technology

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Dragging functionality must also be available using a single pointer.
User Story: As a user with a movement disability, I want to have an alternate to dragging movements so that I can use the functionality.
Success: If you were a user with a movement disability, when you encounter functionality that uses dragging movements, you could interact with it using a single pointer.

Testing Logic

Is there any functionality that uses dragging movements?

  • Yes; Is there an alternate single point means of operation for the same functionality?
    • Yes; PASS
    • No; Is the dragging movement essential for the functionality?
      • Yes; PASS
      • No; See {{ site.data.2507.2507[25070101].name }}.
  • No; PASS

Web Test Procedure

  1. Check if there is any functionality that requires a dragging movement to perform and that is not determined by the user agent (such as the device or the operating system).
  2. If there is, check if there is an alternate way to access the functionality that doesn't require a dragging movement.
  3. If there isn't, verify that the functionality is essential (i.e., would information or functionality change if a dragging motion wasn't used?).

Mobile App Test Procedure

Follow the Web Test procedure.
