Overview

Visual indicators of a control's boundaries must have at least 3:1 contrast with the background.

Note: Visual indicators of the boundaries are not required. This checkpoint only applies if the boundaries are the only way to identify the control.

Exceptions include:

  • User agent default controls
  • Disabled controls
  • Buttons with text or icons
  • Text inputs with text

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Visual indicators of a control's boundaries must have at least 3:1 contrast with the background.
User Story: As a user with low vision, I want controls to have sufficient contrast so that I can distinguish them.
Success: If you had low vision, when you encountered controls, you would be able to distinguish them from surrounding content.

Testing Logic

Does the page include active user interface components which have visual boundaries indicating the hit area?

  • Yes; Are the visual boundaries the only means of distinguishing the control (e.g., no text or icon inside it)?
    • Yes; Does the visual boundary have at least 3:1 contrast with the background?
      • Yes; PASS
      • No; Is the control disabled?
        • Yes; PASS
        • No; Is the control the default appearance for the user agent?
          • Yes; PASS
          • No; See See F1.4.11.a.a Visual boundary of control does not have 3:1 contrast with background.
    • No; PASS
  • No; PASS

Web Test Procedure

Desktop

  1. Locate any active controls that are not the browser default and that do not have text or icons inside them.
  2. Determine if they have a visual boundary that is the only means of distinguishing them from the background.
  3. Use a tool such as one listed in the Testing Tools below to manually check the contrast between the visual boundary and the background.

    Note: Check hex codes rather than using an eyedropper tool whenever possible to ensure you're checking the exact color.

  4. Verify that the boundary has at least 3:1 contrast with the background.

Mobile

  1. Open the page on a desktop computer in Chrome.
  2. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the Developer Tools.
  3. Activate the Toggle device toolbar button or press Ctrl+Shift+M.
  4. Select a mobile device from the Dimensions drop-down menu.
  5. Follow the procedure for Desktop Web.

Mobile App Test Procedure

iOS and Android

  1. Ensure that any contrast enhancements, such as Increase Contrast on iOS, are turned OFF.
  2. Locate any active controls that are not the system default and that do not have text or icons inside them.
  3. Determine if they have a visual boundary that is the only means of distinguishing them from the background.
  4. Visually check for boundaries which appear to have poor contrast with the background.
  5. Take a screenshot on the device and open it on a desktop computer.
  6. Use a manual tool such as one listed in the Testing Tools below to check the contrast ratio of the colors in the screenshot.
  7. Verify that the contrast ratio between the visual boundary and its background is at least 3:1.
  8. If the app supports Dark Mode, test both Light and Dark Mode.
  9. If the app has any alternate themes that affect the control colors, test each of these.

PDF Test Procedure

  1. Locate any active controls that are not the system default and that do not have text or icons inside them.
  2. Determine if they have a visual boundary that is the only means of distinguishing them from the background.
  3. Use a manual tool such as one listed in the Testing Tools below to manually check any visual boundaries.

Failures
