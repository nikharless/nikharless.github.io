Overview

It's not necessary that the reading order matches the visual order unless it affects the meaning. There may be more than one logical reading order, and only one needs to be provided.

Examples:

  • Passages of text, tables, and ordered lists have meaningful sequences that cannot be rearranged.
  • Items in unordered lists do not have a meaningful sequence.
  • Sidebars and main content areas usually do not have to be in a particular order.
  • Headings and associated content have a meaningful sequence.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The reading order of page content must be logical and intuitive.
  • Reading order should match the visual order, but it is not a failure unless the difference changes meaning.
User Story: As a blind screen reader user, I want the content to be read in a meaningful sequence so that I can understand the content.
Success: If you could not see the web page, when you use a screen reader, you would hear the content in an order that makes logical sense.

Web Test Procedure

Desktop

Screen reader testing should be performed in addition, to verify meaningful sequence of dynamic content.

Verify linearized reading order using one of the following:

  1. Use the Web Developer plugin to view the page without CSS:
    1. Open the Web Developer plugin.
    2. Select the CSS tab.
    3. Check the Disable All Styles option.
    4. Verify that the reading order of the non-styled content follows the intended meaning of the content.
  2. Use the ANDI bookmarklet to check reading order and reveal dynamic content:
    1. Activate the ANDI bookmarklet.
    2. Reading Order:
      1. Select structures from the drop-down.
      2. Activate reading order. Web page elements will be numbered according to reading order.
    3. Reveal Hidden Content:
      1. Select hidden content from the drop-down.
      2. Activate reveal all.
      3. Verify that the reading order of the content still follows a logical sequence with the hidden content revealed.

Desktop Screen Reader

  1. Starting at the top of the page, read through the page content with the screen reader.
    • NVDA and JAWS: Use Down Arrow and Up Arrow
    • VoiceOver: Use VO+Down Arrow and Up Arrow
    1. Verify that the reading order of the static content follows the intended meaning of the content.
    2. Verify that all visible text can be read.
    3. Verify that content that is meant to be hidden from all users is hidden from the screen reader.
  2. Starting back at the top of the content being tested, navigate to and activate any controls that add or delete content from the page.
    • Verify that the reading order of the new content is logical in the natural flow of the interaction.
    • Verify that the screen reader does not read outside any modals while they are open.

Mobile Screen Reader

VoiceOver or TalkBack

  1. Starting at the top of the page, read through the page content with the screen reader by swiping left to right.
    1. Verify that the reading order of the static content follows the intended meaning of the content.
    2. Verify that all visible text can be read.
    3. Verify that content that is meant to be hidden from all users is hidden from the screen reader.
  2. Starting back at the top of the content being tested, navigate to and activate any controls that add or delete content from the page.
    • Verify that the reading order of the new content is logical in the natural flow of the interaction.
    • Verify that the screen reader does not read outside any modals while they are open.

Mobile App Test Procedure

iOS or Android

  1. Starting at the top of the page, read through the page content with the screen reader by swiping left to right.
  2. Verify that the reading order of the static content follows the intended meaning of the content.
  3. Verify that all visible text can be read.
  4. Verify that content that is meant to be hidden from all users is hidden from the screen reader.

PDF Test Procedure

Adobe Acrobat Pro

  1. Select All tools > Prepare for accessibility > Fix reading order.
  2. In the Reading Order dialog, check Show page content groups and Page content order. This displays numbers on tagged elements showing the reading order.
  3. Verify that the reading order follows the intended meaning of the content.
