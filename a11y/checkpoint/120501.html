Overview

Visual information such as the following must be communicated to blind and visually impaired users:

  • Actions
  • Characters
  • Visual text
  • Scene changes
  • Speaker identification
  • Location

Audio descriptions are inserted in pauses in dialog. Transcripts or other non-synchronized alternatives do not meet the requirement for this checkpoint.

If the main audio track sufficiently describes all important visual content in the video, including actions, characters, scene changes, onscreen text, speaker identification and location, etc., additional audio descriptions aren't required.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Prerecorded synchronized media must have an audio alternative (audio descriptions) that includes all of the information presented in the visual content.
  • The audio description should be presented next to the video and be easy to find.
  • A text transcript should also be provided for the benefit of deafblind users.
User Story: As a blind user, I want to listen to an audio description along with the video so that I can get all of the same information that is conveyed in the visual portion of the video.
Success: If you could not see the video, when you listen to the audio description you would get all of the information conveyed in the video portion.

Testing Logic

Does the page contain synchronized media (e.g., video with dialogue)?

  • Yes; Does the main audio track sufficiently describe all important visual information?
    • Yes; PASS
    • No; Does the video content in the synchronized media have an audio description?
      • Yes; Does the audio description include all the important actions, visuals, and text in the video?
        • Yes; PASS
        • No; See {{ site.data.1205.1205[12050102].name }}.
      • No; See {{ site.data.1205.1205[12050101].name }}.
  • No; This checkpoint is not applicable.

Web Test Procedure

  1. Locate each synchronized media element on the page.
  2. Determine whether the video content contains any information not provided through the audio content.
  3. If it does, verify that it has an audio description that provides the visual information.
  4. Enable audio descriptions and play the media.
  5. Compare the audio descriptions to the video content and verify that the main audio track along with the audio descriptions describe all important visual details.
  6. Verify that the audio descriptions are synchronized enough to preserve the meaning.
