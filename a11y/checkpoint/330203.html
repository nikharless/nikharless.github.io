Overview

Users need to know what form fields are required. This applies to both visible labels and accessible names.

  • Marking required fields with an asterisk (*) minimally passes. An additional note such as "Fields marked with * are required" should be provided.
  • aria-required="true" may be used to add accessibility support where visual indicators aren't enough.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Required status must be communicated to all users.
User Story: As a user with a cognitive disability, I want form inputs and instructions to have enough information so that I know what is expected.
Success: If you were a user with a cognitive disability, when you encountered required form inputs, labels and instructions would provide enough information for me to know they were required.

Testing Logic

Are there required form inputs on the page?

  • Yes; Do the required form inputs include the required status in the label?
    • Yes; PASS
    • No; Are there form-level instructions indicating which form inputs are required?
      • Yes; PASS
      • No; Do the optional form inputs indicate that they are optional?
        • Yes; PASS
        • No; Is there an inline error message that indicates which fields are required?
          • Yes; PASS
          • No; See {{ site.data.3302.3302[33020301].name }}.
  • No; Not applicable

Web Test Procedure

  1. Locate any form inputs in the content.
  2. Submit the form without entering any input.
  3. For each form input that fails validation due to missing required input, verify that the required status is communicated to the user before the form is submitted.

Mobile App Test Procedure

Follow the procedure for Web. Use a screen reader to verify that the required status is also communicated to assistive technology.

PDF Test Procedure

Follow the procedure for Web.
