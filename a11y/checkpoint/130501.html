Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Form inputs which collect personal data that map to items on W3's Input Purposes for User Interface Components must use the autocomplete attribute set to the matching value on the list.
User Story: As a user with dexterity impairments, I want frequently used personal information automatically filled in for me so that I can complete forms more efficiently.
Success: If you have difficulty entering information, when you view a form with frequently used personal information, it could be filled in automatically for you.

Testing Logic

Does the page contain input fields which collect information about the user that serves a purpose identified in the Input Purposes for user interface components?

  • Yes; Does each field use the autocomplete attribute to identify its purpose?
    • Yes; Does the field use a valid and appropriate autocomplete value?
      • Yes; PASS
      • No; See {{ site.data.1305.1305[13050102].name }}
    • No; See {{ site.data.1305.1305[13050101].name }}
  • No; PASS

    • Web Test Procedure

    Desktop

    1. Locate any form inputs which collect personal data.
    2. Check if any of the inputs match items on W3's Input Purposes for User Interface Components list.
    3. If so, verify that each field uses the autocomplete attribute and that it is set to a valid and appropriate value from the list.

    Mobile

    1. Open the web page in Chrome on a desktop computer.
    2. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the Developer Tools.
    3. Activate the Toggle device toolbar button or press Ctrl+Shift+M.
    4. Select a mobile device from the Dimensions drop-down.
    5. Locate any form inputs which collect personal data.
    6. Check if any of the inputs match items on W3's Input Purposes for User Interface Components list.
    7. If so, verify that each field uses the autocomplete attribute and that it is set to a valid and appropriate value from the list.
