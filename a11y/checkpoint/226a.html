Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • A warning about the length of a time limit that could result in loss of data must be provided.
User Story: As a user with a disability, I want to know how long it will be before my session times out if I am inactive so that I do not lose data unexpectedly.
Success: If you had a disability, when a session has a time out after a period of inactivity, you see a warning ahead of time alerting you of the length of time.

Testing Logic

Is there a session time limit?

  • Yes; Is there a warning notifying users of the length of time they can be inactive before the time limit expires?
    • Yes; PASS
    • No; See F2.2.6.a.a No warning of session time limit length.
  • No; PASS

Web Test Procedure

Desktop

  1. Sign in to the site.
  2. Determine whether there is a session time limit for inactivity.
  3. Verify that there is a warning notifying the user of the length of the time limit.

Mobile

Follow the process for Desktop.

Mobile App Test Procedure

Follow the process for Desktop.

Failures

  • F2.2.6.a.a No warning of session time limit length. (Medium)
