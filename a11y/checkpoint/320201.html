Overview

Examples of state or value changes include:

  • Checkbox checked/unchecked
  • Radio button checked/unchecked
  • Select box option highlighted
  • Menu option highlighted
  • Text entered in text field

This checkpoint does not apply to the results of activating a:

  • Button
  • Link
  • Tab in a tab control

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • When the value or state of a control is changed, it must not result in a change of context unless there is a warning beforehand.
  • A submit button should be provided on forms, even if the form can be implicitly submitted.
User Story: As a screen reader user, I want the context to not change unexpectedly so that I know where I am.
Success: If you were a screen reader user, when you change the state or value of a control, the context would not unexpectedly change.

Testing Logic

Is there a change of context when the state or value of a control is changed?

  • Yes; Is there text notifying the user that this will happen?
    • Yes; PASS
    • No; Is the focus automatically removed or redirected?
      • Yes; See {{ site.data.3202.3202[32020101].name }}.
      • No; Is a new window, tab, or dialog launched?
        • Yes; See {{ site.data.3202.3202[32020102].name }}.
        • No; Is there a significant change of content before the current focus?
          • Yes; See {{ site.data.3202.3202[32020103].name }}.
          • No; Is a form automatically submitted?
            • Yes; See {{ site.data.3202.3202[32020104].name }}.
            • No; See {{ site.data.3202.3202[32020105].name }}.
  • No; PASS

Web Test Procedure

Desktop

  1. Using only the keyboard, change the setting or enter text for each control.
  2. Verify that there is no unexpected change of context when any control state or value changes.

Mobile

  1. Turn on the screen reader.
  2. Change the setting or enter text for each control.
  3. Verify that there is no unexpected change of context when any control state or value changes.

Mobile App Test Procedure

Follow the procedure for Mobile Web.
