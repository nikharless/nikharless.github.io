WCAG | 1.3.1 Info and Relationships

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Data tables must be marked up semantically as tables.
  • Header cells must be programmatically associated with data cells.
    • Empty header cells are not a failure.
  • Tables should not be used for layout purposes.
    • If a table is used for layout, it must be marked up to prevent assistive technologies from reading it as a data table.
    • Layout tables should use role="presentation" or role="none" to prevent assistive technology from trying to read it as a table.
  • Tables with both row and column headers should use the scope attribute to identify what each header is for.
    • The scope attribute is not required for tables with simple row and/or column headers.
  • Complex tables should be broken down into separate simple tables.
  • Tables should use the <caption> element to programmatically associate a visual label with the data table.
Success: You can understand the data in the table without being able to see it.

Testing Logic

Are there data tables on the page?

  • Yes; Are the data tables marked up as tables and read by the screen reader as such?
    • Yes; Are the header cells marked up as such?
      • Yes; Does the table have both row and column headers?
        • Yes; Do the headers have a scope attribute?
          • Yes; Does the scope attribute have the correct value?
            • Yes; PASS
            • No; Correct the value for the scope attribute.
          • No; Add a scope attribute to each header cell, set to the correct value.
        • No; PASS
      • No; Mark up the header cells as header cells.
    • No; Mark up the data tables semantically using the appropriate HTML or ARIA roles.
  • No; PASS

Are there any layout tables on the page semantically marked up as tables?

  • Yes; Are the layout tables read by the screen reader as data tables?
    • Yes; Remove any table HTML, summary, headers, or scope attributes, ARIA table roles, and table-related CSS properties.
    • No; PASS
  • No; PASS

Web Test Procedure

  1. Visually locate any data tables on the page.
  2. Verify that they are marked up as tables.

Screen Reader Testing

NVDA

  1. Cycle through the tables using the T key.
  2. Use the up and down arrow keys to navigate through the cells of the table, or use Control+Alt+ up, down, left, and right arrow keys.

JAWS

  1. Cycle through the tables using the T key.
  2. Use Control+Alt+ up, down, left, and right arrow keys to navigate the table.
  3. Use Control+Alt+5 (on the numeric keypad) to read the row and/or column headers for the current cell.

VoiceOver

  1. Use VO+ up, down, left, and right arrow keys to navigate the table.
  2. Use VO+C to read the column header for the current cell.

Note: VoiceOver does not read row headers, though it will identify the row number.

Mobile App Test Procedure

  • Locate any data tables on the page.
  • Using the screen reader, navigate through the table and determine whether the content is understandable without being able to see it.
  • If the table has interactive content, verify that it can be used with the screen reader turned on.

PDF Test Procedure

PDF Accessibility Checker

  1. Run PAC on the PDF.
  2. Select the WCAG tab and check if 1.3 Adaptable is marked as a failure.
  3. If it is, activate the Results in Detail button.
  4. Check if the following are marked as failures:
    • Table regularity
    • Table header cell assignments

Adobe Acrobat Pro

  1. Select Accessibility > Reading Order from the Task pane.
  2. Make sure the Show page content groups checkbox is checked, and select Structure types.
  3. Right-click on the table and select Table Editor from the context menu.
  4. Verify that header cells in data tables are marked up using <TH> tags.
  5. Right-click each header cell in the Document pane, and select Table Cell Properties from the context menu.
  6. In the Table Cell Properties dialog, verify that the appropriate Scope is set for the header cell.
  7. Verify that the values in the Row Span and Column Span attributes are accurate.

Screen Reader

  1. Press T to navigate through tables in the document.
  2. Use Ctrl+Alt+Up, Down, Right, or Left Arrow keys to navigate inside the table and away from table headers.
  3. Verify that table headers are announced along with the content of the data cells.

Failures

  • F1.3.1.b.a Data table is not marked up as such (High)
  • F1.3.1.b.b Data header cells are not marked up as such (High)
  • F1.3.1.b.c scope attribute value is incorrect (High)
  • F1.3.1.b.d scope attribute value is invalid (High)
  • F1.3.1.b.e scope attribute is used on data cell (Low)
  • F1.3.1.b.f Complex table does not use headers/id association (High)
  • F1.3.1.b.g Complex table has incorrect headers/id association (Medium)
  • F1.3.1.b.h Tables are nested (High)
  • F1.3.1.b.i Table caption is marked up as header (Medium)
  • F1.3.1.b.j Non-tabular data is marked up as a table (Medium)
  • F1.3.1.b.k Screen reader does not read headers with data cells (High)
