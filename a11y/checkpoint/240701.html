Overview

Examples of visible focus include, but are not limited to:

  • A vertical bar is displayed in a text field, or the text in the field is highlighted.
  • A visible border is displayed around the element.
  • The background color is changed.
  • An icon, such as an arrow, is added next to a menu item.
  • A thick line is added under or next to the focused element.
  • A distinctive change is made to the text of the component, such as making it bold or underlined.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • There must be an indicator showing which element is in focus.
User Story: As a keyboard user, I want to see visible focus indicators so that I can tell what element is in focus.
Success: If you were a keyboard-only user, when navigating a website, you could see which element was in focus.

Testing Logic

As you navigate through the page using the keyboard, is it visually apparent which element has focus?

  • Yes; PASS
  • No; Has the focus indicator been disabled using CSS?
    • Yes; See {{ site.data.2407.2407[24070102].name }}.
    • No; Has the focus been removed using a script?
      • Yes; See {{ site.data.2407.2407[24070103].name }}.
      • No; See {{ site.data.2407.2407[24070101].name }}.

Web Test Procedure

Desktop

  1. Using only the keyboard, navigate to and interact with all interactive elements and widgets. This includes focus within controls such as menus, select lists, and date pickers.
  2. Verify that all elements have a visible focus indicator.
  3. Verify that the focus indicator is removed when focus is removed.

Mobile

Follow the desktop procedure using a Bluetooth keyboard.
