Overview

Examples of animation triggered by interaction include:

  • Extra animations when scrolling (i.e., elements moving in and out of view horizontally when user scrolls vertically)
  • Parallax effects when scrolling
  • Loading transitions

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The user must be able to disable any animation triggered by interaction.
User Story: As a user with a vestibular disorder, I want to view content that does not animate on interaction so that it does not trigger headaches and nausea.
Success: If you had a vestibular disorder, when animation is triggered by interactions, you would have the option of turning them off.

Testing Logic

Is there any animation on the page which is triggered by interaction?

  • Yes; Is the animation essential to the functionality or information presented?
    • Yes; PASS
    • No; Is there a mechanism to disable the animation?
      • Yes; PASS
      • No; See F2.3.3.a.a Animation triggered by interaction cannot be disabled.
  • No; PASS

Web Test Procedure

Desktop and Mobile

  1. Locate any non-essential animation that is triggered by user interaction.
  2. Verify that there is a mechanism to disable the animation.

Failures

  • F2.3.3.a.a Animation triggered by interaction cannot be disabled. (Medium)
