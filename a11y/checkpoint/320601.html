Overview

Help mechanisms may include:

  • Link to Contact page
  • Phone number
  • Email address
  • Physical address
  • Hours of operation
  • Messaging system
  • Chat client
  • Social media links
  • FAQ
  • Link to Support page
  • Chatbot

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Help mechanisms which appear on multiple web pages must appear in the same relative order on each page.
User Story: As a user with a disability, I want the help mechanisms to be in the same place each time they appear so that I don't have to rediscover them.
Success: If you were a user with a disability, when help mechanisms appear on multiple web pages in a set, they appear in the same relative position each time.

Testing Logic

Are there help mechanisms that are repeated across multiple web pages in a set of pages?

  • Yes; Do they appear in the same relative order each time they appear?
    • Yes; PASS
    • No; Has the zoom level, orientation, or viewport size been changed by the user between pages?
      • Yes; PASS
      • No; See {{ site.data.3206.3206[32060101].name }}.
  • No; PASS

Web Test Procedure

  1. Review the web pages within a site, a logical section of a site, or a web application. Do not change zoom level, orientation, or viewport size between pages.
  2. Look for help mechanisms such as human contact details, human contact mechanisms, self-help options, and automated contact mechanisms that are repeated on multiple pages.
  3. Verify that the help mechanisms are in the same relative order on each page.

Mobile App Test Procedure

Follow the procedure for Web.

PDF Test Procedure

  1. Review the pages within the document.
  2. Look for help mechanisms that are repeated on multiple pages.
  3. Verify that they are in the same reading order on each page that they appear.
