Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The language of text that is in a different language than the default page language must be marked up with the correct language code.
User Story: As a screen reader user, I want the language of text set so that I my screen reader knows how to pronounce the page content.
Success: If you were a screen reader user, when you read the page, the screen reader would pronounce the content correctly.

Testing Logic

Is there any text on the page that is in a different language from the default page language?

  • Yes; Is the text a proper name, technical term, in an indeterminate language, or a phrase that has become part of the vernacular of the default language?
    • Yes; PASS
    • No; Does the element containing only the alternate language text have a lang attribute?
      • Yes; Is the value of the lang attribute valid?
        • Yes; Is the value of the lang attribute set to the correct language?
          • Yes; PASS
          • No; See F3.1.2.a.c The language code for foreign text is incorrect.
        • No; See F3.1.2.a.b The language code for foreign text is invalid.
      • No; See F3.1.2.a.a The language of foreign text is not set.
  • No; PASS

Web Test Procedure

Desktop

  1. Read the text of the page to identify any changes of language in the page which are not proper names, technical terms, words of indeterminate language, or words or phrases that have become part of the vernacular of the default language.
  2. Inspect the code to ensure that any changes in language within the document are properly marked with a lang attribute set to a valid value.
  3. Verify that the lang attribute value matches the actual language of the text.

Mobile

  1. Open the page on a desktop computer in Chrome.
  2. Open the Developer tools:
    • Right-click the control and select Inspect, OR
    • From the Customize and control Google Chrome menu, select More tools > Developer tools, OR
    • Press Ctrl+Shift+I
  3. Activate Toggle device toolbar (Ctrl+Shift+M) to view the page as a mobile device.
  4. Select a device from the Dimensions drop-down.
  5. Follow the procedure for Desktop Web.

PDF Test Procedure

Adobe Acrobat Pro

  1. Identify passages of text whose language differs from the default document language.
  2. For each instance, locate and select the corresponding tag in the Tags tree.
  3. Right-click the tag or activate the Options menu, and select Propertiesâ€¦.
  4. In the Object Properties modal, select the Tag tab.
  5. Verify that the correct language is selected from the Language field.

Failures

  • F3.1.2.a.a The language of foreign text is not set. (Medium)
  • F3.1.2.a.b The language code for foreign text is invalid. (Medium)
  • F3.1.2.a.c The language code for foreign text is incorrect. (Medium)
  • F3.1.2.a.d The language for foreign text in the PDF file is not set. (Medium)
