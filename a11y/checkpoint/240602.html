Overview

  • Labels may be either text or images.
  • An error message is not sufficient to communicate the expected format.
  • Dynamic changes to form labels must be included.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Labels and instructions must describe the purpose of the associated form elements.
  • A form element's label and programmatic name should be the same. (See {{ site.data.2503.2503[2503].name }} for requirement.)
User Story: As a user with a disability, I want labels and instructions to be descriptive so that I can easily understand how to use the controls.
Success: If you were a user with a disability, when using a website, you could understand the purpose of controls by their labels.

Testing Logic

Does the page contain form elements?

  • Yes; Are all form elements disabled?
    • Yes; This checkpoint is not applicable.
    • No; Do the form elements have visual labels or instructions?
      • Yes; Do the labels or instructions clearly describe the purpose and applicable data requirements of the associated form elements?
        • Yes; PASS
        • No; See {{ site.data.2406.2406[24060201].name }}.
      • No; This checkpoint is not applicable.
  • No; This checkpoint is not applicable.

Web Test Procedure

For each visible label or instruction on the page, verify that it clearly describes the purpose and data requirements of the associated form element.
