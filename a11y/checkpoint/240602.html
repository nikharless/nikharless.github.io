Overview

Note: A label is available to all users, as opposed to the programmatic name which may only be available to assistive technology.

An icon with an appropriate text alternative may serve as a label. Some common examples are:

  • Magnifying glass for search
  • 3 horizontal lines on top of each other for a menu
  • Gear icon for preferences or settings
  • Trash can for delete

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Labels must describe the purpose of the associated control(s) or input(s).
  • A control's label and programmatic name should be the same. (See {{ site.data.2503.2503[2503].name }}.)
User Story: As a user with a disability, I want labels to be descriptive so that I can easily understand how to use the controls.
Success: If you were a user with a disability, when using a website, you could understand the purpose of controls by their labels.

Testing Logic

Does the page contain control or input labels?

  • Yes; Do the labels clearly describe the purpose of the associated controls or inputs?
    • Yes; PASS
    • No; See {{ site.data.2406.2406[24060201].name }}.
  • No; PASS

Web Test Procedure

For each label on the page, verify that it clearly describes the purpose of the associated control or input.

PDF Test Procedure

Follow the same procedure as web.
