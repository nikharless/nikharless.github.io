Overview

This applies to interruptions such as:

  • Page refreshes
  • Content refreshes
  • Status messages
  • Non-emergency alerts

Emergencies include:

  • Alerts regarding dangers to health and safety
  • Alerts regarding data loss

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • There must be a mechanism to turn off or postpone interruptions unless they are an emergency.
User Story: As a user with a disability, I want to interact with the page without interruptions so that I can focus on the content without becoming disoriented.
Success: If you had a visual or cognitive disability, when you use a web page, your task is not interrupted.

Testing Logic

Does the page automatically refresh with updated content?

  • Yes; Does the updated content involve an emergency?
    • Yes; PASS
    • No; Is there a mechanism to suppress or postpone non-emergency updates?
      • Yes; PASS
      • No; See {{ site.data.2204.2204[22040101].name }}.
  • No; Does the page interrupt the user with notifications or alerts?
    • Yes; Does the alert involve an emergency?
      • Yes; PASS
      • No; Is there a mechanism to suppress or postpone non-emergency alerts and notifications?
        • Yes; PASS
        • No; See {{ site.data.2204.2204[22040102].name }}.
    • No; PASS

Web Test Procedure

Desktop

  1. Identify any interruptions such as content refreshing, notifications, updates, and alerts.
  2. Verify that there is a mechanism for the user to suppress or postpone them.

Mobile

Follow the process for Desktop.

Mobile App Test Procedure

Follow the process for Desktop.
