Basic Requirements and Best Practices

= Requirement
= Best Practice

  • If a control is labeled with only an image, the image must have alternative text which describes the function or destination of the control.
  • If a control is labeled with both an image and text:
    • If the image provides information not conveyed by the associated text, the image's alternative text along with the visible text must together describe the function or destination of the control.
    • If the image does not provide any information not conveyed by the associated text, check against 1.1.1.d Decorative Images (Text Alternative).
  • If the image contains meaningful text, the text alternative should contain that exact text.
  • Text alternatives for linked images should not duplicate adjacent link text for the same destination. Combine the image and text into a single link and check the image against 1.1.1.d Alternative Text (Decorative Images).
  • Text alternatives for functional images should not include words such as "link" or "button".
  • Trademarks, copyright symbols, etc. on an image should be included in the text alternative.
  • Text alternatives should be as succinct as possible.
Success: If you could not see the image, you would still understand the destination, purpose, or function of the control.

Text alternatives for functional images include the following:

  • alt attribute value
  • title attribute value
  • aria-label property
  • aria-labelledby property

Testing Logic

Are there images on the page which function as interactive controls?

  • Yes; Does the image have a text alternative?
    • Yes; Does the text alternative repeat the text that it is programmatically associated with?
      • Yes; See 1.1.1.d Decorative Images (Text Alternative).
      • No; Is the text alternative necessary to understand the function of the control?
        • Yes; Does the text alternative sufficiently describe the function of the control?
          • Yes; PASS
          • No; See F1.1.1.a.d Existing text alternative does not serve the same purpose as the functional image.
        • No; See 1.1.1.d Decorative Images (Text Alternative).
    • No; Is a text alternative necessary to understand the control's purpose?
      • Yes; Is the image in an <img> element?
        • Yes; See F1.1.1.a.a Functional <img> is missing an alt attribute.
        • No; Is the image in an <svg> element?
          • Yes; See F1.1.1.a.b Functional <svg> is missing a title element.
          • No; Is the image in an <input type="image"> element?
            • Yes; See F1.1.1.a.c <input> image is missing an alt attribute..
            • No; Is the image a CSS background element for an interactive element?
              • Yes; See F1.1.1.a.d CSS background image in interactive element is missing a text alternative.
              • No; Add a text alternative.
      • No; PASS
  • No; PASS

Is there an image map on the page?

  • Yes; Is it a client-side image map with an <area> tag?
    • Yes; Does the <area> tag have a descriptive text alternative?
      • Yes; PASS
      • No; See F1.1.1.a.e Image map <area> element is missing a text alternative.
    • No; Is there an alternative to the image map, such as a list of links?
      • Yes; Does the image map have a short text alternative summarizing its purpose?
        • Yes; PASS
        • No; See 1.1.1.c Complex Images (Text Alternative)
      • No; See F1.1.1.a.f Server-side image map is missing a text alternative.
  • No; PASS

Are there functional images which change dynamically (e.g., a button that changes from Play to Pause)?

  • Yes; Does the text alternative change to match the current functional image?
    • Yes; PASS
    • No; See F1.1.1.b.i Text alternative does not match dynamic image.
  • No; PASS

Web Test Procedure

Desktop

  1. Run axe DevTools or WAVE to flag missing text alternatives.
  2. Locate each functional image on the page using one or more of the following methods:
    • Mouse: Hover over images, noting cursor change if they are functional.
    • Keyboard: Use the Tab key to focus functional images.
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  3. Verify that each functional image has a text alternative which accurately describes its purpose.

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to each functional image.
  3. Verify that the name spoken by the screen reader accurately describes the purpose of the functional image.

Mobile App Test Procedure

  • Follow the procedure for Web Mobile.
  • iOS: Verify that the button, button image, or link image trait is spoken, AND/OR hint text such as "Double tap to activate" is spoken.
  • Android: Verify that the button role is spoken, AND/OR hint text such as "Double tap to activate" is spoken.

PDF Test Procedure

Find missing text alternatives.

PDF Accessibility Checker

  1. Run PAC on the PDF.
  2. Select the WCAG tab and check if 1.3 Adaptable is marked as a failure.
  3. If it is, activate the Results in Detail button.
  4. Check if Alternative text for "Figure" structure elements is marked as a failure.

Adobe Acrobat Pro

  1. To find missing text alternatives, select Accessibility > Accessibility Check in the Task pane to open the Accessibility Checker Options dialog.
  2. Select Alternate Text and Headings from the Category drop-down, and make sure Figures require alternate text is checked.

Verify the text alternatives are sufficient.

  1. Locate each functional image in the PDF using one or more of the following methods:
    • Mouse: Hover over images, noting cursor change if they are functional.
    • Keyboard: Use the Tab key to focus functional images.
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  2. Verify that each functional image has a text alternative which accurately describes its purpose.

Failures

  • F1.1.1.a.a Functional <img> is missing an alt attribute. (High)
  • F1.1.1.a.b Functional <svg> is missing a title element. (High)
  • F1.1.1.a.c <input> image is missing an alt attribute. (High)
  • F1.1.1.a.d CSS background image in interactive element is missing a text alternative. (High)
  • F1.1.1.a.e Image map <area> element is missing a text alternative. (High)
  • F1.1.1.a.f Server-side image map is missing a text alternative. (High)
  • F1.1.1.a.g Existing text alternative does not serve the same purpose as the functional image. (Medium)
  • F1.1.1.a.h Existing text alternative does not include informational image text. (Medium)
  • F1.1.1.a.i Text alternative does not match dynamic image. (Medium)
  • F1.1.1.a.j PDF image control is missing Alternate Text. (High)
  • F1.1.1.a.h Articulate Storyline functional image is missing Alternate text. (High)
