Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Previously entered information that is required to be entered again in the same process is auto-populated or available for selection.
User Story: As a user with a motor impairment, I want previously entered information to be auto-populated or selectable so that I don't have to manually enter it again.
Success: If you were a user with a motor impairment, when having to re-enter previously entered information, the information would be auto-populated or available for selection.

Testing Logic

Is the same information required to be re-entered by the user in the same process?

  • Yes; Is the previously entered or provided information auto-populated?
    • Yes; PASS
    • No; Is the previously entered or provided information available for selection?
      • Yes; PASS
      • No; Is re-entering the information essential (e.g., a memory game)?
        • Yes; Not applicable
        • No; Is the information required to ensure the security of the content (e.g., passwords)?
          • Yes; Not applicable
          • No; Is the previous information still valid?
            • Yes; See {{ site.data.3307.3307[33070101].name }}.
            • No; Not applicable
  • No; Not applicable

Web Test Procedure

  1. Locate any instances where previously entered or provided information must be entered again.
  2. Verify that there is a way to fill in the repeated information without typing it again.

Mobile App Test Procedure

Follow the procedure for Web.
