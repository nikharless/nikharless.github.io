Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Visual indicators of a control's boundaries must have at least 3:1 contrast with the background.
  • User controls should have visual boundaries to distinguish the hit area.
User Story: As a user with low vision, I want controls to have sufficient contrast so that I can distinguish them.
Success: If you had low vision, when you encountered controls, you would be able to distinguish them from surrounding content.

Overview

Visual indicators of a control's boundaries must have at least 3:1 contrast with the background. This includes both internal and external boundaries. For example, a slider button must have sufficient contrast with the slider bar, and the slider bar must have sufficient contrast with the page background.

Note: Visual indicators of the boundaries are not required. This checkpoint only applies if the boundaries are the only way to identify the control.

Exceptions include:

  • User agent default controls: These colors are controlled via browser or system settings.
  • Disabled controls
  • Buttons with text or icons: The text or icon must have
  • Text inputs with text inside them

Things to Check

The following are examples of things that fall under this criterion. This list does not cover all possible cases.

Border against background

  • The checkbox border (gray in this example) must have 3:1 contrast with the page background./li>
  • The radio button border (gray in this example) must have 3:1 contrast with the page background.
  • The text box border (gray in this example) must have 3:1 contrast with the page background.
  • The switch border (gray in this example) must have 3:1 contrast with the page background.
  • The switch toggle border (white in this example) must have 3:1 contrast with the switch fill (blue).
  • The slider button border (white in this example) must have 3:1 contrast with both colors of the slider fill. It doesn't need to have contrast with the page background, as this is not necessary to distinguish it.

Fill against background

  • The checkbox fill in this example must have 3:1 contrast with the background, since there is no border.
  • The radio button fill in this example must have 3:1 contrast with the background, since there is no border.
  • The text box fill in this example must have 3:1 contrast with the background, since there is no border.
  • If there is both a fill and a border, the border does not have to have 3:1 contrast with both the fill and the background. In this example, the contrast between the fill and the background is enough, and the border can be ignored.
  • The switch fill (blue in this example) must have 3:1 contrast with the page background, since it has no border.
  • Both colors of the slider fill (blue and gray in this example) must have 3:1 contrast with the page background, since it has no border.

Checked indicator against fill

  • The checkmark (white in this example) must have 3:1 contrast with the checkbox fill (blue).
  • The radio button checked indicator (white in this example) must have 3:1 contrast with the radio button fill (blue).
  • The switch toggle (round) fill (white in this example) must have 3:1 contrast with the switch fill (blue).

Icon as functionality indicator

  • The arrow indicates that this is a drop-down, so it must have 3:1 contrast with its background.
  • If an icon is the only visible label on a button, the icon itself is the indicator, so it must have 3:1 contrast with the button background. The border can be ignored because it's not necessary to distinguish the control.

Web Test Procedure

Desktop

  1. Load the page so that user interface controls are in their default state.
  2. Locate any active controls that are not in the browser default style and that do not have text or icons inside them.
  3. Identify the visual indicators of the control that are required to identify that the control exists.
  4. Determine if the visual indicator is the only means of distinguishing it from the background.
  5. Use a tool such as one listed in the Testing Tools below to check the contrast between the visual indicator and the background.

    Note: Check hex codes rather than using an eyedropper tool whenever possible to ensure you're checking the exact color.

  6. Verify that the visual indicator has at least 3:1 contrast with the background.

Mobile

  1. Open the page on a desktop computer in Chrome.
  2. Right-click on the page and select Inspect or press Ctrl+Shift+I to open the Developer Tools.
  3. Activate the Toggle device toolbar button or press Ctrl+Shift+M.
  4. Select a mobile device from the Dimensions drop-down menu.
  5. Follow the procedure for Desktop Web.

Mobile App Test Procedure

iOS and Android

  1. Ensure that any contrast enhancements, such as Increase Contrast on iOS, are turned OFF.
  2. Locate any active controls that are not the system default and that do not have text or icons inside them.
  3. Determine if they have a visual boundary that is the only means of distinguishing them from the background.
  4. Visually check for boundaries which appear to have poor contrast with the background.
  5. Take a screenshot on the device and open it on a desktop computer.
  6. Use a manual tool such as one listed in the Testing Tools below to check the contrast ratio of the colors in the screenshot.
  7. Verify that the contrast ratio between the visual boundary and its background is at least 3:1.
  8. If the app supports Dark Mode, test both Light and Dark Mode.
  9. If the app has any alternate themes that affect the control colors, test each of these.

PDF Test Procedure

  1. Locate any active controls that are not the system default and that do not have text or icons inside them.
  2. Determine if they have a visual boundary that is the only means of distinguishing them from the background.
  3. Use a manual tool such as one listed in the Testing Tools below to manually check any visual boundaries.
