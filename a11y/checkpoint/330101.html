Overview

This checkpoint checks that any errors that are detected by the web page are communicated to users in visible text.

This checkpoint does not check for the existence of error checking. If a form is submitted unsuccessfully and the page simply redisplays the form with no feedback, this implies that error checking is occurring and an error message is required.

The way the error message is displayed may vary according to what makes sense with the particular form. It may be a list of errors at the top of the page, inline error messages next to the fields in error, alert dialogs, toasts, etc.

Errors may be indicated in other ways in addition to text, such as with color or images.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Error feedback must identify the item with the error with visible text.
  • Error message must sufficiently describe the error.
  • If an error is automatically corrected, the error must still be identified and described with visible text.
  • If errors are provided in a list at the top of the page, they should include links to the corresponding fields.
User Story: As a colorblind user, I want text-based error messages describing the error and the item with the error so that I know an error exists and what it is.
Success: If you were a colorblind user, when an error was automatically detected, the item with the error would be identified and the error described in text.

Web Test Procedure

Desktop

  1. Use ANDI to locate any forms in the content.
  2. Find all instructions and cues (textual and graphical) that are related to form components (e.g., groupings, order of completion, special conditions, qualifiers, format instructions, etc.).
  3. Deliberately trigger errors by entering user input that falls outside the required format or values or leaving required fields blank, and submit the form.
  4. If an input error is automatically detected, verify that both of the following are true:
    • The item that is in error is identified in text (e.g., "Error: Password field").
    • The error is described to the user in text (e.g., "The Password you entered is incorrect.").

Mobile

Using VoiceOver or TalkBack, follow the Desktop Web procedure.
