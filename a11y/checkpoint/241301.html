Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Focus indicators must be at least as large as the area of a 2 CSS pixel thick perimeter of the unfocused element.
  • Focus indicators must have a contrast ratio of at least 3:1 between focused and unfocused states.
  • The focus indicator should be a solid outline around the element.
User Story: As a sighted keyboard user, I want the focus indictor to be easy to see so that I can see where I'm at on the page.
Success: If you were a sighted keyboard user, when you navigate through a page, you could easily see what element was in focus.

Testing Logic

As you navigate through the page using the keyboard, is it visually apparent which element has focus?

  • Yes; Is the area of the focus indicator at least as large as the area of a 2 CSS pixel thick perimeter around the element?
    • Yes; Does the focus state have a contrast ratio of at least 3:1 with the unfocused state?
      • Yes; PASS
      • No; See 2.4.13.1.2 Element does not have at least 3:1 contrast between focused and unfocused states.
    • No; See 2.4.13.1.1 Focus indicator is too small.
  • No; See 2.4.7 Focus Visible.

Web Test Procedure

  1. Using only the keyboard, navigate to and interact with all interactive elements and widgets.
  2. Check whether the focus indicator solidly encloses the focused element.
  3. If it doesn‚Äôt, check whether the area of the focus indicator is at least as large as a 2-pixel-thick perimeter of the unfocused element.
  4. If it isn't, check whether any pixels that change have a contrast ratio of at least 3:1 between focused and unfocused states.

Failures

  • 2.4.13.1.1 Focus indicator is too small. (Medium)
  • 2.4.13.1.2 Element does not have at least 3:1 contrast between focused and unfocused states. (Medium)
