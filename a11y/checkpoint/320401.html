Overview

Consistent doesn't have to mean identical. An arrow icon might be named "Go to page 2" on one page and "Go to page 3" on another. This is consistent.

Components may include:

  • Functional icons
  • Informational icons
  • Form components
  • User interface controls
  • Links
  • Objects

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The accessible name and description must be consistent for components that perform the same function within a set of web pages.
  • The accessible name and description should be consistent for components that perform the same function within a single web page.
User Story: As a user with a disability, I want the components with the same function to be named consistently between web pages so that I can easily identify them.
Success: If you were a user with a disability, when components with the same function appear on multiple web pages in a set, they are named consistently each time.

Testing Logic

Are there components with the same function that are repeated across multiple web pages in a set of pages?

  • Yes; Is the accessible name and description consistent each time they appear?
    • Yes; PASS
    • No; Are they form components?
      • Yes; See {{ site.data.3204.3204[32040101].name }}.
      • No; Are they links?
        • Yes; See {{ site.data.3204.3204[32040102].name }}.
        • No; Are they icons?
          • Yes; See {{ site.data.3204.3204[32040103].name }}.
          • No; See {{ site.data.3204.3204[32040104].name }}.
  • No; This checkpoint is not applicable.

Web Test Procedure

  1. Locate any components with labels or accessible names such as links, buttons, active icons, informational icons, objects, etc., that are repeated across web pages.
  2. Use ANDI: focusable elements and examine the ANDI Output for any interactive components.
  3. Use ANDI: graphics/images and examine the ANDI Output for any informational images or icons.
  4. Verify that each component that shares the same function and is repeated across multiple pages is identified using a consistent label, name, or text alternative.

Mobile App Test Procedure

Note: Native software apps are exempt from Section 508 requirements for this criterion.

  1. Locate any user interface components such as buttons, pickers, switches, steppers, etc. that are repeated across screens.
  2. Verify that each component that shares the same function and is repeated across multiple screens is identified using a consistent label, name, or text alternative.

PDF Test Procedure

  1. Locate any user interface elements such as links, buttons, active icons, informational icons, objects, etc. that are repeated across pages of the document.
  2. Verify that each component that shares the same function and is repeated across multiple pages is identified using a consistent label, name, or text alternative.
