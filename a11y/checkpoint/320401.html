Overview

Consistent doesn't have to mean identical. An arrow icon might be named "Go to page 2" on one page and "Go to page 3" on another. This is consistent.

Components may include:

  • Active icons
  • Informational icons
  • User interface controls
  • Links
  • Objects

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Components that have the same functionality within a set of web pages must be identified consistently.
  • Components that have the same functionality within the same web page should be identified consistently.
User Story: As a user with a disability, I want the components with the same function to be named consistently between web pages so that I can easily identify them.
Success: If you were a user with a disability, when components with the same function appear on multiple web pages in a set, they are named consistently each time.

Testing Logic

Are there components with the same function that are repeated across multiple web pages in a set of pages?

  • Yes; Are they identified consistently each time they appear?
    • Yes; PASS
    • No; Are they form controls?
      • Yes; See 3.2.4.1.1 Form control with same function is named or labeled differently between pages.
      • No; Are they links?
        • Yes; See 3.2.4.1.2 Link with same target is named or labeled differently between pages.
        • No; Are they icons?
          • Yes; See 3.2.4.1.3 Icon with same function is named or labeled differently between pages.
          • No; See 3.2.4.1.4 Component with same function is named or labeled differently between pages.
  • No; PASS

Web Test Procedure

  1. Review the web pages within a site, a logical section of a site, or a web application.
  2. Locate any components such as links, buttons, active icons, informational icons, objects, etc. that are repeated across web pages.
  3. Verify that each component that shares the same function and is repeated across multiple pages is identified using a consistent label, name, or text alternative.

Mobile App Test Procedure

Note: Native software apps are exempt from Section 508 requirements for this criterion.

  1. Locate any user interface components such as buttons, pickers, switches, steppers, etc. that are repeated across screens.
  2. Verify that each component that shares the same function and is repeated across multiple screens is identified using a consistent label, name, or text alternative.

PDF Test Procedure

  1. Locate any user interface elements such as links, buttons, active icons, informational icons, objects, etc. that are repeated across pages of the document.
  2. Verify that each component that shares the same function and is repeated across multiple pages is identified using a consistent label, name, or text alternative.

Failures

  • 3.2.4.1.1 Form control with same function is named or labeled differently between pages.
  • 3.2.4.1.2 Link with same target is named or labeled differently between pages.
  • 3.2.4.1.3 Icon with same purpose is named differently between pages.
  • 3.2.4.1.4 Component with same function is named or labeled differently between pages.
