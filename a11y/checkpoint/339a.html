Overview

Examples of cognitive function tests:

  • Remembering a site-specific password
  • Remembering a touch screen gesture
  • Transcribing a website-provided word
  • Recognizing an image the website provided
  • Solving math problems
  • Text-based personal content

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • A cognitive function test must not be required for authentication unless an alternative or assistive mechanism is provided.
  • When entering a password is essential (such as creating one to be saved by a password manager), it should be possible to show the password.
User Story: As a user with a cognitive impairment, I want to be able to authenticate without a cognitive function test so that I can log in without excessive cognitive effort.
Success: If you were a user with a cognitive impairment, when logging in, a cognitive function test would not be required.

Testing Logic

Is a cognitive function test (e.g., remembering a password or solving a puzzle) required for any step in an authentication process?

  • Yes; Is there an alternative authentication method that does not require a cognitive function test (e.g., one-use codes, biometrics, etc.)?
    • Yes; PASS
    • No; Is there a mechanism available to assist the user in completing the cognitive function test (e.g., support for password managers, copy and paste, etc.)?
      • Yes; PASS
      • No; See F3.3.9.a.a A cognitive function test is required and no alternative or assistance is provided.
  • No; Not applicable

Web Test Procedure

  1. For each authentication method, check if a cognitive function test is required.
  2. If so, verify that there is an alternative authentication method or a mechanism to assist the user in completing it.
  3. Test each step of any multi-factor authentication.

Failures

  • F3.3.9.a.a A cognitive function test is required and no alternative or assistance is provided.
