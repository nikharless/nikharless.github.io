Overview

This checkpoint applies to any headings, whether or not they are marked up semantically.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Each heading must describe the topic of its content.
  • Headings should be used to divide and label content.
User Story: As a screen reader user, I want headings to be descriptive so that I can use them to navigate the content.
Success: If you were a screen reader user, when navigating a website, you could identify the sections of content by their headings.

Testing Logic

Does the page contain visual headings?

  • Yes; Do the headings describe the topic of the associated blocks of content?
    • Yes; PASS
    • No; See {{ site.data.2406.2406[24060101].name }}.
  • No; This checkpoint is not applicable.

Web Test Procedure

  1. Identify all visual headings on the page.
  2. Use ANDI: structures to identify all programmatically defined headings.
  3. For each visual heading on the page, verify that it clearly describes the content of its section.
