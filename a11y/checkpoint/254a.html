Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Functionality that can be triggered by moving or gesturing towards a device must have an alternate means of operation.
  • Functionality that can be triggered by moving or gesturing towards a device must have a means of disabling the motion response.
User Story: As a user with a movement disability, I want an alternate means of using the functionality so that I am able to use it.
Success: If you were a user with a movement disability, when you encounter functionality that is triggered by movement, you could turn it off.

Testing Logic

Is there any functionality that uses device motion or user motion?

  • Yes; Is the motion essential for the functionality?
    • Yes; PASS
    • No; Are there alternate accessible user interface controls for the same functionality?
      • Yes; Can the motion actuation be turned off?
        • Yes; PASS
        • No; See F2.5.4.a.b Motion actuation cannot be disabled.
      • No; See F2.5.4.a.a Motion actuation has no alternative user interface controls.
  • No; PASS

Web Test Procedure

Mobile

  1. Because motion actuation might not be obvious, ask the developer if any motion actuation functionality was implemented.
  2. Identify any functionality that can be triggered by:
    • Device motion such as shaking or tilting, OR
    • User motion detected by the device.
  3. Verify that both of the following are true:
    • Motion actuation can be disabled, AND
    • The functionality can be operated without using motion.

Mobile App Test Procedure

Follow the Mobile Web test procedure.

Failures

  • F2.5.4.a.a Motion actuation has no alternative user interface controls. (Medium)
  • F2.5.4.a.b Motion actuation cannot be disabled. (Medium)
