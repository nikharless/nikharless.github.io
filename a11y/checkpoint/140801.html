Overview

Authors can make color choices that are sufficient for most users, but some individuals require color combinations that don't fit into the general guidelines. In these cases, the user may set colors in the browser, using an extension, or using a replacement stylesheet.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Foreground and background colors must be adjustable by the user.
  • Page regions that are visually distinguished should remain distinguishable when the colors are overridden.
User Story: As a user with a visual or cognitive disability, I want to be able to adjust the colors on a web page so that I can more easily read the content.
Success: If you had low vision, when you encountered text that you found difficult to read, you would be able to adjust the foreground and background colors to suit your specific needs.

Testing Logic

Is the page content styled with non-default colors?

  • Yes; Can the foreground and background colors both be overridden with user-specified colors from the user agent?
    • Yes; PASS
    • No; Is there a mechanism on the web page that allows the user to change the colors?
      • Yes; PASS
      • No; See {{ site.data.1408.1408[14080101].name }}
  • No; PASS

Web Test Procedure

Desktop

  1. In Firefox Settings, under Colors, activate Manage Colors....
  2. In the Colors dialog, select colors that are different from the page, uncheck Use system colors, and select Always for "Override the colors specified by the page with your selections above". Click OK.
  3. View the web page and verify that the colors were changed to the ones you selected.
