Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Context-sensitive help must be provided when labels are not enough to explain all functionality.
  • Available context-sensitive help must be obvious to the user.
User Story: As a user with a disability, I want to have access to context-sensitive help so that I can understand how to perform an action.
Success: If you were a user with a disability, when completing a form, you would have access to context-sensitive help.

Testing Logic

Does the page have text inputs where the label is not sufficient to describe all functionality?

  • Yes; Does the text input require a specific data format?
    • Yes; See {{ site.data.3305.3305[33050102].name }}.
    • No; See {{ site.data.3305.3305[33050101].name }}.
  • No; Not applicable

Web Test Procedure

  1. Locate any forms which require text input.
  2. Check if the labels include all information need to understand how to complete the form.
  3. If not, verify that context-sensitive help is available.

Mobile App Test Procedure

Follow the procedure for Web.
