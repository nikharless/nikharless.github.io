Basic Requirements and Best Practices

= Requirement
= Best Practice

  • CAPTCHAs must have a text alternative that describes its purpose (but not the visual elements).
  • CAPTCHAs must have alternate forms for different types of sensory perception.
User Story:
  • As a blind screen reader user, I want to hear a short description of the CAPTCHA so that I know what it is.
  • As a blind screen reader user, I want to have an alternate method (such as audio) of completing the CAPTCHA, so that I am able to complete it.
Success:
  • If you could not see the CAPTCHA, when you read it with a screen reader, you would hear a description of the CAPTCHA.
  • If you could not see the CAPTCHA, when you read it with a screen reader, you could find an alternate method of completion (such as audio) and be able to verify yourself.

    • Yes; Does the text alternative describe the purpose of the CAPTCHA?

Web Test Procedure

Desktop

  1. Locate each CAPTCHA.
  2. Verify that it has a text alternative that describes its purpose.
  3. Verify that it includes an alternate non-visual form (e.g., audio).

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to the CAPTCHA.
  3. Verify that it has a text alternative that describes its purpose.
  4. Verify that it includes an alternate non-visual form (e.g., audio).
