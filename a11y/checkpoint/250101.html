Overview

This criterion does not apply to user agent gestures that are not controlled by the page authored content.

Examples of multipoint gestures:

  • Two-finger pinch zoom
  • Split tap
  • Two or more finger tap
  • Two or more finger swipe

Examples of path-based gestures:

  • Swiping
  • Sliders and carousels which require swiping in a particular direction
  • Gestures that trace a prescribed path
  • Drawing

Note: Drag-and-drop isn't path-based, since the path between the start and end doesn't matter.

Examples of acceptable single-pointer gestures:

  • Single taps and clicks
  • Double taps and clicks
  • Long presses
  • Click and hold
  • Dragging (not path-based)

Note: Keyboard controls are not sufficient to pass this criterion.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • All functionality must be operable with a single pointer.
  • Path-based gestures must not be required to use any functionality.
User Story: As a user with a movement disability, I want all functionality to be usable with one finger and reduced gestures so that I can operate it.
Success: If you were a user with a movement disability, when you need to interact with a page, you could do so with one finger and limited movement.

Testing Logic

Is there any functionality that uses a multipoint gesture?

  • Yes; Is the multipoint gesture essential?
    • Yes; PASS
    • No; Is there an alternate single point, non-path-based means of operation for the same functionality?
      • Yes; PASS
      • No; See {{ site.data.2501.2501[25010101].name }}.
  • No; Is there any functionality that uses a path-based gesture?
    • Yes; Is the path-based gesture essential?
      • Yes; PASS
      • No; Is there an alternate single point, non-path-based means of operation for the same functionality?
        • Yes; PASS
        • No; See {{ site.data.2501.2501[25010102].name }}.

Web Test Procedure

  1. Identify any functionality that can be triggered by multipoint or path-based gestures.
  2. For each such functionality, verify that the functionality can also be operated with the use of single-pointer actions that are not path-based.

Mobile App Test Procedure

Follow the Web test procedure.
