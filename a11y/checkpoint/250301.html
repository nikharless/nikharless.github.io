Overview

For the purpose of this criterion, the label is only a single string of text next to the control. Any other text that should be read along with the control, such as instructions and group labels, is part of the description.

Note: If the control fails 4.1.2 Name, Role, Value due to not having an accessible name and it has a visible label, it also fails this criterion.

Programmatic names are determined in the following order of priority:

  • aria-labelledby value
  • aria-label value
  • <label> text
  • title attribute value
  • control text:
    • textbox value
    • button text
    • combobox or listbox selected option text
    • range (spinbutton or slider)
      • aria-valuetext value
      • aria-valuenow value
      • value
  • pseudo elements (:before, :after)

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Accessible (programmatic) names must contain the text in the visible label.
  • The text of the label should be at the start of the accessible name.
User Story: As a voice control user, I want the accessible name to match the visible label so that I know what to say to activate the control.
Success: If you were a voice control user, when you wanted to activate a control, you would know the name of the control by the visible label.

Testing Logic

Are there controls on the page with visible labels made up of human language text (not symbols) or images of text?

  • Yes; Does the control have an accessible name?
    • Yes; Does the accessible name include the text that is presented visually?
      • Yes; PASS
      • No; See 2.5.3.1.1 Accessible name does not contain the visible label text.
    • No; See 2.5.3.1.2 Control with visible label does not have an accessible name.
  • No; PASS

Web Test Procedure

Desktop

  1. Locate all controls with a visible text label.
    • For the purposes of this checkpoint, when a text field has placeholder text as its only programmatic label, consider the placeholder value to be the field’s visible label.
  2. Compare the accessible name of the element to the visible text and verify that the entire visible text label is either:
    • An exact match to the accessible text, OR
    • Is contained within the accessible text (in the same order presented visually with no words inserted in between)

Mobile

  1. Locate all controls with a visible text label.
    • For the purposes of this checkpoint, when a text field has placeholder text as its only programmatic label, consider the placeholder value to be the field’s visible label.
  2. Using the screen reader, place focus on each control.
  3. Compare the accessible name of the element to the visible text and verify that the entire visible text label is either:
    • An exact match to the accessible text, OR
    • Is contained within the accessible text (in the same order presented visually with no words inserted in between)

Mobile App Test Procedure

Follow the Mobile Web test procedure.

Failures

  • 2.5.3.1.1 Accessible name does not contain the visible label text. (High)
  • 2.5.3.1.2 Control with visible label does not have an accessible name. (High)
