Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • All user submissions must be reversible, verifiable, or confirmable.
User Story: As a user with a disability, I want to be able to check, confirm, or reverse transactions so that I can avoid mistakes.
Success: If you were a user with a disability, when submitting transactions, you would have the opportunity to check, confirm, or reverse the transaction.

Testing Logic

Does the page content include submittable forms?

  • Yes; Is the data entered by the user checked for input errors and the user given the opportunity to correct them?
    • Yes; PASS
    • No; Is the user given the opportunity to review, confirm, and correct information before finalizing the submission?
      • Yes; PASS
      • No; Is the submission reversible?
        • Yes; PASS
        • No; See {{ site.data.3306.3306[33060101].name }}.
  • No; Not applicable

Web Test Procedure

Desktop

  1. Locate any submittable forms.
  2. Verify that the submission can be either:
    • Reversed by the user, OR
      • Instructions are provided explaining how to reverse or cancel the submission.
      • A keyboard-only user can reverse or cancel the submission.
    • Verified by the user, OR
      • Data entered by the user is provided for review before final submission, and the user is able to make changes to this data.
    • Confirmed by the user
      • A mechanism (such as a checkbox) is provided for the user to confirm the submission.
      • Validate that if the checkbox is left unchecked, the submission is not processed.

Mobile

Using VoiceOver or TalkBack, follow the Desktop Web procedure.

Mobile App Test Procedure

Follow the procedure for Mobile Web.
