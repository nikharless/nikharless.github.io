Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Text characters used to represent images or other characters must have a text alternative describing their meaning.
  • ASCII images and emoticons must be marked as images (role="img") so that screen readers will read the text alternative.
User Story: As a blind screen reader user, I want to hear the intended meaning of the characters rather than the literal reading so that I understand the content.
Success: If you could not see the text, when you read it with a screen reader, you would understand the intended meaning.

Testing Logic

Are there text look-alike characters on the page?

For example, the characters U+0033 (or &#x0033;) and U+04E0 (or &#x04E0) both look like the number "3", but the latter is a letter from the Cyrillic alphabet and will not be read as a "3".

  • Yes; Do the characters have a text alternative which represents their meaning?
    • Yes; PASS
    • No; See F1.1.1.i.a Text look-alike characters do not have a text alternative.
  • No; PASS

Is there text on the page which represents an image (such as ASCII art or emoticons)?

  • Yes; Does the text image have a text alternative which describes its meaning?
    • Yes; PASS
    • No; See F1.1.1.i.b Text representing image does not have a text alternative.
  • No; PASS

Web Test Procedure

  1. Locate any text characters which visually mimic other characters.
  2. Locate any text characters which visually represent an image.
  3. Verify that they have a text alternative that conveys the meaning rather than the literal interpretation of the characters.

Mobile App Test Procedure

  • Follow the procedure for Web Mobile.

Failures

  • F1.1.1.i.a Text look-alike characters do not have a text alternative. (Medium)
  • F1.1.1.i.b Text representing image does not have a text alternative. (Medium)
