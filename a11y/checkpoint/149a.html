Overview

Images of text are not just a problem for screen reader users which can be solved with text alternatives. Users with vision or reading disabilities often need to adjust the text in order to read it. This could include enlarging the text, changing the spacing, or changing the colors. Images of text are generally not adjustable. For this reason, real text must be used.

Exceptions include:

  • Essential images of text, such as:
    • Logotypes, branding, type/font samples
    • Graphs, screenshots, and diagrams which visually convey important information through more than just text
    • Photos or graphical representations of actual objects
  • Incidental text in images, such as:
    • A photo of a person speaking at a conference with their name visible on their nametag
    • A photo of a car at an intersection with a stop sign visible in the background
  • Redundant images: If the text in the image is available elsewhere on the page, then the image is redundant and should be tested against checkpoint 1.1.1.d Decorative Images (Text Alternative).

SVG Images

Text in an SVG may or may not be considered real text:

  • Text in a <text> element within an <svg>: PASS
  • Text drawn with a <path> element within an <svg>: FAIL
  • Text in a .svg file: FAIL

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Real text must be used instead of images of text, unless it is purely decorative or essential.
User Story: As a user with a visual impairment, I want text to be zoomable and customizable so that I can adjust it to be more easy for me to read.
Success: If you had a visual impairment, when you encountered text, you would be able to enlarge or adjust it to make it more easy to read.

Testing Logic

Are there any images of text on the page?

  • Yes; Is the presentation of the text essential (e.g., a logo)?
    • Yes; PASS
    • No; Is the image of text purely decorative?
      • Yes; PASS
      • No; See F1.4.5.a.a Image of text is used where real text is possible.
  • No; PASS

Web Test Procedure

Desktop

Use the Web Developer browser extension for Firefox or Chrome to check for images of text.

  1. Open the Web Developer extension, and select Images > Outline All Images.
  2. Verify that none of the outlined images are images of text.

Mobile Screen Reader

  1. Turn on the screen reader and read through the page.
  2. Verify that none of the text is identified as an image.

Mobile App Test Procedure

iOS

  1. In the OS Accessibility settings, select Display & Text Size > Larger Text.
  2. Set the slider to the largest text size (far right).
  3. View the app screen and verify that all text has resized.
  4. If it has not, turn on VoiceOver and check whether the text is identified as an image.

Android

  1. In the OS Accessibility settings, select Text and Display > Font size.
  2. Set the slider to the largest text size (far right).
  3. View the app screen and verify that all text has resized.
  4. If it has not, turn on TalkBack and check whether the text is identified as an image.

PDF Test Procedure

  1. Select all text in the document:
    • Windows: Ctrl+A
    • MacOS: Command+A
  2. Verify that all text is selected.
  3. Check if any unselected text is embedded in an image.

Failures
