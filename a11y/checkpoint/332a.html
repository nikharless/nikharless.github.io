Overview

This criterion applies to form inputs:

  • <input type="button">
  • <input type="checkbox">
  • <input type="color">
  • <input type="date">
  • <input type="datetime-local">
  • <input type="email">
  • <input type="file">
  • <input type="image">
  • <input type="month">
  • <input type="number">
  • <input type="password">
  • <input type="radio">
  • <input type="range">
  • <input type="reset">
  • <input type="search">
  • <input type="submit">
  • <input type="tel">
  • <input type="text">
  • <input type="time">
  • <input type="url">
  • <input type="week">

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Form inputs must have visible labels.
  • Form input options must have visible labels.
  • Labels should specify data format if it isn't clear.
  • Labels should be close to the form input.
  • Labels should always be visible for single select inputs, even when a selection is made and the single select element is collapsed.
  • Labels should be correctly marked up and associated with the form inputs. (For the requirement, see 1.3.1 Info and Relationships.)
  • Form inputs should have accessible names. (For the requirement, see 4.1.2 Name, Role, Value.)
User Story: As a user with a cognitive disability, I want form inputs to have visible labels so that I know what data is expected.
Success: If you were a user with a cognitive disability, when you encounter form inputs, labels are provided describing the expected input.

Testing Logic

Are there form inputs on the page?

  • Yes; Does each form input have a visible label?
    • Yes; Does the label remain visible when the input has a value?
      • Yes; Is the form input part of a group of inputs?
        • Yes; Does the group of inputs require a group label in order to be understood?
          • Yes; Does the group of inputs have a visible group label?
            • Yes; PASS
            • No; See F3.3.2.a.b Input group has no visible label and individual labels are insufficient.
          • No; PASS
        • No; See
      • No; See F3.3.2.a.c Visible label is not persistent.
    • No; See F3.3.2.a.a Form input has no visible label.
  • No; Not applicable

Web Test Procedure

  1. Locate any form inputs in the content.
  2. Verify that each individual form input has a visible label.
  3. If the purpose of individual inputs in a group of inputs is not clear from the individual labels alone, verify that there is a visible group label.
  4. Enter data for each control, and verify that the label remains visible.

Mobile App Test Procedure

Follow the procedure for Web.

PDF Test Procedure

  1. Locate any form inputs in the content.
  2. Verify that each individual form input has a visible label.
  3. If the purpose of individual inputs in a group of inputs is not clear from the individual labels alone, verify that there is a visible group label.

Failures

  • F3.3.2.a.a Form input has no visible label.
  • F3.3.2.a.b Input group has no visible label and individual labels are insufficient.
  • F3.3.2.a.c Visible label is not persistent.
