Overview

Completing an action on the down-event is only allowed when it is essential. An example of this would be keyboard emulation, where a character is typed as soon as the key is pressed rather than when it is released.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Single-pointer functionality must not use the down-event, must have a mechanism to abort or undo the function, or must be reversed on the up-event.
User Story: As a user with a disability, I want to be able to abort or undo an action so that I can avoid accidental activations.
Success: If you were a user with a disability, when you activated a function, you could abort or undo it.

Testing Logic

Does activating a control perform an action that cannot be undone?

  • Yes; Is the action triggered on the down-event?
    • Yes; Is completing the function on the down-event essential?
      • Yes; PASS
      • No; Does the up-event reverse the action of the down-event?
        • Yes; PASS
        • No; Is it possible to abort the action before completion (such as by dragging the pointer off of the control)?
          • Yes; PASS
          • No; See {{ site.data.2502.2502[25020101].name }}.
    • No; PASS
  • No; PASS

Web Test Procedure

  1. Find all controls on the page with actions that cannot be undone.
  2. If that control can be triggered by a single-pointer touch, verify that at least one of the following is true:
    • The action is not triggered on the down-event.
    • The action triggers on the up-event, and a mechanism is available to abort the function before completion or to undo the function after completion.
    • The up-event reverses any outcome of the preceding down-event.

Mobile App Test Procedure

Follow the Web test procedure.
