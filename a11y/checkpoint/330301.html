Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Error feedback must provide suggestions for error correction, if possible.
User Story: As a user with a cognitive disability, I want error messages to provide suggestions for how to correct the error so that I know how to correct it.
Success: If you were a user with a cognitive disability, when an error was automatically detected, suggestions would be provided to help you correct the error.

Web Test Procedure

Desktop

  1. Use ANDI to locate any forms in the content.
  2. Find all instructions and cues (textual and graphical) that are related to form components (e.g., groupings, order of completion, special conditions, qualifiers, format instructions, etc.).
  3. Deliberately trigger errors by entering user input that falls outside the required format or values or leaving required fields blank, and submit the form.
  4. If an input error is automatically detected, verify that the error message gives a suggestion about how to fix it.

Mobile

Using VoiceOver or TalkBack, follow the Desktop Web procedure.
