Overview

This checkpoint applies to server-side session time-outs and client-side time limits such as:

  • Page refresh
  • Content updates
  • Content changes
  • Time limits on user responses
  • Animated readable content
  • Automatically scrolling readable content

This checkpoint does not apply to:

  • Content which is available elsewhere on the page in static format
  • Video captions (assuming video is compliant with 2.2.2 Pause, Stop, Hide)
  • Carousels (compliant with 2.2.2 Pause, Stop, Hide)
  • Scrolling text that repeats
  • Non-timed server redirects (e.g., 3xx response codes)

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • If the content has a time limit, the user must be given options to turn off, adjust, or extend that time limit.
User Story: As a user with a disability, I want to be able to adjust time limits so that I have adequate time to complete tasks.
Success: If you had a physical disability, when you encounter a time limit, you are able to adjust it to allow you adequate time to complete the task.

Testing Logic

Is there a server-side session time limit?

  • Yes; Is there an option to turn off the time limit before encountering it?
    • Yes; PASS
    • No; Is there an option to adjust the time limit at least 10 times the default before encountering it?
      • Yes; PASS
      • No; Is the time limit longer than 20 hours?
        • Yes; PASS
        • No; Is the user warned before the time expires and given at least 20 seconds to extend the time limit with a simple action?
          • Yes; Can the user extend the time limit at least 10 times?
            • Yes; PASS
            • No; Is the time limit essential?
              • Yes; PASS
              • No; See {{ site.data.2201.2201[22010102].name }}.
          • No; See {{ site.data.2201.2201[22010101].name }}.
  • No; PASS

Is there a time limit controlled by a script on the page?

  • Yes; Is there an option to turn off the time limit before encountering it?
    • Yes; PASS
    • No; Is there an option to adjust the time limit at least 10 times the default before encountering it?
      • Yes; PASS
      • No; Is the time limit longer than 20 hours?
        • Yes; PASS
        • No; Is the user warned before the time expires and given at least 20 seconds to extend the time limit with a simple action?
          • Yes; Can the user extend the time limit at least 10 times?
            • Yes; PASS
            • No; Is the time limit required as part of a real-time event (i.e., an auction)?
              • Yes; PASS
              • No; Is the time limit essential?
                • Yes; PASS
                • No; See {{ site.data.2201.2201[22010104].name }}.
          • No; See {{ site.data.2201.2201[22010103].name }}.
  • No; PASS

Web Test Procedure

Desktop

  1. Identify any content that has a time out function.
  2. Interact with the site and then leave the site idle for a period of time until the time-out activates.
  3. Verify that at least one of the following is true:
    • The user can turn off the time limit before encountering it.
    • The user can adjust the time limit before encountering it over a wide range that is at least ten times the length of the default setting.
    • The user is warned before time expires and given at least 20 seconds to extend the time limit with a simple action, and the user can extend the time limit at least 10 times.

Desktop Screen Reader

  1. Turn on the screen reader.
  2. Follow the process for Desktop Keyboard.

Mobile

Screen Reader (iOS and Android)

Follow the process for Desktop Screen Reader.

Mobile App Test Procedure

Follow the procedure for Web Mobile.
