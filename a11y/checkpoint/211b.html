Overview

Shortcut keys must not conflict with existing device and screen reader shortcuts.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Shortcut keys must not conflict with existing device and screen reader shortcuts.
User Story: As a keyboard and screen reader user, I want to be able to access all functionality with the keyboard so that I can use the page.
Success: If you used a screen reader along with the keyboard, when you attempt to use the functionality on the page, it does not conflict with screen reader shortcuts.

Testing Logic

Does the page contain any page-specified shortcut keys and/or accesskey attributes?

  • Yes; Does the shortcut key and/or accesskey attribute conflict with browser or screen reader preset keys/shortcuts?
    • Yes; See F2.1.1.b.a Keyboard shortcut conflicts with user agent or assistive technology shortcut.
    • No; If it's an accesskey, does the attribute have a unique value?
      • Yes; PASS
      • No; See F2.1.1.b.b accesskey attribute does not have a unique value.
  • No; PASS

Web Test Procedure

Desktop

  1. Locate each occurrence of an accesskey attribute in the page.
    1. Open the Web Developer browser extension.
    2. Select Information > Display Access Keys.
  2. For each accesskey defined, verify that it is unique and does not conflict with existing browser and screen reader shortcut keys.

Failures

  • F2.1.1.b.a Keyboard shortcut conflicts with user agent or assistive technology shortcut. (Medium)
  • F2.1.1.b.b accesskey attribute does not have a unique value. (Low)
