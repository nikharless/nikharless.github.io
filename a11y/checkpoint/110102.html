Overview

Text alternatives depend on context. The same image might convey different information in different situations.

Example

If this image were used in an article about Frank Lloyd Wright, the text alternative might be "Frank Lloyd Wright's Fallingwater". However, if it were in a tutorial about photography, it might include information relevant to the lesson's content.

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Meaningful images must have a descriptive text alternative that conveys the same information as the image in context.
    • If the image does not provide any information not already conveyed elsewhere on the page, it may be considered redundant. Check against {{ site.data.1101.1101[110104].name }}.
  • Meaningful images must not be presented as background images.
  • If the image contains meaningful text, the text alternative must contain that exact text.
  • Text alternatives for meaningful images should not include words such as "image" or "graphic".
  • Trademarks, copyright symbols, etc. on an image should be included in the text alternative.
  • Text alternatives should be as succinct as possible.
User Story: As a blind screen reader user, I want to hear a description of the meaningful image so that I can get the information it conveys.
Success: If you could not see the image, when you read it with a screen reader, you would hear a meaningful description of the image.

Methods for providing text alternatives for meaningful images include the following:

  • alt attribute value
  • title attribute value
  • aria-label property
  • aria-labelledby property
  • CSS clipped text

Web Test Procedure

Desktop

  1. Run axe DevTools to flag missing text alternatives.
  2. Use ANDI: graphics/images to find all meaningful images on the page.
  3. Verify that the ANDI Output provides an accurate description of the image.

Screen Reader

  1. Locate each meaningful image on the page using one or more of the following methods:
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  2. Verify that each meaningful image has a text alternative which accurately describes its purpose.

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to each meaningful image.
  3. Verify that the name spoken by the screen reader accurately describes the purpose of the image.

PDF Test Procedure

Find missing text alternatives.

PDF Accessibility Checker

  1. Run PAC on the PDF.
  2. Select the WCAG tab and check if 1.3 Adaptable is marked as a failure.
  3. If it is, activate the Results in Detail button.
  4. Check if Alternative text for "Figure" structure elements is marked as a failure.

Adobe Acrobat Pro

  1. Select Accessibility > Accessibility Check in the Task pane to open the Accessibility Checker Options dialog.
  2. Select Alternate Text and Headings from the Category drop-down, and make sure Figures require alternate text is checked.

Verify the text alternatives are sufficient.

  1. Locate each meaningful image in the PDF using one or more of the following methods:
    • Visually
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  2. Verify that each image has a text alternative which accurately describes its purpose.
