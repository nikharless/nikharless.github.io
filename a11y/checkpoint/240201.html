Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Pages must have descriptive and informative page titles.
  • When page content is changed dynamically, such as in Single Page Applications (SPAs), the page title must be updated to reflect the current contents.
  • Each page title should be unique within the website.
User Story: As a screen reader user, I want to be able to hear a descriptive title for each web page so that I know what the content is without having to read through the page.
Success: If you were a screen reader user, when you have multiple browser tabs open, you would be able to differentiate between them.

Web Test Procedure

Desktop

  1. Run axe DevTools to flag a missing <title> element in the <head> of the page, OR
  2. Run the Page Title bookmarklet, OR
  3. Run ANDI
    1. Select structures from the drop-down.
    2. Select page title from the "more details" menu.
  4. Verify that the text in the <title> element properly describes the content or purpose of the page.
  5. If the page content updates dynamically, look at each version of the page content presented with the same url.
  6. Verify that the title text updates to match the currently displayed content.

Mobile Screen Reader

  1. With VoiceOver or TalkBack turned on, load or reload the web page.
  2. If the web page title is not spoken on page load, view the browser tabs. The page title should be read when a tab is focused.
  3. Verify that the page title spoken is an accurate description of the screenâ€™s content or purpose.
    • Note: It is not necessary to hear a role.

Mobile App Test Procedure (iOS)

Note: This is not applicable to Android apps, because there is no way for a tester without access to the code to be sure that anything is a screen title.

  1. With VoiceOver turned on, navigate to the screen.
  2. When the screen loads, verify that both of the following are true:
    • An earcon sounds (indicating a new screen has loaded), AND
    • One of the following is announced:
      • A descriptive screen title, OR
      • The first content element that serves as a descriptive heading, OR
      • An element within the navigation bar AND you can swipe to a descriptive heading or title.

Screen titles are usually the text in the app bar or navigation bar at the top of the screen, but could also be the first text element in the content.

PDF Testing

Look at the top bar of the Adobe Acrobat window or the browser tab and verify that the information displayed meaningfully describes the content of the document.

PAC

  1. Check the document with PAC.
  2. Select the WCAG tab.
  3. Look for 2.4 Navigable failures.
  4. Activate the Results in Detail button.
  5. Under WCAG, navigate to the 2.4.2 Page Titled entry and check whether it shows a failure.

Adobe Acrobat

  1. Select All tools > Prepare for accessibility > Check for accessibility.
  2. In the Accessibility Checker Options dialog, ensure Document title is showing in title bar is checked.
  3. Activate the Start Checking button.
  4. In the Accessibility Checker results pane, check under the Document header for Title.
  5. If there is no error, go to Document Properties:
    1. Select Menu > Document properties...
    2. Windows: Control+D
    3. MacOS: Command+D
  6. In the Document Properties dialog, go to the Description tab.
  7. Verify that a meaningful and descriptive Title value is provided for the document.
