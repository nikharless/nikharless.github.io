Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • If a single character key shortcut exists, then it can be turned off, remapped, or are only active when the relevant user interface component is in focus.
User Story: As a speech input user, I want to be able to access all functionality without accidentally triggering a shortcut.
Success: If you used speech input, when you use a page, you cannot accidentally trigger a single-character shortcut.

Testing Logic

Are there any keyboard shortcuts in the content that use a single letter (upper or lower case), punctuation, number, or symbol character?

  • Yes; Is the shortcut only active when the component has focus?
    • Yes; PASS
    • No; Is there a mechanism available to turn the shortcut off?
      • Yes; PASS
      • No; Is there a mechanism available to remap the shortcut to include non-printable keys (Ctrl, Alt, etc.)?
        • Yes; PASS
        • No; See 2.1.4.1.1 Single-character shortcut is always active and cannot be turned off or remapped.
  • No; PASS

Web Test Procedure

Desktop

  1. Identify all keyboard shorcuts implemented on the page. (Consult the developer or documentation rather than relying on discovering them.)
  2. Locate any keyboard shortcuts that are a single character, including upper and lower case letters, numbers, punctuation, and symbols.
  3. Verify that:
    • The shortcut is only active when the appropriate control is in focus, OR
    • The shortcut can be remapped to include non-printable keys (e.g., Ctrl, Alt), OR
    • The shortcut can be turned off.

Failures

  • 2.1.4.1.1 Single-character shortcut is always active and cannot be turned off or remapped. (High)
