Overview

Exceptions to this criterion include:

  • Undersized targets which are spaced so that a 24 CSS pixel diameter circle is centered on the bounding box of each, the circles do not intersect
  • When there is another control on the same page that performs the same function and meets the criterion
  • The control is inline with text
  • The control is the user agent default
  • The presentation of the control is essential

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The target size for controls must be at least 24 x 24 CSS pixels.
User Story: As a user with a movement disability, I want controls to have a larger target size so that I can more easily use them.
Success: If you were a user with a movement disability, when you interact with controls, you can easily hit the target area.

Testing Logic

Is the target size of all interactive elements at least 24 x 24 CSS pixels?

  • Yes; PASS
  • No; Is the size of the target determined by the user agent and not the author?
    • Yes; PASS
    • No; Is the target inline with a sentence or otherwise constrained by the line-height of non-target text?
      • Yes; PASS
      • No; Is there sufficient spacing between targets to make up for the size?
        • Yes; PASS
        • No; Can the function be achieved using a different control on the same page which passes this criterion?
          • Yes; PASS
          • No; Is the presentation of the target essential or legally required?
            • Yes; PASS
            • No; See {{ site.data.2508.2508[25080101].name }}.

Web Test Procedure

Desktop

  1. Locate any interactive controls on the page.
  2. Open the Developer tools:
    • Right-click the control and select Inspect, OR
    • From the Customize and control Google Chrome menu, select More tools > Developer tools, OR
    • Press Ctrl+Shift+I
  3. Verify that each control has a target area of at least 24 x 24 CSS pixels or falls under one of the exceptions.

Mobile

  1. Open the page on a desktop computer in Chrome.
  2. Open the Developer tools:
    • Right-click the control and select Inspect, OR
    • From the Customize and control Google Chrome menu, select More tools > Developer tools, OR
    • Press Ctrl+Shift+I
  3. Activate Toggle device toolbar (Ctrl+Shift+M) to view the page as a mobile device.
  4. Select a device from the Dimensions drop-down.
  5. Follow the procedure for Desktop Web.
