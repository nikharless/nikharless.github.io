Overview

Examples of moving, blinking, or scrolling content include:

  • Videos
  • Animations
  • Synchronized media presentations
  • Real-time games
  • Scrolling stock tickers

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • The user must be able to pause, stop, or hide automatically moving, blinking, or scrolling content that lasts longer than 5 seconds.
  • Real-time updates should resume with the most current information when unpaused.
  • Updates with static information should resume where it left off when unpaused.
User Story: As a user with an attention deficit disorder, I want to be able to pause, stop, or hide continuous animations so that I can focus on other page content.
Success: If you had an attention deficit disorder, when you encounter a continously moving animation, you are able to pause, stop, or hide it to allow you to focus on other page content.

Web Test Procedure

Desktop

  1. Identify all moving, blinking, or scrolling content that starts automatically, lasts for more than 5 seconds, and is presented along with other content.
  2. Verify that there is a mechanism available to stop, pause, or hide the content.

Desktop Screen Reader

  1. Turn on the screen reader.
  2. Follow the process for Desktop.

Mobile

Screen Reader (iOS and Android)
