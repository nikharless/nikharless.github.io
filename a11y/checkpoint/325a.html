Overview

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • Changes of context must only be initiated by the user, unless there is a means to turn off automatic changes of context.
User Story: As a screen reader user, I want the context to not change unexpectedly so that I know where I am.
Success: If you were a screen reader user, when you interacted with a web page, the context would not unexpectedly change.

Testing Logic

Is there an automatic change of context anywhere on the page?

  • Yes; Is there a mechanism available to turn off such changes?
    • Yes; PASS
    • No; Is the focus automatically removed or redirected?
      • Yes; See F3.2.5.a.a Focus is removed or redirected without user request.
      • No; Is a new window, tab, or dialog launched?
        • Yes; Does the window, tab, or dialog have an error message and a button that closes it returning the focus to the triggering component?
          • Yes; PASS
          • No; See F3.2.5.a.b A new window is launched without user request.
        • No; Is there a significant change of content, especially before the current focus?
          • Yes; See F3.2.5.a.c There is a significant change of content without user request.
          • No; Is a form automatically submitted?
            • Yes; See F3.2.5.a.d A form is automatically submitted without user request.
            • No; Is the page content automatically refreshed?
              • Yes; See F3.2.5.a.e Page content is refreshed without user request.
              • No; See F3.2.5.a.f A change of context occurs without user request.
  • No; PASS

Web Test Procedure

Desktop

  1. Verify that there is no unexpected change of context when the page loads.
  2. Using only the keyboard, focus on each control.
  3. Verify that there is no unexpected change of context when any control gains or loses focus.
  4. Using only the keyboard, change the setting or enter text for each control.
  5. Verify that there is no unexpected change of context when any control state or value changes.
  6. If there are any unexpected changes of context, verify that there is a mechanism available to turn them off.

Mobile

  1. Turn on the screen reader.
  2. Follow the procedure for Desktop by swiping through the content.

Mobile App Test Procedure

Follow the procedure for Mobile Web

Failures

  • F3.2.5.a.a Focus is removed or redirected without user request.
  • F3.2.5.a.b A new window is launched without user request.
  • F3.2.5.a.c There is a significant change of content without user request.
  • F3.2.5.a.d A form is automatically submitted without user request.
  • F3.2.5.a.e Page content is refreshed without user request.
  • F3.2.5.a.f A change of context occurs without user request.
