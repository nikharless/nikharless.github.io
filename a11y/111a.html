WCAG | 1.1.1 Non-text Content

All linked images must have a descriptive text alternative that conveys the same information as the linked image.

Success: If you could not see the image, you would still understand the destination, purpose, or function of the control.

Text alternatives for active images include the following:

  • alt attribute value
  • title attribute value
  • aria-label property
  • aria-labelledby property

Active images include the following:

  • Images inside a link
  • Images inside a button
  • <area> elements in an image group
  • <input type="image"> elements

Basic Requirements and Best Practices

= Requirement
= Best Practice

  • If a control is labeled with only an image, the image must have alternative text which describes the function or destination of the control.
  • If a control is labeled with both an image and text:
    • If the image provides information not conveyed by the associated text, the image's alternative text along with the visible text must together describe the function or destination of the control.
    • If the image does not provide any information not conveyed by the associated text, check against 1.1.1.d Alternative Text (Decorative Images).
  • If the image contains meaningful text, the text alternative should contain that exact text.
  • Text alternatives for linked images should not duplicate adjacent link text for the same destination. Combine the image and text into a single link and check the image against 1.1.1.d Alternative Text (Decorative Images).
  • Text alternatives for active images should not include words such as "link" or "button".
  • Trademarks, copyright symbols, etc. on an image should be included in the text alternative.
  • Text alternatives should be as succinct as possible.

Testing Logic

Are there images on the page which function as interactive controls?

  1. Yes; Does the image have alternative text?
    1. Yes; Does the alternative text repeat the text that it is programmatically associated with?
      1. Yes; Remove the alternative text.
      2. No; Is the alternative text necessary to understand the function of the control?
        1. Yes; Does the text alternative sufficiently describe the function of the control?
          1. Yes; PASS
          2. No; Change the text to sufficiently describe the function of the control.
        2. No; Remove the alternative text.
    2. No; Is alternative text necessary to understand the control's purpose?
      1. Yes; Add alternative text to the active image.
      2. No; PASS
  2. No; PASS

Is there an image map on the page with an <area> tag?

  1. Yes; Does the <area> tag have a descriptive text alternative?
    1. Yes; PASS
    2. No; Change the alternative text to text which succinctly describes the link's purpose.
  2. No; PASS

Web Test Procedure

Desktop

  1. Run axe DevTools or WAVE to flag missing text alternatives.
  2. Locate each interactive image on the page using one or more of the following methods:
    • Mouse: Hover over images, noting cursor change if they are interactive.
    • Keyboard: Use the Tab key to focus interactive images.
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  3. Verify that each image has a text alternative which accurately describes its purpose.

Mobile

  1. For iOS devices, turn off image recognition features.
  2. Navigate to each interactive image.
  3. Verify that the name spoken by the screen reader accurately describes the purpose of the image.

Mobile App Test Procedure

  • Follow the procedure for Web Mobile.
  • iOS: Verify that the button, button image, or link image trait is spoken, AND/OR hint text such as "Double tap to activate" is spoken.
  • Android: Verify that the button role is spoken, AND/OR hint text such as "Double tap to activate" is spoken.

PDF Test Procedure

Find missing text alternatives.

PDF Accessibility Checker

  1. Look for failures under Logical Structure > Alternative Descriptions > Alternative text for "Figure" structure elements.

Adobe Acrobat Pro

  1. To find missing text alternatives, select Accessibility > Accessibility Check in the Task pane to open the Accessibility Checker Options dialog.
  2. Select "Alternate Text and Headings" from the Category drop-down, and make sure "Figures require alternate text" is checked.

Verify the text alternatives are sufficient.

  1. Locate each interactive image in the PDF using one or more of the following methods:
    • Mouse: Hover over images, noting cursor change if they are interactive.
    • Keyboard: Use the Tab key to focus interactive images.
    • Windows / NVDA: Use the G key to cycle through images.
    • MacOS / VoiceOver: Use VO+Command+G to cycle through images.
  2. Verify that each image has a text alternative which accurately describes its purpose.

Failures

  • F1.1.1.a.a Interactive image is missing a text alternative. (High)
  • F1.1.1.a.b Image map <area> element is missing a text alternative. (High)
  • F1.1.1.a.c Server-side image map is missing a text alternative. (High)
  • F1.1.1.a.d Existing text alternative does not serve the same purpose as the interactive image. (Medium)
  • F1.1.1.b.c Existing text alternative does not include informative image text. (Medium)
  • F1.1.1.b.d Text alternative does not match dynamic image. (Medium)
